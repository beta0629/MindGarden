## 1. 목적

1. **채널 혼선 방지**: 초기에는 앱에 **전면 어드민**을 넣지 않는다. 구현·요구 범위가 흔들릴 때 **본 문서가 SSOT**로 “웹 본체 vs 앱 보조” 경계를 잡는다.  
2. **추후 확장**: 앱 **제한적 어드민**은 **§2.1 원칙**을 만족할 때만 추가하고, 항목마다 **웹 이관 링크** 또는 **읽기 전용** 여부를 명시한다.  
3. **체크아웃·카탈로그 구현 시** 어떤 기능을 **웹 / Expo / 공통 API**에 둘지 한 표로 고정한다.  
4. [ONLINE_PAYMENT_CATALOG_CHECKOUT_SPEC.md](./ONLINE_PAYMENT_CATALOG_CHECKOUT_SPEC.md) **§8.1 P0~P5**와 번호가 겹치지 않도록, **온라인 결제 문서의 Phase(P0~P5)**를 체크아웃 트랙의 공식 페이즈로 쓰고, **Expo 네이티브 기획서의 “Phase 5(테스트)”**와는 **별개**임을 명시한다.

### 2.1 추후 — 앱(Expo) **제한적** 어드민 원칙

아래를 **동시에** 만족할 때만 앱에 어드민 성격 화면을 추가한다.

| 원칙 | 설명 |
|------|------|
| **최소 권한** | `ROLE_ADMIN`이라도 앱에서는 **서브셋 API**만 호출(별도 스코프·기능 플래그 권장). |
| **얕은 UI** | **한 화면·한 목적**(예: 긴급 승인 1건, 오늘 처리 대기 건수, 푸시 딥링크로 연결된 단건 처리). **목록·폼 지옥·대량 편집**은 웹으로 이관 링크. |
| **읽기 우선** | KPI·알림 요약·대기열 조회는 **읽기 전용** 우선; 쓰기는 **멱등·사유 코드·감사 로그** 필수. |
| **웹 본체 유지** | 상품 마스터·가격 이력·환불 정책 편집·RBAC 전체 등 **WA-1~7 본질**은 **웹이 계속 SSOT**; 앱은 보조 통로. |
| **보안** | 앱 패키지에 **시크릿·PG 키 없음**; 민감 설정은 웹 또는 서버만. |

**후보 예시**(기획 승인 후 택일·조합):

- 오늘의 **처리 대기(N건)** 카드 + “웹에서 계속” CTA  
- **푸시/알림 → 단건 승인·확인** (이미 로그인된 ADMIN만)  
- **모바일 전용** “현장 체크인/출석 확인” 등 **도메인 좁은** 운영 액션(어드민 역할이지만 전면 CRUD 아님)

**지속 금지(앱에서 완결하지 않음)** — 정책 완화 시에도 **문서 개정**으로만 변경:

- 전체 **상품·가격·노출** CRUD를 앱만으로 완결  
- **ERP 대사·분쟁·부분취소** 전체 플로우를 소형 화면에만 우겨 넣기(웹 또는 전담 도구 권장)

### 2.2 원칙(공통)

- 백엔드 API는 **역할·테넌트**로 보호한다. 앱에 ADMIN UI가 없어도, **잘못된 토큰으로 ADMIN API 호출**은 서버에서 **403**으로 막는다 (기존 멀티테넌트 스킬 유지).  
- **초기**: `expo-app` 라우트 그룹에 **`(admin)`** 등을 두지 않는다. **추후** 도입 시 `(admin-lite)` 등 **별도 그룹명·가드·기능 플래그**로 웹 어드민과 혼동하지 않게 한다.

## 5. API·보안 (공통)

- **tenantId 필수** — ONLINE §1.3 및 멀티테넌트 스킬.  
- **클라이언트 금액 신뢰 금지** — 체크아웃 금액은 서버 `Order` 스냅샷 기준.  
- **역할 검증** — 전면 어드민용 `ROLE_ADMIN` API는 **웹 클라이언트가 본체**로 사용한다. 앱에서 **제한적 어드민**을 도입하면 **서브셋 엔드포인트**만 호출하고, 서버는 **스코프·감사 로그**로 구분한다(앱 전용 클라이언트 타입이 없으면 토큰·스코프만으로 구분).  
- **ERP** — `confirm-payment`, `amount-info`, `relatedTransactions` 대사 등 [core-solution-erp](../../.cursor/skills/core-solution-erp/SKILL.md) 유지.

## 6. 위임·구현 순서 (권장)

1. **core-planner**: 본 문서 + ONLINE §8을 **스프린트 백로그**로 쪼개고, 웹/Expo 병렬 가능 구간 표시.  
2. **core-designer**: P0 UI 골격(ONLINE §8.2) — **웹 어드민 + 내담자 PLP/PDP**, Expo **내담자**용 와이어; **추후** 앱 제한 어드민은 §2.1에 맞는 **별도 와이어(한 화면·한 목적)**.  
3. **core-coder**: Order/Payment 스키마·PG·웹 어드민 P4 → Expo EA-1~5 순; 앱 어드민 라이트는 **별도 라우트 그룹·API 스코프** 후행.  
4. **core-tester**: 멱등·tenant 교차·금액 변조·환불 시나리오; **초기** 앱에 **전면 ADMIN 라우트 없음** 정적 검사(`expo-app/app` 트리 grep 등). **추후** 제한 어드민 추가 시 §2.1 체크리스트 회귀.
