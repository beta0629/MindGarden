# Trinity 공개 파트너 입점 신청 폼 UI/UX 개선 스펙

## 1. 개요
본 문서는 Trinity 시스템의 공개 파트너 입점 신청 폼(Public Partner Onboarding Application Form)에 대한 UI/UX 개선 스펙을 정의합니다.

## 2. 목표
- `pencil-new.pen` 디자인(참조 노드: SqpCX, E7w72, qgIzo, eCumG)을 기반으로 한 UI/UX 전면 개편
- 사용자 친화적이고 직관적인 입점 신청 경험 제공

## 3. 주요 디자인 요소
- **좌우 스플릿 뷰 (Split View):** 화면을 좌우로 분할하여 한쪽에는 안내 및 비주얼 요소, 다른 한쪽에는 입력 폼을 배치하여 시각적 안정감 제공
- **3D 일러스트:** 브랜드 이미지를 강화하고 친근감을 주는 3D 일러스트레이션 적용
- **친화적 안내 문구:** 사용자가 신청 과정을 쉽게 이해할 수 있도록 돕는 따뜻하고 명확한 카피라이팅
- **헬퍼 텍스트 (Helper Text):** 각 입력 필드 하단에 입력 조건 및 예시를 제공하여 입력 오류 최소화

## 4. 제약사항 및 원칙
- **공개 폼 (Public Context):** 테넌트 로그인 컨텍스트가 없는 상태에서 접근 가능한 퍼블릭 페이지로 구현
- **하드코딩 금지:** 텍스트, 색상, 여백 등 모든 UI 요소의 하드코딩을 엄격히 금지하며, 디자인 시스템 토큰 및 다국어/상수 파일 활용

## 5. 기술 스택 및 아키텍처
- **프레임워크:** React
- **스타일링:** `unified-design-tokens.css` 기반의 CSS 변수 활용
- **컴포넌트 구조:** 아토믹 디자인(Atomic Design) 패턴 준수 (Atoms → Molecules → Organisms → Templates → Pages 계층화)
