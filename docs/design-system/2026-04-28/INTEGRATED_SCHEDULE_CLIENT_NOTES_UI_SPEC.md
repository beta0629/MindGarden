## 1. 개요

### 1.1 목적

통합 스케줄(`/admin/integrated-schedule`)에서 캘린더 이벤트 클릭 시 열리는 **`ScheduleDetailModal`**을 확장하여, 동일한 **`UnifiedModal`** 셸 안에서 **내담자 맥락의 특이사항**(제목·본문·유형·약속일·선택 금액) **목록 조회 및 CRUD 진입**을 제공한다. 별도 풀스크린·전용 모달 추가는 하지 않는다.

### 1.2 범위 (1차)

- **In**: 모달 내부 레이아웃·탭/패널·목록·빈/로딩/에러·P1·휴가(P2)·`adminNote`와의 시각·카피 분리, `noteType` 배지 토큰 매핑, 반응형 지침.  
- **Out (Phase 2)**: 캘린더 배지·「이번 주 약속」필터·대시보드 위젯 등은 SSOT 10절에 따라 본 문서 범위 밖.

### 1.3 탭 vs 단일 스크롤 — **권장 1안: 탭「상세 | 특이사항」**

| 대안 | 요약 |
|------|------|
| **권장: 탭** | 모달 본문 상단에 **세그먼트형 탭 2개** — 「상세」「특이사항」. 활성 패널만 렌더 영역을 채운다. |
| 대안: 단일 스크롤 섹션 | 기존 상세 블록 아래에 접이식 또는 연속 섹션으로 특이사항을 쌓는 방식. |

**권장 이유 (필수 관점 반영)**

1. **기존 액션과 충돌 최소화** — 예약 확정·취소·상담일지 등 기존 플로우는 대부분 「상세」맥락에 집중된다. 특이사항 목록이 길어져도 **「상세」탭에서는 기존 높이·스크롤 패턴을 유지**할 수 있어 푸터 액션과의 거리감이 안정적이다.  
2. **모바일** — 뷰포트가 좁을 때 **한 번에 하나의 역할**(상세 vs 목록/CRUD)만 보여 주어 스크롤 피로와 오조작을 줄인다. 탭 행은 **가로 스크롤 가능한 세그먼트**(최소 터치 높이 44px, `RESPONSIVE_LAYOUT_SPEC`의 모바일 기준)로 설계한다.  
3. **접근성** — `tablist` / `tab` / `tabpanel` 패턴으로 **역할·상태(선택됨)·패널 연결**을 표준적으로 표현할 수 있다. 단일 긴 스크롤은 동일 모달 내 여러 **랜드마크**가 섞여 스크린 리더 탐색 비용이 커질 수 있다.  
4. **SSOT 4절 정합** — 오케스트레이션 문서가 **「탭 또는 본문 내 접이식」** 중 탭을 먼저 나열하며, 기존 모달 확장을 권장한다.

**단일 스크롤을 택할 만한 경우** (비권장이 아닌 예외 검토용): 팀이 탭 포커스 관리 구현 부담을 줄이고, 특이사항이 항상 0~1건 수준으로 제한될 때. 1차에서는 **탭 권장**으로 고정한다.

## 2. 정보 구조(IA)

### 2.1 모달 루트 (`UnifiedModal`)

- **제목 영역**: 기존과 동일하게 일정/이벤트 맥락 제목(코더가 유지).  
- **부제/메타**(선택): 날짜·시간·내담자명 등 기존 패턴 유지.  
- **본문 (`mg-modal__body`)**  
  1. **탭 바** (휴가 이벤트 시에는 아래 5.4대로 탭 1개만 또는 「특이사항」비표시)  
  2. **패널 A — 상세**  
     - 기존 스케줄 상세 필드 블록  
     - **구역 분리**: 「예약 확정용 메모(`adminNote`)」블록과 **물리적으로 인접하되 카피·제목으로 명확히 구분** (아래 3.3)  
     - 기존 주요 액션으로 이어지는 버튼 행(모달 하단 `actions`와의 관계는 기존 유지)  
  3. **패널 B — 특이사항**  
     - 상단: 짧은 설명 문구(1줄) + **「추가」** 주조 버튼(우측 정렬 가능)  
     - 본문: 목록(카드/행) 또는 빈/로딩/에러 영역  
- **푸터 (`mg-modal__actions`)**: 기존 모달 전역 액션 유지. 특이사항 **작성·수정은 보통 별도 `UnifiedModal` 확인/폼 모달**로 열리는 패턴을 허용(SSOT: 중첩은 **기존 확인 모달 패턴만**). 본 스펙은 **목록 + 진입 버튼**까지만 1차 와이어에 포함.

### 2.2 특이사항 목록 항목(카드 1건)

표시 우선순위(한 줄 요약 + 메타):

| 우선 | 요소 | 비고 |
|------|------|------|
| 1 | **제목** (`title`) | `var(--mg-color-text-main)`, 본문 굵기 600 근접 |
| 2 | **`noteType` 배지** | 칩/배지, 토큰 매핑은 [5.6절](#56-notetype-배지칩-색상-계층-토큰) |
| 3 | **약속일** (`promiseDate`) | 있을 때만, 라벨 + 값, `var(--mg-color-text-secondary)` |
| 4 | **금액** (`amount` + 통화) | 있을 때만, **표시 경계·마스킹**은 `COMMON_DISPLAY_BOUNDARY` 및 기존 ERP/스케줄 화면과 동일 정책(코더 적용) |
| 5 | **본문 미리보기** (`body`) | 1~2줄 클램프, 보조 색상 |

행 우측 또는 케밥: **수정** / **삭제**(정책에 따름). 터치 영역 44px 이상.

### 2.3 CRUD 진입 (1차)

- **생성**: 「특이사항」패널 상단 **주조 버튼** — 라벨 예: **「특이사항 추가」**.  
- **수정**: 목록 항목의 **텍스트 버튼 또는 아이콘+라벨**(스크린 리더용 `aria-label`).  
- **삭제**: Destructive 보조 동작 — `var(--mg-error-600)` 계열 텍스트/아웃라인만 사용(배경 남발 금지), 확인은 기존 UnifiedModal 확인 패턴.

## 3. 와이어 설명(텍스트)

### 3.1 데스크톱(뷰포트 ≥ 768px)

```
┌──────────────────────────────────────────────┐
│ [X]  모달 제목                                │
│      부제(선택)                               │
├──────────────────────────────────────────────┤
│  [ 상세 ]  [ 특이사항 ]   ← 탭, 활성은 주조   │
├──────────────────────────────────────────────┤
│  (활성 패널만 표시)                            │
│  · 상세: 기존 필드 + adminNote 구역(3.3)      │
│  · 특이사항: 설명 + 추가 버튼 + 목록/상태     │
├──────────────────────────────────────────────┤
│  [ 기존 푸터 액션들… ]                        │
└──────────────────────────────────────────────┘
```

- 탭 컨테이너: 본문 최상단, 하단 **1px** `var(--mg-color-border-main)`로 패널과 구분.  
- 패널 내부: 섹션 블록 규칙 — 배경 `var(--mg-color-surface-main)`, 테두리 `var(--mg-color-border-main)`, **radius 16px**, 패딩 **24px**, 내부 gap **16px** (`PENCIL_DESIGN_GUIDE` 섹션 블록).  
- 섹션 제목: 좌측 **세로 악센트 4px** `var(--mg-color-primary-main)` + 제목 텍스트.

### 3.2 모바일(뷰포트 768px 미만)

- 탭: **가로 한 줄**, 탭이 넘치면 **가로 스크롤**(스크롤바는 시각적으로 최소화해도 됨, 포인터/터치로 이동 가능). 각 탭 최소 너비 확보로 **44px 높이** 충족.  
- 목록 카드: **1열 스택**, 카드 간 gap **12px**(`RESPONSIVE_LAYOUT_SPEC` 모바일 섹션 gap에 맞춤).  
- 「추가」버튼: 모바일에서 **전체 너비** 또는 우측 고정 중 택일 — 권장은 **특이사항 패널 상단 한 줄**(좌: 설명, 우: 버튼); 좁으면 버튼을 다음 행으로 **풀와이드** 스택.

### 3.3 `adminNote`(예약 확정·입금 확인) vs 신규 특이사항 — **시각·카피 분리**

| 구분 | 제목(카피 예시) | 성격 | 시각 |
|------|-----------------|------|------|
| **기존 `adminNote`** | **「예약 확정 메모(일회성)」** 또는 제품 확정 문구 | 당일 입금 확인 등 **일회성** | 섹션 악센트: **주조** `var(--mg-color-primary-main)` 유지. 본문은 단일 textarea 패턴 유지. |
| **신규 특이사항** | **「내담자 특이사항(지속 기록)」** | 약속·리스크 등 **누적 관리** | 「특이사항」**탭**으로 물리 분리. 탭 미선택 시 상세 화면에서 **혼동 방지**. 목록 카드 악센트는 **포인트** `var(--mg-color-accent-main)` 또는 **보조** `var(--mg-color-secondary-main)` 중 하나로 `adminNote` 블록과 차별(코더는 한 종류로 통일). |

보조 문구 예시(상세 탭 `adminNote` 인근): *「이 메모는 예약 확정 처리용이며, 내담자 특이사항 목록과 연동되지 않습니다.」* — 짧게 1줄, `var(--mg-color-text-secondary)`, 12px.

## 4. 컴포넌트 계층 제안 (atoms / molecules)

### 4.1 Atoms

- **TabButton**: 선택/미선택 상태, `border-radius` 10px, 패딩 수평 16px·수직 12px.  
- **Badge / Chip** (`noteType`): 패딩 좌우 8px, 높이 ~24px, radius 8px, 테두리 또는 연한 배경만(토큰 참조).  
- **TextLink** / **IconButton**: 수정·삭제 진입.  
- **SectionTitleRow**: 악센트 바 + 제목 + 선택적 우측 액션.

### 4.2 Molecules

- **ScheduleDetailTabBar**: 두 탭 + (휴가 시) 단일 탭 또는 특이사항 숨김.  
- **AdminNoteSection**: 기존 라벨·textarea·안내 한 줄(신규).  
- **ClientNoteListToolbar**: 설명 텍스트 + 「특이사항 추가」버튼.  
- **ClientNoteListItem**: 배지 + 제목 + 메타 행 + 미리보기 + 액션.  
- **EmptyStateBlock** / **InlineErrorBanner** / **LoadingSkeletonRow**: 상태용.

### 4.3 Organisms (코더가 배치할 위치)

- **`ScheduleDetailModal` 내부**  
  - `Organism: ModalBodyWithTabs` — 탭 상태 + 패널 스위치  
  - `Organism: ClientNotesPanel` — 툴바 + 목록 + 상태 블록

공통 모듈 우선: 기존 **`ContentHeader`/`BadgeSelect`/`UnifiedModal`** 등 재사용 가능 여부는 `docs/standards/COMMON_MODULES_USAGE_GUIDE.md`에 따름. 배지가 공통 컴포넌트와 겹치면 동일 스타일 토큰만 맞출 것.

## 5. 상태별 스펙

### 5.1 빈 목록

- **카피(제목)**: **「등록된 특이사항이 없습니다」**  
- **보조**: *「상담비 수령 약속 등 잊기 쉬운 일을 남겨 두세요.」* (`var(--mg-color-text-secondary)`, 12~14px)  
- **주 액션**: 「특이사항 추가」— `var(--mg-color-primary-main)` 배경, 텍스트 `var(--mg-color-text-on-primary)` 또는 B0KlA 모달 대비 규칙에 맞는 온프라이머리 텍스트 토큰.  
- **시각**: 빈 상태 일러스트는 **필수 아님**. 섹션 블록 중앙 정렬 텍스트 + 버튼으로 충분.

### 5.2 로딩

- **카피**: 스크린 리더용 **「특이사항 목록 불러오는 중」** (`aria-live="polite"` 영역).  
- **시각**: 목록 자리에 **스켈레톤 3~5줄**(회색 막대 `var(--mg-color-border-main)` 배경, `surface` 대비) 또는 스피너(주조 색). **탭 전환은 막지 않음** — 사용자가 「상세」로 이동 가능.  
- **중복 요청**: 동일 패널 내 로딩만 표시, 상세 탭 콘텐츠는 기존대로.

### 5.3 API 에러

- **위치**: 「특이사항」패널 상단 또는 목록 상단 **인라인 배너** (섹션 블록 내부).  
- **카피(제목)**: **「특이사항을 불러오지 못했습니다」**  
- **보조**: *「잠시 후 다시 시도해 주세요. 문제가 계속되면 관리자에게 문의해 주세요.」*  
- **액션**: **「다시 시도」** — 아웃라인 버튼(`var(--mg-color-border-main)` 테두리).  
- **색**: 경고/에러 구분 시 배경 `var(--mg-error-50)` 수준(토큰 존재 시), 테두리 `var(--mg-error-200)`, 텍스트 `var(--mg-error-700)` — 구현 가능한 토큰 세트로 코더 매핑.

### 5.4 휴가 이벤트 (SSOT 9절 P2)

**분기**: `extendedProps.type === 'vacation'`(코더가 실제 키 확인).

| 방식 | 스펙 |
|------|------|
| **권장 A: 숨김** | 「특이사항」**탭 자체 비표시**. 탭 바에는 「상세」만 남거나, 탭 바 전체를 숨기고 본문만 상세로 동일. |
| **대안 B: 비활성 + 이유** | 「특이사항」탭은 보이나 `aria-disabled="true"`, 클릭 불가. 옆 또는 툴팁/패널 자리에 *「휴가 일정에서는 내담자 특이사항을 등록할 수 없습니다.」* |

**권장 A** 이유: 휴가에는 내담자 맥락이 없어 CRUD가 무의미하므로 **탐색 노이즈 제거**. 제품에서 탭 개수 일관성이 필요하면 **B** 채택 가능 — 코더는 SSOT P2만 충족하면 됨.

### 5.5 `clientId` 없음 (SSOT 9절 P1)

- **목록 영역**: 빈 목록과 동일 레이아웃이 아닌 **안내 전용 블록** 권장.  
- **카피(제목)**: **「내담자가 연결되지 않은 일정입니다」**  
- **보조**: *「매칭·일정 연결 후 특이사항을 남길 수 있습니다. 지금은 일정·매칭 정보만 확인할 수 있어요.」* (API가 `mappingId` 등으로 조회 가능해지면 보조 문구를 제품 문구로 조정)  
- **액션**: 읽기 전용이면 **「닫기」만** 또는 통합 스케줄의 **매칭 설정으로 이동** 링크(해당 페이지가 있을 때만 — 네비게이션은 코더).

### 5.6 `noteType` 배지/칩 색상 계층 (토큰)

hex 나열 지양. **코드값 → 시맨틱 토큰** 매핑(라벨은 공통코드 문자열).

| `noteType`(예시) | 의미 계층 | 배경/테두리/텍스트 토큰 제안 |
|------------------|-----------|------------------------------|
| `PAYMENT_PROMISE` | 약속·금전 — **강조** | 배경: `var(--mg-color-accent-main)`의 **저채도 10~15% 오버레이**가 불가하면 `var(--mg-warning-100)` + 텍스트 `var(--mg-warning-800)` (금전 인지) |
| `ATTENDANCE` | 일반 운영 | 배경: `var(--mg-color-surface-main)` + 테두리 `var(--mg-color-border-main)` + 텍스트 `var(--mg-color-secondary-main)` |
| `RISK` | 주의 | 배경: `var(--mg-error-50)` + 텍스트 `var(--mg-error-700)` + 테두리 `var(--mg-error-200)` |
| `OTHER` / 기본 | 중립 | 배경: `var(--mg-primary-100)` + 텍스트 `var(--mg-color-primary-main)` **또는** `var(--mg-color-text-secondary)` 단일 톤 칩 |

구현 시 **한 컴포넌트**에서 `variant="payment" | "attendance" | "risk" | "neutral"`로 매핑하면 하드코딩 분산을 줄인다. **B0KlA 팔레트 밖 색 금지** — 위는 모두 `unified-design-tokens.css`에 존재하는 스케일·시맨틱 계열.

## 6. 접근성(a11y)

### 6.1 탭

- **역할**: 컨테이너 `role="tablist"`, 각 탭 `role="tab"`, 패널 `role="tabpanel"`.  
- **상태**: 선택된 탭 `aria-selected="true"`, 나머지 `false`.  
- **`tabpanel`**: `aria-labelledby` 해당 탭 id.  
- **포커스**: 탭 간 이동 시 **roving tabindex** 또는 키보드 화살표(디자인 시스템 기존 탭 패턴과 통일).  
- **모달**: 기존 `UnifiedModal` 포커스 트랩 유지; 탭 전환 시 포커스는 **선택 탭에 유지** 또는 패널 첫 포커스 가능 요소로 이동(팀 규칙 단일화).

### 6.2 라벨·이름

- 「특이사항 추가」: 버튼에 명시적 visible label.  
- 수정/삭제: 아이콘만일 경우 **`aria-label`** 예: 「특이사항 수정, {제목}」.  
- `adminNote` textarea: 기존 라벨 유지 + 상단 안내 문구는 **`id` + `aria-describedby`**로 연결.

### 6.3 라이브 리전

- 로딩 완료/에러 전환: `aria-live="polite"`로 목록 영역 갱신 알림.  
- 삭제 성공: 토스트 또는 인라인 메시지 시 동일.

### 6.4 색만으로 정보 전달 금지

- `noteType`은 **색 + 텍스트 라벨** 병행.
