# Core Solution 상담센터 — 영상 합성·뒷부분 컷 가이드 (Omni + 화면녹화)

**목적**: Google Omni B-roll과 화면녹화를 **한 타임라인에 합치고**, 본편 후반을 **잘라** 60·90초 숏폼을 만드는 실무 절차.  
**프롬프트 SSOT**: [CORE_SOLUTION_GOOGLE_OMNI_PROMPTS.md](./CORE_SOLUTION_GOOGLE_OMNI_PROMPTS.md)  
**타임코드 SSOT**: [촬영 스크립트 §4.0·§4.2·§4.3](./CORE_SOLUTION_COUNSELING_CENTER_VIDEO_SHOOTING_SCRIPT.md)  
**최종 갱신**: 2026-07-15

---

## 1. 합성 전 준비물

| # | 항목 | 확인 |
|---|------|------|
| 1 | 화면녹화 `cut-01` … `cut-11` (개발 서버·PII 익명화) | ☐ |
| 2 | Omni `omni-01` … `omni-10` (또는 사용분) — 화면 글씨·로고 없음 | ☐ |
| 3 | 내레이션 WAV/AI 보이스 — 스크립트 §4.1 문장 | ☐ |
| 4 | 자막 SRT — 본편 §4.2 / 숏폼 §4.3 | ☐ |
| 5 | CTA·로고 PNG (투명 배경) — **실사 브랜드 자산만**, AI 생성 로고 금지 | ☐ |

프로젝트: **1920×1080 · 30fps** (앱 세로 원본은 타임라인에 레터박스 또는 별도 9:16 시퀀스).

## 2. 트랙 템플릿

```
V3  Titles / Logo / CTA text     ← 편집기만
V2  Omni B-roll                  ← O1–O10
V1  Screen recordings            ← cut-01 … cut-11
A1  Narration
A2  Bed / ambience (optional)
S1  Subtitles (SRT)              ← burn-in은 최종 export 시에만
```

## 3. 본편 조립 순서 (권장 15분 러프컷)

1. **빈 시퀀스**에 §4.2 절대 시각 마커를 찍는다.  
   `OP | 01 | 02 | 03 | 04 | 05 | 06 | 07 | 08 | 09 | 10 | 11 | ED`
2. **V1**에 컷을 순서대로 올리고 §5.1 편집 초로 트림.  
3. **A1** 내레이션을 올리고 컷 경계를 0.2초 단위로 맞춤.  
4. **S1**에 §4.2 SRT 임포트 — **이 타임코드를 기준 잠금**.  
5. **V2** Omni를 [프롬프트 문서 §4 매핑](./CORE_SOLUTION_GOOGLE_OMNI_PROMPTS.md)대로 삽입.  
   - Omni가 길면 **Omni를 자름**. 자막 OUT을 늘리지 않음.  
6. **V3** OP 타이틀(0:00–0:10), ED CTA(3:32–3:38).  
7. V1↔V2 디졸브 8–12f. 컷 01·03·05 본문은 V1 불투명 유지.  
8. 러프 재생 → 립싱크·자막만 수정 (Omni 길이로 스토리 늘리지 않음).

### 레이어 규칙 (한 줄)

> **증거(화면) > 분위기(Omni) > 글자(타이틀)**. 같은 순간에 셋이 경쟁하면 증거를 남긴다.

## 5. 뒷부분 자르기 — 본편 → 숏폼

### 5.1 원칙

- 본편 타임라인은 **보존**한다. 숏폼은 **복제 시퀀스**에서 작업.  
- “뒷부분 영상은 편집해서 잘라야 한다” = 센터장 숏폼에서는 **앱 후반(대략 1:49 이후)** 을 통째로 제거하거나 티저만 남긴다.  
- Outro/CTA는 **숏폼 끝에 1회**. 본편 ED 클립을 붙인 채 또 ED를 붙이지 않는다.

### 5.2 자를 지점 (본편 절대 시각)

| 작업 | In | Out | 설명 |
|------|-----|-----|------|
| 앱 블록 제거 (센터장 A안) | **1:49:00** | **3:32:00** | 컷05–11 삭제. 필요 시 1:49에 O6+컷05 **8초만** 재삽입 |
| 웹만 남기기 | 0:10 | 1:49 | OP·컷01–04 (+OP Omni) |
| 내담자만 | 2:39 | 3:24 | 컷08–10 + 앞뒤 Omni |
| ED만 단독 export 금지 시 | 3:32 | 3:38 | 숏폼 끝으로 **복사**할 때 원본 본편 ED는 그대로 둠 |

### 5.3 60초 — 남길 / 버릴

**A안 센터장 (웹 중심)** — 스크립트 §4.3·§5.2와 동일

| 남김 | 버림 |
|------|------|
| OP(짧게), 01, 03, 04, 05 티저 8초, ED 2–3초 | 02(또는 4초로 축약), 06–11 전체, 긴 O7–O8 |

**B안 상담사**

| 남김 | 버림 |
|------|------|
| 05, 06, 03(브릿지), 07, O6, ED | 01–02, 04, 08–11 |

**C안 내담자**

| 남김 | 버림 |
|------|------|
| 08, 09, 10, 01/03 짧은 B-roll, ED | 02, 04–07, 11 |

### 5.4 CapCut / Premiere에서 “후반 잘라내기” 클릭 순서

1. 본편 시퀀스 **복제** → 이름 `promo-60-director` 등.  
2. playhead를 **1:49**에 두고 **razor** → 오른쪽(후반) **삭제**(ripple delete).  
3. 끝이 잘리면 **O6(2초)+cut-05(6–8초)+O9(2초)+CTA**만 붙인다.  
4. §4.3 A안 SRT를 **새 시퀀스에 다시 임포트** (본편 SRT를 그대로 쓰면 타임코드가 어긋남).  
5. 총 길이가 60±1초인지 확인. 넘으면 컷03을 줄인다.

### 5.5 90초 확장

1. A안 60초 복제 → `promo-90-mix`.  
2. 컷05 티저를 **12–15초**로 늘리고 O7(2초)+컷08(8초) 삽입.  
3. ED 3초 유지. 90 초과 시 컷03을 18초로.

### 5.6 Outro/CTA 중복 방지 체크리스트

- [ ] 숏폼에 본편 ED 클립이 **두 번** 있지 않다  
- [ ] CTA 문구 burn-in이 Omni 파일에 없다  
- [ ] V3 CTA가 마지막 **2–4초**만이다  
- [ ] 내레이션이 CTA보다 먼저 끝난다  
- [ ] 페이드 아웃 0.3–0.5초

## 6. 오디오 · 자막 주의

- Omni와 화면 전환 시 **내레이션 중간에서 끊지 않음** — 문장 경계(스크립트 §4.1)에서만 레일로드.  
- 자막은 **화면이 Omni여도** 같은 IN–OUT(스크립트 §4.0.3).  
- 숏폼은 **§4.3 SRT만** — 본편 SRT를 슬라이스하지 말 것.
