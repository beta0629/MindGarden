## 📌 개요

MindGarden 프로젝트의 표준화 문서(43개)를 기반으로 실제 코드베이스를 표준에 맞추는 실행 계획입니다.  
우선순위에 따라 단계별로 표준화를 진행합니다.

### 참조 문서
- [표준 문서 목록](../standards/README.md) - 43개 표준 문서
- [표준화 현황 검토 보고서](../standards/STANDARDIZATION_REVIEW_REPORT.md)

## 🎯 표준화 목표

### 전략 목표
1. **보안 강화**: 보안 표준 준수 및 취약점 제거
2. **아키텍처 통일**: 테넌트 시스템 완전 적용, 브랜치 코드 제거
3. **코드 품질 향상**: 하드코딩 제거, 표준 패턴 적용
4. **UI/UX 통일**: 디자인 시스템 적용, 반응형 개선

### 성공 지표
- 표준 준수율: 100%
- 보안 취약점: 0개
- 하드코딩: 0개
- 브랜치 코드 사용: 0개

## 📊 현재 상태 분석

### 발견된 표준 위반 사항

#### 🔴 최우선 (Critical)

1. **브랜치 코드 사용** ⚠️
   - `TenantContextFilter.java`: `branchCode`, `branchId` 추출 로직
   - 여러 Repository에서 브랜치 필터링 사용
   - **표준**: 브랜치 개념 완전 제거, `tenant_id`만 사용

2. **하드코딩된 역할 이름** ⚠️
   - Backend: 5+ 파일에서 문자열로 역할 체크
   - Frontend: 14 파일에서 역할 문자열 사용
   - **표준**: 공통코드 테이블 또는 enum 사용

3. **비표준 API 경로** ⚠️
   - 약 70개 컨트롤러가 `/api/`만 사용 (표준: `/api/v1/`)
   - **표준**: 모든 API는 `/api/v1/` 접두사 사용

#### 🟡 높은 우선순위 (High)

4. **하드코딩된 값들**
   - 색상값: 50+ 파일
   - 상태값: 다수
   - URL/경로: 일부
   - **표준**: CSS 변수, 환경 변수, 공통코드 사용

5. **에러 처리 불일치**
   - 일부 컨트롤러에서 try-catch 직접 사용
   - 커스텀 예외 미사용
   - **표준**: GlobalExceptionHandler 사용, 커스텀 예외 사용

6. **프론트엔드 컴포넌트 표준화 미비**
   - 템플릿 미사용
   - 하드코딩된 로딩바
   - 비표준 버튼 사용
   - **표준**: 컴포넌트 템플릿, 표준 컴포넌트 사용
