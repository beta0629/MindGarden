# 어드민 대시보드 V2 시각화 강화 제안

**대상**: AdminDashboardV2.js 메인 콘텐츠  
**기준**: 어드민 대시보드 샘플(B0KlA), mindgarden-design-system.pen, unified-design-tokens.css  
**산출**: 제안 목록 + 와이어프레임/블록 배치 설명 (코드·CSS 수정 없음)

---

## 1. 현재 V2 vs 샘플 갭 분석

### 1.1 어드민 대시보드 샘플(B0KlA) 구조 (참조 URL 기준)

| 블록 순서 | 영역 | 구성 요소 | 비고 |
|-----------|------|-----------|------|
| 1 | **헤더** | 제목 + 부제목 / 우측: 통합 검색 + 아이콘 그룹(캘린더, 알림, 테마) | 상단 바 풍부함 |
| 2 | **KPI Row** | 3카드: 총 사용자, 이번 달 예약, 상담 완료율 (트렌드 배지) | 한 줄 타일 |
| 3 | **Chart + Counselor Row** | 2열(2fr : 1fr): 왼쪽 상담 현황 추이(월/주 토글 + 차트), 오른쪽 우수 상담사 평점(리스트+진행바) | 시각화 밀도 |
| 4 | **Admin Grid** | 섹션 제목 + 4카드(신규 입주사, 상담사 승인, 공지 발송, 시스템 설정) | 진입점 명확 |

### 1.2 Admin Dashboard V2 현재 구조

| 블록 순서 | 영역 | 구성 요소 | 비고 |
|-----------|------|-----------|------|
| 1 | **ContentHeader** | 제목 "대시보드 개요" + 부제목만 | 검색·아이콘 없음 |
| 2 | **ContentKpiRow** | 4카드: 총 사용자, 예약된 상담, 완료율 등 | KPI는 있음 |
| 3 | **Pipeline** | AdminMetricsVisualization variant="pipeline" — 5단계 카드형 파이프라인 | 샘플에 없는 비즈니스 블록 |
| 4 | **차트 + 상담사** | 2열: 상담 현황 추이(막대 1종, 월/주 토글) + 우수 상담사 평점 | 샘플과 유사 |
| 5 | (조건부) | 환불 현황 ContentSection + StatCard 4개 | HIDE 시 숨김 |
| 6 | **ManualMatchingQueue** | 미배정 내담자 수동 매칭 목록 | 샘플에 없음 |
| 7 | **Detail Row** | 입금 대기 목록 + 스케줄 대기 목록 | 샘플에 없음 |
| 8 | **관리 기능** | ContentSection + admin grid 카드들 | 샘플의 Admin Grid에 대응 |

### 1.3 갭 요약

| 구분 | 샘플有 / V2無 | V2有 / 샘플無 | 시각화 관점 갭 |
|------|----------------|----------------|-----------------|
| **상단** | 통합 검색, 캘린더/알림/테마 아이콘 | — | V2는 헤더가 단순해 “대시보드다움”이 약함 |
| **KPI** | 3카드, 트렌드 강조 | 4카드 | 개수·트렌드 표시는 V2가 동등 이상 |
| **중단** | 차트 1 + 상담사 1 | 파이프라인 + 차트 1 + 상담사 1 | V2는 파이프라인으로 단계 가시화는 우수, **차트/위젯 다양성은 부족** |
| **차트** | 1종(추이) | 막대 1종(완료 추이) | **라인·도넛·비율 등 추가 차트 없음** |
| **그리드 타일** | Admin Grid(기능 진입) | 동일 개념 있음 | **지표용 그리드(variant=grid) 미구현** |
| **하단** | Admin Grid로 종료 | 매칭 큐, 입금/스케줄 대기, 관리 기능 | V2가 액션·업무 블록은 더 많음 |

**결론**:  
- **구조**: V2는 파이프라인·매칭·입금/스케줄 등으로 샘플보다 **업무 정보는 많음**.  
- **시각적 풍부함**: 샘플 대비 **차트 종류 단일**, **지표 그리드(variant=grid/option-c) 미구현**, **상단 헤더 단순**이라 “한눈에 보이는 대시보드” 느낌이 약함.

## 2. 시각화 강화 제안 (레이아웃·차트·위젯·배치)

### 2.1 상단 블록 (한눈에 보기)

- **헤더 보강(선택)**  
  - 샘플처럼 **통합 검색**(placeholder) + **캘린더/알림/테마** 아이콘을 ContentHeader 우측에 두면 “어드민 대시보드” 톤이 샘플에 가까워짐.  
  - 토큰: `var(--mg-color-border-main)`, `var(--mg-color-text-secondary)`, 기존 `mg-v2-ad-b0kla__*` 헤더 패턴 재사용.

- **KPI Row 유지·강화**  
  - 현재 4카드 유지.  
  - 각 카드 **좌측 세로 악센트**(4px, 주조/보조/포인트) + **트렌드/전월대비 배지**를 B0KlA 스펙에 맞게 명시해 일관성 확보.  
  - 클래스: `mg-v2-content-kpi-row`, `mg-v2-content-kpi-card`, `mg-v2-content-kpi-card__icon--green|orange|blue|gray`.

### 2.2 중단 블록 (지표·추이·파이프라인)

- **variant="grid" 구현 제안**  
  - **위치**: 파이프라인 **위** 또는 **왼쪽**에 배치하는 옵션 권장.  
  - **내용**: 5단계(매칭→입금확인→회기권한→스케줄등록→회계처리)를 **타일 그리드**로 표시.  
    - 카드당: 단계명, 건수(숫자 강조), 전일 대비 증감(선택), 좌측 악센트 색(variant).  
  - **레이아웃**: `repeat(auto-fit, minmax(160px, 1fr))` 또는 5열 고정, B0KlA 카드(배경 `var(--mg-color-surface-main)`, 테두리 `var(--mg-color-border-main)`, radius 16px).  
  - **효과**: 파이프라인과 동일 데이터, **다른 뷰** 제공 — “숫자로 한눈에” 보는 관리자 니즈 충족.

- **variant="option-c" 구현 제안**  
  - **상단**: 위 grid와 동일한 KPI 타일 그리드(5단계).  
  - **하단**: 기존 pipeline 카드형 5단계 흐름.  
  - **블록 순서**:  
    1) KPI Row  
    2) **지표 그리드(option-c 상단)**  
    3) **파이프라인(option-c 하단)**  
    4) 차트 + 상담사 Row  
  - 사용성: “숫자 먼저 → 흐름 확인” 순서로 정보 계층 명확.

- **차트 다양화 제안**  
  - **현재**: 상담 현황 추이 — 막대 1종, 월간/주간 토글.  
  - **추가 검토**  
    - **라인 차트**: 월별 예약 vs 완료 2선 (추이 비교).  
    - **도넛/파이**: 단계별 건수 비율(매칭·입금대기·스케줄대기 등).  
    - **미니 스파크라인**: KPI 카드 내 “최근 7일 추이” (선택).  
  - **배치**:  
    - 1열: 기존 “상담 현황 추이”(막대 또는 라인).  
    - 2열: “단계별 현황”(도넛/파이) 또는 “예약 vs 완료”(라인) 중 하나를 **같은 Row**에 배치해 2열(2fr : 1fr) 유지.  
  - 색상: `var(--mg-color-primary-main)`, `var(--mg-color-accent-main)`, `var(--mg-color-secondary-main)` 등 B0KlA 팔레트만 사용.

- **파이프라인 위치**  
  - **옵션 A**: 현재처럼 KPI Row 직후(상단에 가깝게) 유지.  
  - **옵션 B**: option-c 적용 시 그리드 아래로 내려 “지표 타일 → 파이프라인 → 차트+상담사” 순서.

### 2.3 하단 블록 (업무·액션)

- **환불 현황**(노출 시): 섹션 제목 + 좌측 악센트 + StatCard 4개 그리드 — 현재 구조 유지, B0KlA 섹션 블록 스타일만 통일.

- **ManualMatchingQueue / 입금·스케줄 대기**:  
  - 리스트형이므로 “시각화”보다는 **테이블·카드 정리**가 중요.  
  - 섹션 제목에 **세로 악센트 바**(4px, 주조) + “N건 대기” 배지로 한눈에 개수 표시 권장.

- **관리 기능 그리드**:  
  - 샘플의 Admin Grid와 동일한 역할.  
  - 카드: 아이콘(배경색 B0KlA 팔레트) + 라벨 + 짧은 설명, 클릭 시 해당 메뉴로 이동.  
  - 클래스: `mg-v2-ad-b0kla__admin-grid`, `mg-v2-ad-b0kla__admin-card`.

## 3. 사용성·정보 노출·레이아웃 관점 정리

### 3.1 사용성 (관리자가 한눈에 볼 정보)

- **1초**: KPI Row 4가지 수치 + (option-c 시) 5단계 타일 숫자.  
- **5초**: 파이프라인으로 “어디에 병목인지” + 차트로 “추이 상승/하락”.  
- **10초**: 우수 상담사, (노출 시) 환불 요약, 매칭/입금/스케줄 대기 건수.  
- **액션**: 관리 기능 그리드 + 매칭 큐·입금 대기 등에서 바로 이동.

제안 반영 시 **숫자(그리드) → 흐름(파이프라인) → 추이(차트) → 업무(리스트/기능)** 순서가 명확해짐.

### 3.2 정보 노출 범위

- **유지**: 총 사용자, 예약된 상담, 완료율, 5단계 파이프라인, 상담 현황 추이, 우수 상담사, (조건부) 환불, 매칭/입금/스케줄 대기, 관리 기능.  
- **추가 권장**:  
  - 5단계를 **타일 그리드**로 보는 뷰(variant=grid / option-c).  
  - **차트 1종 추가**: 단계 비율(도넛/파이) 또는 예약 vs 완료(라인).  
- **선택**: KPI 카드 내 미니 스파크라인, 헤더 검색·아이콘.

### 3.3 레이아웃(상단/중단/하단 블록 순서) 제안

**기본 순서(현재와 동일 + grid/option-c 선택 가능)**

1. **상단**  
   - ContentHeader (옵션: 검색 + 아이콘).  
   - ContentKpiRow(4카드).

2. **중단**  
   - **A. pipeline만 사용 시(현재)**  
     - ContentCard → AdminMetricsVisualization variant="pipeline".  
   - **B. option-c 사용 시**  
     - AdminMetricsVisualization variant="option-c" → 상단 그리드 + 하단 파이프라인.  
   - 그 다음: **차트 + 상담사 Row** (2fr : 1fr).  
     - 왼쪽: 상담 현황 추이(막대/라인) + (여유 시) 오른쪽에 단계 비율(도넛) 또는 2열 하단에 작은 차트 1개.

3. **하단**  
   - (조건부) 환불 현황 ContentSection.  
   - ManualMatchingQueue.  
   - 입금 대기 + 스케줄 대기 Row.  
   - ContentSection “관리 기능” + admin grid.

  - placeholder 제거 후 실제 UI 구현.

- [ ] **차트 1종 추가**  
  - 단계별 비율(도넛/파이) 또는 예약 vs 완료(라인) 중 하나.  
  - 배치: 차트+상담사 Row 내 왼쪽 하단 또는 오른쪽 카드 하단.

- [ ] **헤더 보강(선택)**  
  - 통합 검색 placeholder + 캘린더/알림/테마 아이콘.
