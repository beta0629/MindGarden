# 매칭-스케줄 통합 원스톱 화면 (Matching & Schedule Integration) UI/UX 스펙

## 1. 개요 및 배경
가장 빈번하게 발생하는 '내담자-상담사 매칭' 작업과 '스케줄(일정 등록)' 작업이 별도 화면으로 분리되어 있어, 페이지 이동에 따른 업무 효율 저하가 발생했습니다. 이를 융합하여 전체 대기 현황과 캘린더를 한눈에 보며 즉각적으로 스케줄링할 수 있는 직관적이고 효율적인 원스톱 화면을 기획합니다.

---

## 2. 창의적 UI 레이아웃 아이디어 제안

### 아이디어 A: 스플릿 뷰 (Split View) + 드래그 앤 드롭
- **구조**: 메인 화면을 좌우 두 개의 패널로 분할합니다. 좌측은 **'매칭/스케줄 대기자 목록'**, 우측은 **'전체 상담사 통합 캘린더'**로 구성됩니다.
- **상호작용**: 좌측 대기자 카드를 우측 캘린더의 빈 시간대(Slot)로 마우스 드래그 앤 드롭(Drag & Drop)하여 즉시 스케줄을 생성합니다.
- **장점**: '대상(내담자)'과 '시간/공간(캘린더)'을 한 화면에서 직관적으로 연결하여 업무 속도가 비약적으로 상승합니다.

### 아이디어 B: 칸반 보드 스타일 (Kanban Board)
- **구조**: 상태별(예: 신규 접수, 매칭 중, 스케줄 확정)로 열(Column)을 나누어 카드를 배치합니다.
- **상호작용**: 카드를 다음 단계 열로 드래그하면, '스케줄 확정' 열로 이동할 때 미니 캘린더 팝업이 떠서 일정을 선택하게 합니다.
- **장점**: 전체적인 진행 상태 및 파이프라인 관리에 매우 탁월합니다.
- **단점**: 캘린더 자체의 가시성이 상대적으로 낮아, 상담사별 빈 시간대를 한눈에 파악하기 어렵습니다.

### 아이디어 C: 슬라이딩 드로어 캘린더 (Sliding Drawer Calendar)
- **구조**: 평소에는 대기자 목록이 화면 전체를 차지하고, 카드를 클릭하면 우측에서 캘린더 드로어(Drawer)가 슬라이딩되어 나옵니다.
- **상호작용**: 카드 선택 후 나타난 드로어 내 캘린더에서 빈 시간을 클릭하여 할당합니다.
- **장점**: 좁은 모니터 환경에서도 화면을 넓고 쾌적하게 사용할 수 있습니다.

---

## 3. 최적의 아이디어 선정: 아이디어 A (스플릿 뷰 + 드래그 앤 드롭)

**선정 사유**: 상담 센터 관리 업무의 핵심은 **"어떤 내담자를 어느 상담사의 빈 시간에 넣을 것인가"**를 빠르게 결정하는 것입니다. 전체 캘린더 현황(우측)을 상시 확인하면서 대기자(좌측)를 바로 꽂아 넣을 수 있는 **스플릿 뷰 방식**이 업무 효율(One-stop) 극대화에 가장 적합합니다.

---

## 4. 세부 UI/UX 스펙 (마인드가든 디자인 시스템 적용)

본 스펙은 `mindgarden-design-system.css` 및 `unified-design-tokens.css`의 토큰과 어드민 대시보드 샘플의 비주얼 언어를 완벽히 준수합니다.

### 4.1. 전체 레이아웃 (Layout)
- **상단 바 (Top Bar)**: 
  - 배경: `var(--mg-color-background-main)` (#FAF9F7)
  - 하단 구분선: `1px solid var(--mg-color-border-main)` (#D4CFC8)
  - 내용: 브레드크럼 (`매칭 및 스케줄 관리`), 타이틀 (`통합 스케줄링 센터`), 우측에 `+ 신규 내담자 등록` 주조색 버튼 배치.
- **본문 영역 (Main Content)**:
  - 패딩: `var(--mg-spacing-24)`
  - 구조: `display: flex; gap: 24px;` (좌측 리스트 320px 고정, 우측 캘린더 flex: 1)

### 4.2. 좌측 섹션: 매칭 대기자 목록 (List View)
- **섹션 블록 컨테이너**:
  - 배경: `var(--mg-color-surface-main)` (#F5F3EF)
  - 테두리: `1px solid var(--mg-color-border-main)`, `border-radius: 16px`
  - 패딩: `var(--mg-spacing-24)`
- **섹션 제목**:
  - 세로 악센트 바: 폭 4px, `background-color: var(--mg-color-primary-main)` (#3D5246), `border-radius: 2px`
  - 텍스트: "스케줄 대기 목록", `font-size: 16px`, `font-weight: 600`, `color: var(--mg-color-text-main)` (#2C2C2C)
- **대기자 카드 (Draggable Item)**:
  - 상태: hover 시 커서 모양 `grab`, 드래그 중 `grabbing` 및 `box-shadow` 추가.
  - 배경: `#FFFFFF`, 테두리 `1px solid var(--mg-color-border-main)`, `border-radius: 12px`, 패딩 `16px`
  - 내용: 내담자명(14px, bold), 희망 상담 분야, 희망 시간대 뱃지(12px, background: #E8EBE9, color: #3D5246).
  - 좌측 악센트 바: 상담 종류(개인/부부/가족)에 따라 다른 색상의 포인트 악센트(예: 주조/보조/포인트 색상) 적용.

### 4.3. 우측 섹션: 통합 캘린더 (Calendar View)
- **섹션 블록 컨테이너**:
  - 배경: `var(--mg-color-background-main)` (#FAF9F7) (캘린더의 시인성을 위해 표면보다 약간 밝게 처리)
  - 테두리: `1px solid var(--mg-color-border-main)`, `border-radius: 16px`
  - 패딩: `var(--mg-spacing-24)`
- **캘린더 헤더 (툴바)**:
  - 상담사 필터 (다중 선택 드롭다운), 날짜 이동 버튼, 일/주/월 뷰 전환 탭 (활성 탭: 주조색 #3D5246 텍스트 및 하단 border 적용).
- **캘린더 그리드 (Time Slots)**:
  - 배경: `#FFFFFF`, 눈금선: `1px solid #EBE9E4`
  - **Drop Zone 표시**: 좌측 카드를 드래그하여 그리드 위로 올렸을 때, 드롭 가능한 빈 슬롯은 배경이 `rgba(61, 82, 70, 0.1)` (주조색 10%)로 하이라이트 되며 점선 테두리가 활성화됩니다.
- **확정된 스케줄 블록**:
  - 배경: `var(--mg-color-primary-light)` (#4A6354) 또는 보조색 (#6B7F72)
  - 텍스트: `#FAF9F7` (흰색 계열)
  - `border-radius: 6px`, 내부 패딩 `4px 8px`, 타이포그래피 12px.

### 4.4. 매칭 및 스케줄 등록 플로우 (Interaction Flow)
1. **드래그 (Drag)**: 관리자가 좌측 대기자 목록에서 카드를 클릭 & 드래그.
2. **드롭 (Drop)**: 캘린더의 원하는 날짜/시간(상담사 열)에 드롭.
3. **모달 확인 (Confirm)**: 드롭 즉시 팝업 모달(UnifiedModal 사용)이 호출되어 시간, 상담사, 내담자 정보가 사전 채워진 상태(Pre-filled)로 표시됨. (기존 스케줄 모달 재활용)
4. **저장**: 모달에서 [저장] 클릭 시 스케줄 블록이 캘린더에 생성되고, 좌측 대기자 목록에서 해당 카드가 사라짐.

## 5. 서브에이전트 회의 안건 (기술 검토용)

이 화면을 실제 구현할 **core-coder**를 위한 기술적 검토 및 우려 사항(질문 형태)입니다.

1. **드래그 앤 드롭 라이브러리 선정 및 호환성**
   - *"React 환경에서 좌측 리스트(DOM)와 우측 캘린더 라이브러리(예: FullCalendar 등) 간의 외부 Drag & Drop 연동이 자연스럽게 구현 가능한가요? 어떤 라이브러리(예: dnd-kit, react-beautiful-dnd)를 사용할 계획인지 검토가 필요합니다."*
2. **성능 및 렌더링 최적화**
   - *"대기자 목록이 길어지고 캘린더에 렌더링될 스케줄 블록(이벤트)이 많아질 경우, 드래그 앤 드롭 시 리렌더링 지연(Lag)이 발생하지 않도록 상태 관리를 어떻게 최적화할 계획인가요?"*
3. **캘린더 뷰의 가로/세로 스크롤 동기화**
   - *"상담사가 많아 가로 스크롤이 생길 경우, 캘린더의 시간축(세로축)과 상단 상담사 헤더(가로축) 고정이 필요합니다. 기존 UI 구조 안에서 스크롤 처리가 깔끔하게 떨어지도록 구현할 수 있을까요?"*
4. **상태 관리 구조 (Zustand 등)**
   - *"대기자 목록(비할당) 배열과 캘린더 이벤트(할당) 배열 간의 데이터 이동 시, 낙관적 업데이트(Optimistic Update)를 적용하여 사용성을 높일 것인지, 아니면 API 응답 후 UI를 갱신할 것인지 정책적인 확인이 필요합니다."*
5. **모바일/태블릿 반응형 처리 정책**
   - *"스플릿 뷰 특성상 화면 폭이 좁은 태블릿(iPad 등)에서는 좌우 배치가 불편할 수 있습니다. 1024px 이하 해상도에서는 슬라이딩 드로어(서랍) 방식이나 상하 배치로 우회할지, 아니면 이 화면은 PC(데스크탑) 전용으로 고정할지 기술적/기획적 합의가 필요합니다."*
