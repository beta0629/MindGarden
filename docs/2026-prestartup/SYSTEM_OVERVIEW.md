# 시스템 개요 (2025-12 기준)

## 1) 제품 정의

**CoreSolution**은 상담센터/교육기관/1인 사업자 등 “서비스 업종”이 운영에 필요한 핵심 기능을 **하나의 SaaS**로 제공하는 통합 운영 플랫폼입니다.

- **운영(관리자)**: 온보딩/승인, 사용자·권한, 상담/예약/결제/정산, 고객관리, 대시보드 KPI
- **보안/표준화**: 멀티테넌트 데이터 분리, 공통코드 기반 동적 구성, 표준화된 저장 규칙
- **AI 확장 모듈(신규 축)**: **심리검사 리포트 AI(TCI/MMPI 우선)** — 스캔 PDF 업로드 → 추출/검증 → 리포트 생성 → 지표 저장/통계

## 2) 타겟 고객 및 가치 제안

- **상담센터/심리서비스 기관**
  - 상담 운영(예약/기록/결제) + **심리검사 리포트 생성/관리 자동화**
  - 검증·근거 기반 리포트로 “오판 리스크”를 낮추고, 기록 품질을 표준화
- **학원/교육 서비스**
  - 테넌트(기관) 단위 운영/정산/권한 통제 중심의 운영 효율화
- **확장 가능한 1인 서비스업**
  - 동일한 “예약→결제→정산→고객관리” 흐름을 업종별 템플릿으로 확장

## 3) 핵심 차별점(요약)

1. **온보딩 표준화(신규 고객 유입/전환 엔진)**
2. **Ops(운영 포털: 승인·관제·보안·배포 표준의 집약)**
3. **멀티테넌트 + 표준화 원칙(공통코드/동적 조회/하드코딩 금지) 기반의 운영 일관성**
4. **심리검사 AI 파이프라인(MVP)**
   - 민감 문서(스캔 PDF) **암호화 저장**
   - 추출 데이터 **검증(타당성/범위/누락)**
   - LLM 출력 **오판 방어(근거 매칭/금지문구/구조 검증/폴백)**
   - 내담자별 **지표 표준화 저장 + 통계**

## 4) 현재 구현 범위(상태)

### 운영/플랫폼 공통
- 멀티테넌트(테넌트 분리), 역할 기반 권한, 대시보드 구조
- CI/CD: GitHub Actions 자동 배포(개발: `develop`, 운영: `main`)

### 심리검사 리포트 AI (TCI/MMPI 우선)
- 관리자 화면: `/admin/psych-assessments`
  - PDF 업로드(드래그&드롭), 최근 문서 목록, 리포트 생성 트리거
- 서버 API(요약)
  - 업로드: `POST /api/v1/assessments/psych/documents`
  - 리포트 생성: `POST /api/v1/assessments/psych/documents/{documentId}/report`
  - 통계/최근목록: `GET /api/v1/assessments/psych/stats`, `GET /api/v1/assessments/psych/documents/recent`
- 저장(요약)
  - 원본 PDF: AES-GCM 기반 **암호화 저장**
  - 추출/검증 결과: extraction + validation JSON
  - 표준 지표: 척도 코드/라벨/점수/백분위/해석 태그 등
  - 리포트: markdown + evidence JSON + 버전(프롬프트/모델/룰)
