# Core Solution 상담센터 — 화면 가독성 · 줌 · 모션 **구현** 가이드 (Mac)

**용도**: 단순 A-roll(화면 녹화)만으로는 UI 글자가 작아 읽히지 않을 때, **편집기에서 어떻게** 확대·인서트·자막 애니·클릭 하이라이트를 넣는지.  
**이것은 코드·앱 개발이 아님.** CapCut / Final Cut Pro / DaVinci Resolve 중 **하나**만 쓰면 된다.  
**나레이션·자막 문장 SSOT**: [CORE_SOLUTION_VIDEO_NARRATION_CAPTIONS.md](./CORE_SOLUTION_VIDEO_NARRATION_CAPTIONS.md)  
**합성 순서**: [CORE_SOLUTION_GOOGLE_OMNI_EDIT_ASSEMBLY.md](./CORE_SOLUTION_GOOGLE_OMNI_EDIT_ASSEMBLY.md)  
**최종 갱신**: 2026-07-15

---

## 0. 한 줄 결론

| 하고 싶은 것 | 구현 수단 | 쓰지 말 것 |
|--------------|-----------|------------|
| UI 글자 크게 보이게 | 타임라인 **Transform Scale + 키프레임** 또는 **같은 구간 복제 클로즈 인서트** | Omni / AI로 UI 다시 그리기 |
| 클릭이 보이게 | 녹화 시 커서 크게·느리게 + 편집 시 **클릭 프레임에 원형 하이라이트 ~1초** | 이모지·과도한 스티커 |
| 설명 글자 | V3 챕터 카드 · S1 하단 자막 (나레이션 문서 1:1) | AI burn-in 한글 |

**Omni B-roll은 분위기·전환용이다. UI 확대 용도가 아니다.**

## 1. 추천 스택 (Mac · 하나만 고르면 됨)

| 툴 | 추천 상황 | 설치 |
|----|-----------|------|
| **CapCut** (데스크톱) | 가장 쉬움. 숏폼·본편 모두 충분 | Mac App Store / CapCut 사이트 |
| **Final Cut Pro** | 맥 전용, 키프레임·역할(Roles)에 익숙할 때 | App Store |
| **DaVinci Resolve** | 무료로 키프레임·노드까지 다루고 싶을 때 | Blackmagic Design |

프로젝트 기본값(세 툴 공통):

- 시퀀스 **1920×1080 · 30fps**
- Desktop 원본(예: 4096×1728 @120fps)은 **가져와 놓고** 시퀀스에 맞게 스케일 다운 · Dock/메뉴바는 크롭

## 2. 워크플로 순서 (반드시 이 순서로)

```
1) 원본 .mov 트림 (R01–R10 권장 초)
2) 합성 순서대로 V1에 올림  (OP → R01 … R10 → ED)
3) 줌 · 인서트 (이 문서 §3–§4)
4) VO(A1) · 자막(S1) · 챕터(V3)   ← 나레이션 문서 문장만
5) 클릭 하이라이트 (필요한 프레임만)
6) 색보정은 약하게 (대비·노출만)
7) Export: 1080p30 · H.264 · 고비트레이트(본편 8–12 Mbps)
```

Omni(V2)는 **2번과 3번 사이 또는 3번 직후**에 경계만 끼운다. UI를 크게 보이게 하려고 Omni를 쓰지 않는다.

## 3. 확대(줌) — 구현 단계

**목표값(공통)**: Scale **110~130%** · 초점(Position)을 버튼·KPI·패키지명에 맞춤 · 키프레임 **Ease In/Out** · 길이 **1.5~3초**.

### 3.1 CapCut (Mac)

1. 타임라인에서 **클립 선택**.
2. 우측 **비디오** → **트랜스폼(Transform)** → **크기(Scale)** 를 110~130으로.
3. **위치(Position)** X/Y를 드래그해 읽히게 할 UI(버튼·KPI·패키지명)가 화면 중앙~중앙 하단에 오게.
4. Scale(또는 Position) 옆 **마름모(키프레임)** 를 구간 **시작**에 찍고 → playhead를 1.5~3초 뒤로 → Scale을 조금 더 올리거나 Position을 미세 이동 → 키프레임 자동 생성.
5. 키프레임 사이를 우클릭하거나 그래프에서 **Ease in / Ease out**(또는 스무스) 적용.

### 3.2 Final Cut Pro

1. 클립 선택 → 인스펙터 **Transform** → **Scale** 110~130.
2. **Position**으로 초점 이동.
3. 인스펙터 왼쪽 **키프레임 버튼**(또는 `Ctrl+K` 계열 키프레임 추가)으로 Scale/Position에 키프레임.
4. Timeline Index / Video Animation 펼침 → 키프레임 우클릭 **Smooth** / Ease.

### 3.3 DaVinci Resolve (Edit 페이지)

1. 클립 선택 → Inspector → **Transform** → **Zoom** (또는 Scale) 1.10~1.30.
2. **Position**으로 초점.
3. 다이아몬드 키프레임 → 1.5~3초 뒤 값 변경.
4. 키프레임 우클릭 → **Ease In / Ease Out** (또는 Curve Editor에서 S커브).

### 3.4 R01–R10 확대 초점 (어디에 Scale을 맞출지)

| R# | 초점(Position 맞출 대상) | 권장 Scale | 듀레이션 |
|----|--------------------------|------------|----------|
| R01 | 로그인 후 대시보드 진입 CTA·첫 KPI | 110–120% | 1.5–2초 |
| R02 | 핵심 메뉴/매칭 영역 클릭 지점 | 115–125% | 1.5–2초 |
| R03 | **KPI 숫자 카드** | 120–130% | 2–3초 |
| R04 | **스케줄 카드 패키지명** · **예약/확정 버튼** | 120–130% | 2–3초 |
| R05 | 월간 캘린더 일정 밀도 한 블록 | 115–125% | 1.5–2초 |
| R06 | 사용자 카드 요약(이름 대신 역할·상태 라벨) | 115–120% | 1.5–2초 |
| R07 | 수입·연동 요약 숫자 | 115–125% | 2초 |
| R08 | 거래 테이블 상태 열(금액 과도 클로즈업 금지) | 110–120% | 1.5–2초 |
| R09 | **발송/확인·템플릿** UI | 120–130% | 2–3초 |
| R10 | **저장/완료 버튼**(일지 본문 글자는 줌 금지) | 115–125% | 1.5–2초 |

**본편(3:40–4:50)**: 구간당 줌 **1~2회**, Scale 상한 120% 위주로 여유 있게.  
**숏폼(60초)**: 같은 기법을 **더 공격적**(Scale 125–130%, 구간당 줌 빈도↑).

## 4. 인서트(cut-in) — 구현 단계

와이드만으로는 여전히 작을 때: **같은 클립을 복제**해 짧게 끼운다.

```
[와이드 2초] → [클로즈 인서트 2초] → [와이드]
```

1. 타임라인에서 해당 R 구간 선택 → **복제**(CapCut: 복사/붙여넣기 · FCP: `Cmd+D` 또는 Option-드래그 · Resolve: Alt-드래그).
2. 복제본만 Scale 120–130% + Position으로 초점.
3. 복제본 길이를 **약 2초**로 트림.
4. 원본(와이드) 중간에 razor로 자르고 복제본을 끼움. 앞뒤 와이드는 각 **약 2초** 유지.
5. 컷 경계는 하드컷 또는 Cross Dissolve **4–8프레임**만 (긴 transition 금지).

**cut-on-action**: 클릭·호버가 일어나는 프레임에서 와이드→클로즈로 자르면 자연스럽다.

## 5. 자막 · 챕터 애니메이션 — 구현 단계

**문장·SRT·챕터 카드는 반드시** [CORE_SOLUTION_VIDEO_NARRATION_CAPTIONS.md](./CORE_SOLUTION_VIDEO_NARRATION_CAPTIONS.md) **와 1:1**. 새로 지어 쓰지 않는다.

| 레이어 | 내용 | 애니 |
|--------|------|------|
| S1 하단 자막 | 문서 §2 / §3 SRT | **fade 또는 slide-up** **0.2–0.4초** 등장, OUT도 동일 |
| V3 챕터 카드 | 문서 §2.2 큐 (구간 IN부터 **약 1.0초**) | **fade 0.3초** |

### CapCut
- 자막: 자동/임포트 후 텍스트 → 애니메이션 → **페이드** 또는 **아래에서 위로**, Duration 0.2–0.4s.
- 챕터: 별도 텍스트 클립, 불투명도 키프레임으로 fade 0.3s.

### Final Cut Pro
- SRT → Captions/Titles. Title 인스펙터 **Build In/Out** = Fade, 0.2–0.4s.
- 챕터는 Custom Title 또는 Lower Third, 구간 IN에서 1초만.

### DaVinci Resolve
- Subtitles 트랙에 SRT 임포트. Fusion/타이틀 템플릿에서 Opacity fade, 또는 Inspector **Soften** / Animation.
- 챕터 Text+ 클립, Opacity 키프레임 0→100 (0.3s).

VO(A1)와 자막 IN–OUT이 어긋나면 **컷 길이를 바꾸지 말고** 자막·VO만 문장 경계에 맞춘다.

## 6. 클릭 가독성 — 구현 단계

### 6.1 녹화 단계 (재촬영 시)

Mac: `Cmd+Shift+5` → **전체 화면** 또는 필요 시 **선택한 영역(창만)**.

| 체크 | 방법 |
|------|------|
| 커서 보이기 | 옵션에서 커서 표시 · 가능하면 “클릭 시 강조” |
| 느리게 | 마우스를 **의도적으로 느리게**, 호버 0.5–1초 |
| 동일 시나리오 | R01–R10 시나리오 그대로 · PII 클로즈업 금지 |

### 6.2 편집 단계

클릭이 일어난 **1프레임 근처**에만:

1. 원형(타원) 도형 또는 “클릭 효과” 오버레이 추가.
2. 클릭 지점에 배치, **약 1초** 후 페이드아웃.
3. 색은 차분한 브랜드 톤(소프트 accent). **보라 네온·이모지·반짝이 스티커 금지**.

부드러운 팬/줌(Ken Burns 느낌)은 §3 Scale/Position 키프레임만으로 충분하다. 추가 파티클은 넣지 않는다.

## 8. 재촬영 권장 샷 (가독성용 close-up)

기존 와이드 mov가 있으면 **편집 줌 먼저**. 그래도 픽셀이 깨지면 같은 시나리오로 **창만** 재녹화:

| 우선 | 샷 |
|------|----|
| 1 | R04 — 패키지명 + 예약/확정 버튼 (마우스 느리게) |
| 2 | R03 — KPI 숫자 카드 |
| 3 | R09 — 문자 발송/확인 UI |
| 4 | R10 — 저장 버튼 (본문 텍스트는 프레임에 넣지 않음) |
| 5 | R01 — 로그인 → 운영 화면 전환 CTA |

이 close-up은 **편집기 인서트(§4)** 로 끼운다. Omni로 “확대해 보여 달라”고 생성하지 않는다.

## 10. Export 체크

- [ ] 시퀀스 1920×1080 · 30fps  
- [ ] 줌/인서트 때문에 자막이 잘리지 않음 (safe area)  
- [ ] Omni에 제품 UI·한글 없음  
- [ ] 나레이션 문서 문장만 사용  
- [ ] PII 클로즈업 없음
