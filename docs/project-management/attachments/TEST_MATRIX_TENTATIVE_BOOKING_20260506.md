## 4) 퇴장 조건 (`core-coder` PR 제출 전 Green 조합)

본 매트릭스 범위에서 PR 전 **최소**로 볼 조합은 다음이다. 저장소 루트에서 **백엔드** `mvn -q test`(또는 팀 표준 모듈 단위 `mvn test` 동등)로 가예약·스케줄·권한·레이트리밋 등 **해당 배치에 포함된 Java 단위·통합**이 Green일 것. **프론트**는 변경이 있으면 `frontend` 기준 Jest로 변경 모듈 및 관련 `__tests__`가 Green일 것. **E2E**는 로컬에서 **백엔드 8080 + 프론트 3000** 기동 후 [`tests/e2e/README.md`](../../../tests/e2e/README.md) 경로 규칙에 따라 `cd tests/e2e && npx playwright test … --project=chromium`으로 **통합 스케줄·가드·모달 관련 스모크 스펙**을 1회 이상 Green으로 둔다(시크릿·8080 미기동 시 스킵되는 스펙은 “Green”이 아닌 **스킵**으로 구분해 PR 설명에 적는다). 보안 회귀는 해당 PR이 공개면·콜백을 건드리면 통합 단위에 포함하고, 재현 어려운 구간은 아래 SEC-01 한 줄을 따른다.
