# SessionIdleWarningModal — 탭 백그라운드·타이머 보강 머지 전후 회귀 체크리스트

**목적**: `SessionIdleWarningModal`의 백그라운드 탭·타이머 보강(코더 병렬 작업) **머지 전·후** 수동 검증에 사용한다.

**대상 컴포넌트**: `frontend/src/components/common/SessionIdleWarningModal.js`

---

## P0 — 수동 (탭 백그라운드)

- [ ] 앱을 **백그라운드 탭**에 둔 채 **25~35분** 대기한다. (운영 환경의 idle·경고 타이밍과 다르면 **운영 idle에 맞춘 시간**으로 조정한다.)
- [ ] 세션 만료 **약 1분 전**에 **모달이 표시**되는지 확인한다.
- [ ] **연장** 클릭 시 `checkSession`이 **성공**하고 모달이 **닫히는지** 확인한다.

## P0 — 수동 (탭 포그라운드)

- [ ] 위와 동일 시나리오를 **포그라운드(활성 탭)**에서 반복한다. (만료 1분 전 모달, 연장 시 `checkSession` 성공·모달 닫힘.)

## P0 — 수동 (탭 전환·복귀)

- [ ] 모달이 **열린 상태**에서 다른 탭으로 전환했다가 **복귀**한다.
- [ ] **카운트다운**이 기존 스펙과 일치하는지 확인한다.
- [ ] **자동 닫기**(연장으로 surplus가 초과되는 경우 등) 동작이 기존 스펙과 **동일하게 유지**되는지 확인한다.

## P1 — `current-user` 401

- [ ] `current-user`가 **401**을 반환한 **직후**, 로그인 화면으로의 이동은 **“세션 만료 후”**에만 일어나야 한다. (세션 만료 임박 **모달이 선행**되는 흐름이 기대된다.)
