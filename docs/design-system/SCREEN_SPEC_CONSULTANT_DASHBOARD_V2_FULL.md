## 1. 항목별 개선 스펙 (B0KlA 정합)

### 1.1 전체 레이아웃 (AdminCommonLayout & Content Area)
- **레이아웃 래퍼**: 전체 페이지 레이아웃은 `AdminCommonLayout`을 사용하여 어드민과 동일한 LNB/GNB 구조를 공유.
- **ContentHeader 도입**: 기존 하드코딩된 웰컴 메시지 영역을 `ContentHeader` 공통 모듈로 마이그레이션.
  - `title`: "환영합니다, {이름} 상담사님"
  - 상단 패딩 24px, 하단 16px 마진 적용.

### 1.2 Phase 1 신규 컨텐츠 정합 (Flat UI)
- **빠른 액션 바 (Quick Action Bar)**
  - G2-08 수준의 플랫한 UI 디자인 적용.
  - `ContentSection` 스타일(`mg-v2-ad-b0kla__section`) 래퍼 사용.
  - 좌측: 섹션 타이틀 (아이콘 + 텍스트), 우측: `mg-v2-btn` 그룹.
  - CTA 버튼: 주조 액션(상담일지 작성)은 `mg-v2-btn-primary`, 보조 액션은 `mg-v2-btn-outline`으로 단일화.
- **미작성 상담일지 알림 (Alert)**
  - Warning 테마 (`var(--mg-color-warning-light)` 배경, `var(--mg-color-warning-main)` 좌측 악센트 바).
  - 단일 CTA: "바로 작성하기" (`mg-v2-btn-primary`).
- **다음 상담 준비 카드**
  - Surface 테마 (`var(--mg-color-surface-main)` 배경, `var(--mg-color-primary-main)` 좌측 악센트 바).
  - 본문 정보는 Grid(1fr 1fr 1fr) 플랫 타이포그래피. 다중 버튼 금지(Primary 상세보기 1개 + Secondary 1개).
- **긴급 확인 필요 내담자 (Urgent Clients)**
  - Error 테마 (`var(--mg-color-error-main)` 좌측 악센트 바).
  - 내부 아이템 카드는 그림자를 제거하고 플랫한 Border(`var(--mg-color-border-main)`)만 적용. 

### 1.3 기존 위젯 및 KPI 개선 (ContentKpiRow & ListTableView)
- **주요 통계 (ContentKpiRow)**
  - 기존 4개의 통계 카드를 공통 모듈 `ContentKpiRow`로 통일하여 1280px에서 가로 스크롤 없이 4-grid가 1줄에 노출되도록 패딩/마진 최적화.
- **목록형 위젯 (최근 일정, 다가오는 상담, 최근 알림)**
  - 기존 `ProfileCard` 오용을 전면 금지하고 `ListTableView` (Compact 밀도) 모듈로 롤백.
  - 각 행(Row)별 복잡한 다중 CTA 버튼을 제거하고 플랫 텍스트 또는 1개의 Overflow 메뉴로 교체.
  - 섹션 상/하단에 '전체 보기' 링크 1개만 배치 (CTA ≤ 1 원칙).

## 3. 토큰 및 스타일 스펙 (Must Not)

### 3.1 사용 토큰 (B0KlA 전역 표준)
- **배경 (Background)**: `var(--mg-color-background-main)` (#FAF9F7)
- **서페이스 (Surface/Card)**: `var(--mg-color-surface-main)` (#F5F3EF)
- **테두리 (Border)**: `var(--mg-color-border-main)` (#D4CFC8)
- **주조 텍스트 (Primary Text)**: `var(--mg-color-text-main)` (#2C2C2C)
- **버튼 공통**: `mg-v2-btn` 베이스 클래스 적용 필수 (`mg-v2-btn-primary`, `mg-v2-btn-outline` 등)
- **섹션 블록 래퍼**: `mg-v2-ad-b0kla__section` 또는 `ContentSection` 활용.

### 3.2 Must Not (금지 사항)
1. **ProfileCard 오용 금지**: 대시보드 하단의 '최근 일정', '다가오는 상담' 등 목록형 UI에 `ProfileCard` 재사용을 절대 금지. `ListTableView` (Compact)로 교체.
2. **다중 CTA 금지**: 리스트나 위젯 내부 행(row) 단위에 다중 액션 버튼 배치 금지. 핵심 액션 1개 + 전체보기 링크로 최소화 (CTA ≤ 1 원칙).
3. **그림자(Shadow) 남용 금지**: 플랫 UI(G2-08) 기조에 맞게 카드/섹션에 드롭 섀도우를 빼고, 1px 테두리(`border-main`)와 배경색 분리만으로 Depth를 표현할 것.
4. **하드코딩 Hex 금지**: 패딩, 마진, 색상, 라디우스 등 모든 값은 `unified-design-tokens.css`의 CSS 변수(`var(--mg-*)`)를 사용할 것.
5. **코드 작성 금지**: 본 스펙은 디자인 및 레이아웃 정의 문서이며, 실제 소스 코드 수정은 `core-coder`에게 위임함.

## 4. UI/UX Quality Gate (최고 품질 기준)

해당 문서를 바탕으로 구현 시 다음의 **최고 품질 기준**을 통과해야 합니다.

1. **B0KlA 토큰 100%**: 하드코딩 hex/rgba 0건, `var(--mg-*)` 100% 사용.
2. **타이포·8px Grid**: `ContentHeader` G-14 h1 SSOT 준수 및 8px 기반 간격 시스템 유지.
3. **다크모드 Cascade**: 다크모드(`dark-c3b`) 토글 시 컴포넌트 스타일 완전 호환.
4. **반응형 보장**: 모바일 414px ~ 데스크톱 1280px 레이아웃 회귀(깨짐/잘림) 0.
5. **접근성 (A11y)**: Focus ring 시각화, 아이콘 `aria-label` 적용, 명도 대비 WCAG AA 기준.
6. **Must-not**: 
   - 레거시 CSS 및 인라인 스타일 잔존 금지.
   - 이중 제목 (Header Title + 컨텐츠 Title 중복) 금지.
   - `MGButton` / `ActionBar` SSOT 위반 금지 (커스텀 스타일링 금지).
7. **시각 정합**: `admin-dashboard-sample` 톤앤매너 완벽 동기화.
