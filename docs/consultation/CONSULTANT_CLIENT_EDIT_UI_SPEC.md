# 상담사 수정·내담자 수정 화면 UI/UX 스펙

> MindGarden — 수정 모달에 주민번호·주소·상담사 자격/경력 추가  
> @author core-designer | @since 2026-03-02  
> **참조**: [CONSULTANT_CLIENT_REGISTRATION_UI_SPEC.md](./CONSULTANT_CLIENT_REGISTRATION_UI_SPEC.md) §3 (등록 폼 스펙)

---

## 1. 개요 및 적용 범위

- **목적**: 상담사 수정 모달·내담자 수정 모달에 등록 폼과 동일한 데이터(주민번호, 주소, 상담사 한정 자격증·경력사항)를 추가하되, **수정 시 동작**(기존 값 노출 여부, 변경 시에만 저장 등)을 명확히 정의한다.
- **대상 화면**
  1. **상담사 수정**: `ConsultantComprehensiveManagement`에서 `modalType === 'edit'`일 때 열리는 모달 폼. 기존 필드: name, email, phone, status, specialty, profileImageUrl.
  2. **내담자 수정**: `ClientComprehensiveManagement`의 `ClientModal`에서 `type === 'edit'`일 때. 기존 필드: name, email, phone, status, grade, notes, profileImageUrl.
- **디자인 기준**: 등록 스펙과 동일. AdminCommonLayout, B0KlA(mindgarden-design-system), `unified-design-tokens.css`, `docs/design-system/PENCIL_DESIGN_GUIDE.md`. 코드 작성 없이 스펙만 산출.

## 3. 상담사 수정 모달 (modalType === 'edit')

### 3.1 폼 필드 배치 순서 (위에서 아래)

**공통 순서(상담사·내담자 동일)**: ① 주민번호 ② 이름 ③ 전화번호 ④ 주소 → 이후 역할별 나머지 필드.

| 순서 | 필드 | 비고 |
|------|------|------|
| 1 | 프로필 이미지 (ProfileImageInput) | 기존 |
| 2 | **주민번호 앞 6자리 / 뒤 1자리** | §5.1 참조. 수정 시 기존 값 미노출. |
| 3 | 이름 * | 기존 |
| 4 | 전화번호 | 기존 |
| 5 | **주소 블록** (주소 검색 + 상세 주소 + 우편번호) | §5.2 참조. 기존 값 있으면 표시. |
| 6 | 이메일 * (disabled 수정 시, 도움말 "이메일은 변경할 수 없습니다.") | 기존 |
| 7 | 전문분야 (specialty 태그) | 상담사만. 기존 |
| 8 | **섹션 제목 "자격·경력"** | 좌측 세로 악센트 바 4px, 주조색. 상담사만. |
| 9 | **자격증** (qualifications/certification) | §5.3 참조. 기존 값 표시. |
| 10 | **경력사항** (workHistory) | §5.3 참조. 기존 값 표시. |

- **데스크톱**: 2단 그리드 적용 가능한 블록(주민번호 한 줄 두 칸, 주소 내 우편/상세 등)은 등록 스펙과 동일하게 2단, 나머지는 한 줄 전체 너비.
- **모바일**: 1단 배치.

### 3.2 상담사 수정 모달 제목

- 기존 유지: "상담사 정보 수정".
- 모달 푸터: "취소" | "수정" 버튼. 스타일 `mg-v2-button`, `mg-v2-button-secondary` / `mg-v2-button-primary`.

## 5. 수정 화면 공통 — 추가 필드 상세 스펙

라벨·placeholder·검증·에러 메시지는 **등록 스펙 CONSULTANT_CLIENT_REGISTRATION_UI_SPEC.md §3**과 동일하게 적용한다. 아래는 수정 모달에서만의 **차이점·추가 규칙**을 정리한다.

### 5.1 주민번호 (공통: 상담사·내담자 수정)

- **수정 시 동작**
  - **기존 값 노출 금지**: 저장된 주민번호 앞 6자리·뒤 1자리는 수정 폼에 **표시하지 않는다**. 입력란은 **빈 칸**으로 둔다.
  - **변경 시에만 저장**: 사용자가 앞 6자리·뒤 1자리를 **입력한 경우에만** 해당 값을 API로 전송하여 갱신한다. 빈 칸으로 두면 기존 DB 값을 변경하지 않는다(기존 값 유지).
- **도움말 문구 (필수 표시)**
  - 주민번호 블록 위 또는 왼쪽 칸 라벨 아래에 도움말을 둔다.
  - **문구**: "수정 시 기존 값은 표시하지 않습니다. 변경할 때만 입력해 주세요."
  - 스타일: `mg-v2-form-help`, 12px, `var(--mg-color-text-secondary)`.
- **라벨·placeholder·입력 제한·검증**
  - 등록 스펙 §3.1과 동일.
  - 라벨: "주민번호 앞 6자리", "주민번호 뒤 1자리". 필수/선택 정책에 따라 `*` 또는 "(선택)".
  - placeholder: 앞 6자리 `900101`, 뒤 1자리 `1`.
  - 숫자만 입력, 앞 6자리 maxLength 6, 뒤 1자리 maxLength 1.
  - 검증 오류 메시지: 형식 오류 "주민번호 앞 6자리는 6자리 숫자, 뒤 1자리는 1자리 숫자로 입력해 주세요." / 체크섬 오류 "주민번호가 올바르지 않습니다. 다시 확인해 주세요."
- **배치**: 한 줄 두 칸(앞 6자리 | 뒤 1자리). 등록 폼과 동일한 시각적 묶음.

### 5.2 주소 (공통: 상담사·내담자 수정)

- **수정 시 동작**
  - **기존 값 표시**: API에서 내려준 주소·상세 주소·우편번호가 있으면 해당 값을 **폼에 채워서 표시**한다. 사용자가 수정 가능.
  - **주소 변경**: "주소 검색" 버튼 클릭 → 카카오 주소 API(팝업/레이어) 호출 → 선택 시 주소·우편번호 필드가 새 값으로 갱신되고, 상세 주소는 사용자가 필요 시 수정.
- **배치·라벨·placeholder**
  - 등록 스펙 §3.2와 동일.
  - 순서: ① 주소 검색 버튼 + 주소 표시 영역 ② 상세 주소 ③ 우편번호.
  - 라벨: "주소 검색", "상세 주소", "우편번호".
  - placeholder: 주소(검색 전) "주소 검색 버튼을 눌러 주소를 입력하세요." / 상세 주소 "동, 호수, 상세 주소를 입력하세요." / 우편번호 "00000".
- **버튼**: "주소 검색" 또는 "주소 찾기". 아웃라인 스타일, height 40px, radius 10px, 테두리 `var(--mg-color-border-main)`.

### 5.3 상담사만 — 자격증·경력사항 (수정 모달)

- **수정 시 동작**
  - **기존 값 표시**: API에서 내려준 `qualifications`/`certification`, `workHistory` 값이 있으면 해당 입력란에 **그대로 채워서 표시**한다. 사용자가 수정·추가 입력 가능.
- **섹션 제목**
  - "자격·경력". 좌측 세로 악센트 바 4px `var(--mg-color-primary-main)`, 제목 텍스트 16px, fontWeight 600, `var(--mg-color-text-main)`. 등록 스펙 §3.3과 동일.
- **자격증**
  - 라벨: "자격증" (선택 시 "(선택)").
  - placeholder: "보유 자격증을 입력하세요. 예) 정신건강임상심리사 1급, 상담심리사 1급".
  - 기존 값이 있으면 input/textarea에 표시.
- **경력사항**
  - 라벨: "경력사항" (선택).
  - placeholder: "관련 경력 및 근무 이력을 입력하세요.".
  - textarea, 여러 줄. 기존 값이 있으면 표시.
- **배치**: 자격증 한 줄, 그 아래 경력사항 textarea. 전체 너비(한 칸).

## 6. 상호작용·상태 요약

| 항목 | 수정 모달 동작 |
|------|----------------|
| 주민번호 | 기존 값 미노출, 빈 칸. 입력 시에만 해당 값 전송·저장. 도움말 "수정 시 기존 값은 표시하지 않습니다. 변경할 때만 입력해 주세요." 표시. |
| 주소 | 기존 값 로드 시 표시. "주소 검색"으로 카카오 API 호출 후 주소·우편번호 갱신, 상세 주소 직접 수정 가능. |
| 자격증·경력(상담사만) | 기존 값 로드 시 표시. 사용자가 수정 후 저장 시 갱신. |
| 검증 | 주민번호는 값이 입력된 경우에만 형식·체크섬 검증. 등록 스펙 §3과 동일한 오류 메시지·표시 위치(`var(--mg-error-500)`, 12px). |

## 8. 코더 전달 체크리스트

- [ ] **상담사 수정**: 기존 필드(프로필·이름·이메일·전화·전문분야) 아래에 주민번호(빈 칸 + 도움말) → 주소 블록 → "자격·경력" 섹션 제목 → 자격증 → 경력사항 순서로 배치.
- [ ] **내담자 수정**: 기존 필드(프로필·이름·이메일·전화·상태·등급·메모) 아래에 주민번호(빈 칸 + 도움말) → 주소 블록 순서로 배치.
- [ ] **주민번호**: 수정 시 기존 값 미노출, 빈 칸. 도움말 "수정 시 기존 값은 표시하지 않습니다. 변경할 때만 입력해 주세요." 표시. 입력 시에만 저장. 라벨·placeholder·검증 메시지는 등록 스펙 §3.1과 동일.
- [ ] **주소**: 기존 값 있으면 표시. "주소 검색" 버튼(카카오 API), 상세 주소·우편번호 라벨·placeholder는 등록 스펙 §3.2와 동일.
- [ ] **자격증·경력(상담사만)**: 기존 값 로드 시 표시, 섹션 제목 "자격·경력" + 라벨·placeholder는 등록 스펙 §3.3과 동일.
- [ ] 모든 색·간격·radius는 `var(--mg-*)` 토큰 사용. 모달은 UnifiedModal, `mg-v2-ad-b0kla` 유지.
