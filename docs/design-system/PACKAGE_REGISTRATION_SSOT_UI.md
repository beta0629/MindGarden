# 패키지 요금 등록 SSOT UI 명세서

## 1. 개요 및 목적
본 명세서는 마인드가든 어드민의 **패키지 요금 등록 화면(`/admin/package-pricing`)을 단일 진실 공급원(SSOT, Single Source of Truth)**으로 통합하기 위한 UI/UX 설계 및 레이아웃 명세입니다. 
기존 매칭 생성 모달에서 파편화되어 있던 즉석 상품 조합 기능을 제거하고, 모든 패키지(상담+검사 조합, 단일 검사 등) 구성을 패키지 등록 화면 빌더로 중앙화합니다.

## 2. 공통 디자인 시스템 및 레이아웃 (Pencil Guide 준수)
- **전체 레이아웃**: `AdminCommonLayout`을 사용하여 좌측 다크 사이드바(260px, `#2C2C2C`)와 우측 메인 콘텐츠 영역으로 구성.
- **상단 바**: `ContentHeader` 공통 모듈 사용 (브레드크럼, 페이지 제목, 우측 주요 액션 버튼 배치).
- **섹션 블록 (`mg-v2-section-block`)**:
  - 배경: `#F5F3EF` 또는 `#FAF9F7` (토큰: `var(--mg-surface)`)
  - 테두리: `1px solid #D4CFC8` (토큰: `var(--mg-border)`)
  - Corner Radius: `16px`
  - Padding: `24px`, 내부 Gap: `16px`
- **섹션 제목 (`mg-v2-accent-bar` 활용)**:
  - 좌측 세로 악센트 바 (폭 `4px`, 색상 `#3D5246`, radius `2px`)
  - 텍스트: `18px`, `fontWeight 600`, `#2C2C2C`
- **타이포그래피**: Noto Sans KR 적용.

## 3. 화면 1: 패키지 등록/수정 화면 (`PackagePricingDetailPage.js`)

이 화면은 패키지의 기본 정보, 상품 구성(빌더), 요금 및 할인율을 설정하는 SSOT 화면입니다.

### 3.1. 상단 바 (`ContentHeader`)
- **제목**: 패키지 요금 등록 (또는 수정)
- **액션 버튼**: 
  - [취소] 버튼 (아웃라인 스타일: `mg-v2-btn-outline`)
  - [저장] 버튼 (주조색 스타일: `mg-v2-btn-primary`, 배경 `#3D5246`, 텍스트 `#FAF9F7`)

### 3.2. 섹션 1: 기본 정보 블록
- **패키지명**: Text Input (필수)
- **패키지 설명**: Textarea (선택)
- **노출 여부**: Toggle Switch 또는 Radio Button (활성/비활성)

### 3.3. 섹션 2: 상품 구성 빌더 (핵심 SSOT 영역)
패키지에 포함될 상담 회기 및 검사 항목을 명확한 UI로 추가/조합하는 영역입니다.
- **레이아웃**: 좌측(추가 가능한 상품 목록) / 우측(현재 구성된 패키지 항목)의 2단 분할 구조 또는 상하 리스트 구조. 사용자 편의를 위해 **상하 리스트 + [항목 추가] 드롭다운 버튼** 구조를 권장합니다.
- **기능 요구사항**:
  - **단일 검사 패키지 지원**: 상담 회기가 0회인 패키지(예: MMPI 단일 검사)도 등록 가능해야 합니다.
  - **항목 추가 UI**: `[+ 상품 추가]` 버튼 클릭 시 드롭다운으로 "1회 상담", "MMPI 검사", "TCI 검사" 등 선택 가능.
  - **구성 리스트 UI**: 추가된 항목들은 카드 형태의 리스트로 나열되며, 우측에 `[삭제]` 아이콘 및 드래그 핸들(`DnD` 순서 변경용) 제공.
  - **디자인 토큰**: 
    - 리스트 아이템 배경: `#FFFFFF`, 테두리: `1px solid #D4CFC8`, Radius: `8px`
    - 드래그 핸들 아이콘 색상: `#8B7355` (Accent)

### 3.4. 섹션 3: 요금 및 할인 설정 블록
- **총 원가**: 구성 빌더에서 추가된 항목들의 원가 합계를 자동 계산하여 Read-only 텍스트로 노출 (`24px`, `fontWeight 600`, `#2C2C2C`).
- **할인 설정**: 
  - 할인율(%) 또는 할인 금액 입력 필드 제공.
- **최종 패키지 판매가**: (원가 - 할인) 금액을 크게 강조하여 노출 (`var(--mg-primary)` 색상 사용).

## 4. 화면 2: 매칭 생성 모달 (`MappingCreationModal.js`)

기존의 복잡한 즉석 상품 조합 기능을 완전히 제거하고, SSOT 원칙에 따라 사전에 만들어진 패키지를 **선택**만 하도록 단순화합니다.

### 4.1. 모달 레이아웃 (`UnifiedModal` 공통 모듈 사용)
- **너비**: `480px` ~ `600px` (기존 대비 축소 및 간소화)
- **배경**: `#FAF9F7`
- **버튼**: 하단 우측 정렬 [취소] [매칭 생성]

### 4.2. 입력 폼 구성
1. **내담자 선택**: Searchable Select (기존 유지)
2. **상담사 선택**: Searchable Select (기존 유지)
3. **패키지 선택 (SSOT 적용)**:
   - **UI**: 단일 Select Dropdown (`BadgeSelect` 또는 기본 Select)
   - **동작**: `/admin/package-pricing`에서 등록된 패키지 목록만 노출.
   - **설명**: "패키지 구성 및 할인은 [패키지 요금 관리] 메뉴에서 설정할 수 있습니다." 라는 안내 문구(Caption, `12px`, `#5C6B61`)를 하단에 배치.
4. **일정 선택**: 첫 회기 일정 선택 (상담이 포함된 패키지인 경우에만 노출)
