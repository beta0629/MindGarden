# 젠스파크(GenSpark) 제출용 PDF 제작 프롬프트

> 2026 예비창업패키지 **제출용 사업계획서 PDF**를 디자인 있게 만들어 달라고 요청할 때 사용하는 프롬프트입니다.  
> 아래 「===== 프롬프트 시작 =====」 ~ 「===== 프롬프트 끝 =====」 블록 전체를 복사해 젠스파크에 붙여넣으세요.

---

## 사용 방법

1. 젠스파크에서 **문서/PDF 생성** 또는 **디자인 문서 작성** 기능을 연다.
2. **프롬프트 시작 ~ 끝**까지 전부 복사해 입력란에 붙여넣는다.
3. 생성된 PDF를 다운로드한 뒤, 필요 시 일부 문구·숫자만 수정한다.
4. 출력이 마크다운/HTML이면 “PDF로 내보내기” 또는 인쇄 → PDF 저장으로 최종 PDF를 만든다.

===== 프롬프트 시작 =====

**[요청]**
2026년 예비창업패키지 **제출용 사업계획서**를 **디자인이 깔끔하고 보기 좋은 PDF**로 만들어 주세요.  
서류 심사관이 빠르게 읽을 수 있도록 **계층 구조가 분명하고, 표와 강조가 잘 된, 단정하면서도 세련된 레이아웃**으로 작성해 주세요.

---

**[디자인 지침 — 반드시 적용]**

1. **전체 톤**
   - **스타일**: 미니멀하고 전문적인 비즈니스 문서. 과한 장식 없음.
   - **색상**: 메인은 **진한 네이비(#1a2b3c) 또는 다크 그레이**, 보조는 **연한 그레이(#f5f6f8)** 배경, 강조는 **한 가지 액센트 컬러**(예: 틸/그린 #3D5246 또는 블루 #2563eb)만 사용해 통일감 있게.
   - **여백**: A4 기준 좌우·상하 여백 충분히(최소 20~25mm). 문단이 뭉쳐 보이지 않게.

2. **타이포그래피**
   - **표지 제목**: 크고 굵은 서체, 한 줄로 임팩트 있게.
   - **대제목(1장)**: 18~20pt, 볼드, 메인 컬러. 장 구분이 될 수 있도록 상단 여백 후 배치.
   - **중제목(1.1, 2.1 등)**: 14~16pt, 볼드. 본문과 시각적으로 구분.
   - **본문**: 10~11pt, 줄간격 1.4~1.5. 가독성 우선.
   - **표·캡션**: 표 내부 9~10pt, 표 제목은 11pt 볼드.

3. **레이아웃**
   - **표지**: 상단 또는 중앙에 문서 제목, 부제, 연도(2026). 로고나 서비스명만 간단히 넣어도 됨. 나머지는 여백으로 정리.
   - **목차**: 1페이지. 대제목만 나열해도 되고, 있으면 페이지 번호 연결.
   - **각 장 시작**: 대제목 전에 얇은 구분선(액센트 컬러 1px) 또는 작은 사각형 블록을 넣어 섹션 시작을 표시.
   - **표**: 헤더 행 배경색(연한 그레이 또는 액센트 10% 투명도), 구분선은 얇게. 셀 안쪽 패딩 충분히.
   - **핵심 수치**: 숫자·금액·비율은 **볼드** 또는 **액센트 컬러**로 강조해 한눈에 들어오게.
   - **불릿**: 동일한 기호(● 또는 -)로 통일, 들여쓰기 정렬.

4. **일관성**
   - 모든 장에서 제목 레벨별 폰트 크기·색상이 동일하게.
   - 표 스타일(헤더, 테두리, 셀 패딩) 통일.
   - 페이지 하단에 **페이지 번호**(예: 1 / 24) 넣어 주세요. (선택: 회사명·문서명 푸터)

5. **출력**
   - 최종 결과물은 **PDF**로 생성 가능하게 해 주세요. (또는 PDF 내보내기가 가능한 형식)

**표지(1페이지)**  
- 제목: **CoreSolution 2026 예비창업패키지 종합 창업계획서**  
- 부제: 상담·교육·1인 서비스업 통합 운영 SaaS  
- 2026년 예비창업자패키지 지원 신청  
- (선택) 작성·점검 반영일: 2025-12 작성, 2026-02 점검 반영  

**목차(1페이지)**  
1. 일반 현황  
2. 문제 인식 (Problem)  
3. 실현 가능성 (Solution)  
4. 현재 구현도  
5. 성장 전략 (Scale-up)  
6. 수익 모델 (Revenue Model)  
7. 실행 계획 (2026)  
8. 정부지원 필요성  
9. 리스크 및 대응  
10. 법인·팀·파트너십 (요약)  
11. 결론  

**1. 일반 현황**  
- 1.1 창업 개요: CoreSolution은 상담센터·교육기관·1인 서비스업의 운영에 필요한 핵심 기능을 하나의 SaaS로 제공하는 **멀티테넌트 통합 운영 플랫폼**이다.  
- 3대 강점: (1) 온보딩 표준화 (2) Ops 운영 포털 (3) CoreSolution(멀티테넌트+표준화+AI 모듈).  
- 1.2 팀 구성: **표** — 역할(대표·디자이너·추가 개발자·보안 외주), 현황, 계획(지원금 활용), 핵심 역량 한 줄.  
  - 현재 2명 → 2026년 목표 3명. **지원금 인건비는 비가족 채용만, 배우자·직계가족 제외** 문구 반드시 포함.

**2. 문제 인식 (Problem)**  
- 파편화된 운영(예약/기록/결제/정산/고객관리 분리), 정산·운영 리스크, 확장 시 통제·표준화 부재, 상담 도메인(심리검사 수기 정리·근거 부족).  
- 불릿 4~5개로 간결히.

**3. 실현 가능성 (Solution)**  
- 운영 통합 + 표준화 + AI 보조.  
- (A) 멀티테넌트 (B) 온보딩 표준화 — 필드 단위 저장, 하드코딩 금지, 표준 API (C) Ops — 환경 분리, 승인·관제 (D) 심리검사 AI — 암호화 저장, 추출/검증, 오판 방어.  
- **표**: 표준화 근거 5~6개만 — 보안/인증, DB 스키마, API 설계, 온보딩 저장, Ops 운영 등 문서명·버전·핵심 포인트.

**4. 현재 구현도**  
- 4.1 시스템 구현: 멀티테넌트·인증·상담 운영·ERP·심리검사 AI MVP·표준화 — 완료 항목 불릿.  
- **통계 표**: 엔티티 148개 | 서비스 370개 | 컨트롤러 132개 | 리포지토리 127개 (2026-02 점검 기준).  
- 1단계 MVP: 원장/상담사/내담자 대시보드·기본 예약·과금 시연 가능(React+Java).  
- **반드시 포함**: **현재 이미 동작 중인 관리자(대시보드) 화면 스크린샷** 1~2장을 넣어 "선정 즉시 사업화 가능"한 팀임을 시각적으로 보여 줄 것.  
- 온보딩·Trinity·Ops 요약 2~3줄씩.  
- 4.2 표준화 현황: **표** — 카테고리(CSS 변수화, 에러 처리, API 경로 등), 준수율, 상태. 전체 85%.  
- 4.3 배포: GitHub Actions CI/CD, 개발/운영 분리, 모니터링.

**5. 성장 전략 (Scale-up)**  
- 5.1 시장 진입: 2026 상담 → 2027 학원 → 2028 1인 기업.  
- 5.2 차별화: ERP급 정산, 온보딩 자동화, 심리검사 AI(근거·검증·오판 방어).  
- 5.3 경쟁사 분석: **표** — 경쟁사 A/B vs CoreSolution (All-in-One, ERP 정산, 멀티테넌시, 온보딩, AI, 타겟).  
- 5.4 시장 규모: **표** — 1단계(2026) 상담 시장 | 2단계(2027) 학원 90,385개 | 3단계(2028) 1인 기업 100만 개+.

**6. 수익 모델 (Revenue Model)**  
- SaaS 구독(MRR)+부가서비스+커스터마이징/SI.  
- 가격: Basic 월 3만 원, Pro 월 10만 원, Enterprise 별도.  
- **6.3 3개년 재무 표**: 2026 매출 25/비용 105/적자 80 | 2027 매출 180/비용 135/흑자 45(BEP) | 2028 매출 550/비용 190/영업이익 360 (단위 백만 원).  
- 손익분기점 2027년 Q2~Q3. PoC 10개·유료 전환 2개(20%).

**7. 실행 계획 (2026)**  
- 1단계 지원금 전략: 중간 평가 후 상위 약 50%만 2단계 추가 지급 → **1단계에서 시연 가능한 MVP**(대시보드·예약·과금) 집중 완성.  
- 제품: 1단계 MVP 안정화, 심리검사 AI 고도화, 품질 지표 수집.  
- 시장: PoC 확보, 유료 전환 기준 확립.  
- **KPI 표**: 온보딩 처리 시간 50% 단축, 표준 준수율 90%+, AI 검증 통과율 70%+, PoC 10개·유료 전환 20%+, MRR 2M+.  
- 분기별 요약(협약 6월 기준): **1단계(6~9월)** 핵심 대시보드·결제 모듈 안정화, 중간평가 시연 준비 | **2단계(10~1월)** AI 고도화, 유료 전환 실적 확보.

**8. 정부지원 필요성**  
- 8.1 필요 이유: 온보딩 표준화 완성도, Ops 운영 증빙, 심리검사 AI 양식 다양성 대응(인력·기간).  
- 8.2 미확보 시: 유료 확장 지연, AI 품질 루프 붕괴, 2026 상용화 마일스톤 지연.  
- **8.3 지원금 사용 계획 표**:  
  - **문구 필수**: 지원금 2단계 분할(1단계 약 2,000만 원 → 중간 평가 후 상위 약 50%에 2단계 추가). **배우자·직계가족 인건비 집행 불가.**  
  - 비목: 인건비(비가족 한함)·외주(보안·디자인)·인프라(서버·클라우드)·실증/초기 마케팅 — 집행 목적·핵심 산출물 간단 표기.

**9. 리스크 및 대응**  
- 보안/개인정보: 암호화·키관리·감사.  
- AI 오판: 구조 검증·근거 매칭·금지문구·폴백·사람 검수.  
- 양식 다양성: 템플릿 감지·검증 엔진·단계적 커버리지.  
- 법인/자금: 2026 Q1 법인 설립, 지원 미확보 시 단계적 자금·시드 유치.

**10. 법인·팀·파트너십 (요약)**  
- 법인: 2026 Q1 주식회사, 초기 자본금 5천만 원.  
- 지적재산권: 저작권·상표 2026 Q2.  
- 파트너: 클라우드(AWS/NCP), PG(토스페이먼츠), 소상공인 스마트상점 공급기업 등록.

**11. 결론**  
- MVP 완성으로 기술적 실현 가능성 확보.  
- 타겟·BM·시연 가능 MVP로 2026 선정(70점 이상)·2단계 수급 목표.  
- 지원금으로 온보딩/Ops/AI 고도화 → 2026년 내 상용화 마일스톤 달성.  
- 핵심 성공 요인 5개 불릿(기술 실현, 시장 전략, 차별화, 재무 계획, 단계적 로드맵).

## 프롬프트 수정 팁

- **페이지 수**: 보통 15~25장 내외가 적당합니다. "각 장을 1~2페이지로 압축"이라고 넣으면 더 짧게 만들 수 있습니다.
- **로고**: "표지에 CoreSolution 로고 이미지를 넣을 자리만 비워 두어라"고 하면, 나중에 PDF 편집으로 로고만 삽입할 수 있습니다.
- **특화분야** 제출 시: "심리검사 AI(3, 4, 7, 8장) 비중을 1~2페이지 더 늘려라"고 추가하면 됩니다.
