# MindGarden Phase 1 (Academy) MVP Plan

> **제품 원칙 (80/20 전략)**  
> 모든 학원의 100% 요구사항을 맞추려는 대신, 전체 학원의 약 80%가 공통으로 사용하는 핵심 로직을 표준 기능으로 제공하고, 나머지 20%는 MindGarden의 설정/규칙 엔진으로 대응한다.  
> Phase 1에서는 아래 다섯 가지 흐름을 “표준화된 베스트 프랙티스”로 정의한다.
>
> 1. 신규생 상담 및 등록  
> 2. 반(Class)/강좌(Course) 배정 및 수강 관리  
> 3. 출결 관리 및 학부모 알림  
> 4. 월별 수강료 청구 및 결제  
> 5. 정산(수강료/강사/본사) 자동화

## 1. 사용자 페르소나 및 핵심 시나리오

### 1.1 페르소나

- **학원 원장 (테넌트 관리자)**
  - 목표: 지점 운영, 수강생 관리, 매출 파악
  - Pain Point: 수강료 정산 지연, 수동으로 처리되는 회원/출결 관리
- **데스크 직원 (지점 스태프)**
  - 목표: 상담 예약, 수강 등록, 결제 처리, 알림 발송
  - Pain Point: 여러 시스템을 오가며 정보를 입력해야 하는 비효율
- **학부모/수강생 (소비자)**
  - 목표: 강좌 탐색, 상담 예약, 수강료 결제, 일정 확인
  - Pain Point: 지점별로 다른 가입/결제 방식, 결제 영수증/내역 확인의 어려움

### 1.2 핵심 시나리오

1. **상담 예약/수강 등록**
   1. 소비자가 포털에서 학원을 탐색 후 상담 예약을 생성
   2. 데스크 직원이 예약을 확인하고 상담을 진행
   3. 상담 결과에 따라 수강 등록(수강권 생성) 및 초기 결제 진행
2. **반 배정 및 수강 관리**
   1. 상담 완료된 학생을 반(Class)/강좌(Course)에 배정
   2. 시간표/좌석/담당 강사를 지정하고 수강 일정 관리
3. **출결 및 알림**
   1. 모바일/태블릿에서 출석/지각/결석 기록
   2. 학부모에게 출결/공지 알림 발송 (앱/문자/카카오톡 등)
4. **수강료 결제 및 청구**
   1. 월별 청구 스케줄을 설정(고정일, 회차 기반 등)
   2. 학부모에게 청구서(Invoice) 발송 → 결제 완료 → 영수증 저장
5. **정산(수강료/강사/본사)**
   1. 소비자가 카드/계좌/간편결제로 수강료를 결제
   2. 결제 정보가 ERP와 연동되어 정산 배치 대상에 포함
   3. 정산 배치(PL/SQL)가 매출/수수료/정산금을 계산 -> 관리자 포털 리포트 제공
6. **회원/통계 관리**
   1. 원장이 관리자 포털에서 수강생 정보, 결제 이력, 매출 추이를 확인
   2. 상위 본사(HQ)가 전체 학원들의 KPI를 모니터링
7. **커뮤니티 DM 기능** (선택적, Phase 2에서 우선순위 높음)
   1. 학생 간 실시간 메시지 교환 (1:1 DM)
   2. 입력 중 표시, 읽음 상태 표시
   3. **비즈니스 가치**: 시작 단계 기능이라도 없는 것보다는 있는 것이 학원 선택 시 차별화 포인트

## 2. 기능 요구사항 및 우선순위

### 2.1 P0 (MVP 필수)

- 테넌트 온보딩: 학원 정보, 지점, 직원 계정 등록
- 소비자 포털: 학원 탐색, 상담 예약, 수강 등록/결제
- 관리자 포털:
  - 상담/예약 승인, 수강 등록 관리
  - 반(Class)/강좌(Course) 관리, 학생 배정, 시간표 설정
  - 출결 기록 및 학부모 알림 발송
  - 월별 청구 스케줄 작성, 청구서 발송, 결제 상태 확인
- 결제 연동: 카드/계좌/간편결제, 영수증 발급
- 청구/정산 배치:
  - 월별 청구 생성(Invoice Scheduler)
  - PL/SQL 기반 매출/수수료/강사 정산 계산, HQ 로열티 리포트
- 알림/통지: 예약 확정, 출결, 청구, 결제 확인, 정산 정보 알림 (이메일/문자/앱)
- SessionManager/보안: 공개 페이지 접근 허용, 인증 흐름 정의

### 2.2 P1 (MVP 이후 빠른 확장)

- 출결 관리 고급 UI (좌석/좌석배치), 학부모 알림 템플릿 커스터마이징
- 할인 정책/요금제 규칙 엔진(형제 할인, 장기 등록, 수강권 패키지)
- 강사 급여 방식 확장(시급+성과 혼합, 외부 강사 정산)
- 쿠폰/프로모션 관리(학원별)
- 분석 대시보드(기초 KPI 시각화)
- 규칙 기반 설정으로 20% 커스터마이징 대응 (재원관리, 휴원 처리 등)
- **학원 Quick Win 기능** (간단하면서도 비즈니스 가치 높음) ⭐ **우선순위: 높음**
  - **공지사항 게시판** (2-3일) - 학원-학부모 소통 채널 강화, 수업 변경/이벤트 전달
  - **출석 체크 QR 코드** (3-4일) - 출석 체크 자동화, 실시간 출석 현황 확인
  - **학부모 일일 리포트** (2-3일) - 매일 저녁 자동 발송 (출석/결석 요약, 오늘의 수업 내용)
  - **수강권 잔여 횟수 표시** (1-2일) - 투명한 수강권 관리, 수강권 연장 유도
  - **생일 축하 알림** (1일) - 인간적인 서비스로 브랜드 이미지 향상
  - **과제/숙제 관리** (3-4일) - 과제 관리 자동화, 제출 현황 확인
  - **간단한 설문조사** (2-3일) - 학원 운영 개선을 위한 피드백 수집

### 2.3 P2 (중기 확장)

- 모바일 앱 알림 푸시 연동
- 외부 학원 플랫폼/마케팅 채널과의 연동 (API 파트너십)
- BI/심화 분석, 추천 시스템
- HQ용 정책 브로드캐스트, 프랜차이즈 수수료 변동 시뮬레이터
- **커뮤니티 DM 기능** (WebSocket 기반 실시간 메시징) - 학생 간 소통 활성화

## 4. QA 및 테스트 전략

- **테스트 범위**
  - 유닛 테스트: 핵심 도메인 서비스, 정산 로직, 알림 전송
  - 통합 테스트: 예약→결제→정산→리포트 흐름, 멀티테넌시 접근 제어
  - E2E 테스트: 소비자 예약/결제 플로우, 관리자 승인/리포트 확인
  - 성능 테스트: 정산 배치(10만 건 기준), API SLA 95% ≤ 500ms
- **QA 계획**
  - QA 환경에서 자동화 테스트(Cypress/Playwright) 실행
  - 시각적 회귀(Percy/Chromatic)로 디자인 시스템 검증
  - 보안 테스트: OWASP Top 10 점검, 세션/쿠키 관리 검토
  - 사용자 검증: 파일럿 학원 1~2곳 대상 UAT(User Acceptance Test)
- **릴리즈 체크리스트**
  - 멀티테넌시 마이그레이션 스크립트 적용 여부
  - 정산 결과/알림 로그 검증, 롤백 플랜 준비
  - 운영 매뉴얼 및 지원 조직 교육 완료
