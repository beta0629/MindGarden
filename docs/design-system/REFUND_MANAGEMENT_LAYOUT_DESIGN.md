## 1. 개요 및 설계 원칙

### 1.1 목적

- **환불 관리 시스템** 페이지(`/erp/refund-management`)를 B0KlA(마인드가든 어드민 대시보드) 스타일로 **완전히 새로운 레이아웃**으로 설계한다.
- 관리자가 **한눈에** 환불 건수·금액·회기·연동 상태를 보고, **기간·상태 필터**로 이력을 조회하며, 필요 시 **ERP 환불 반영 수동 트리거**를 사용할 수 있게 한다.
- **예약금**: 노출/입력 없음. 수기 처리만 전제.

### 1.2 단일 소스·토큰

- **디자인 단일 소스**: `mindgarden-design-system.pen`, `pencil-new.pen`, `unified-design-tokens.css`
- **색상**: `var(--mg-color-*)`, `var(--ad-b0kla-*)` 토큰만 사용. hex 직접 사용 금지.
- **레이아웃**: 사이드바 260px, 상단 바 56~64px, 본문 패딩 24~32px. 섹션은 **섹션 블록**(배경·테두리·radius·좌측 악센트) 단위.
- **타이포**: Noto Sans KR. 제목 20~24px/600, 본문 14~16px, 라벨/캡션 12px.

### 1.3 적용 체크리스트 (PENCIL_DESIGN_GUIDE)

- [x] 색상은 팔레트·토큰만 사용
- [x] 레이아웃: 사이드바 + 메인, 상단 바(브레드크럼·제목·액션), 본문 섹션 블록
- [x] 섹션 블록: 배경·테두리·radius·좌측 세로 악센트(4px) + 제목
- [x] 카드·메트릭: 숫자 24px/600, 라벨 12px, 좌측 악센트로 구분
- [x] 스펙에 토큰명·클래스명(`var(--mg-*)`, `mg-v2-*`) 명시

## 2. 페이지 구조 (정보 계층)

전체는 **AdminCommonLayout** 내 **ContentArea**로, 순서는 아래와 같이 고정한다.

| 순서 | 영역 | 역할 |
|------|------|------|
| 0 | **ContentHeader** | 페이지 제목·부제·주요 액션(운영 현황으로 돌아가기) |

| 2 | **필터·제어 영역** | 기간·상태 선택, 새로고침, 엑셀 내보내기(선택), ERP 환불 반영 트리거(선택 배치) |

| 6 | **회계 처리 현황** | 회계 반영 여부·현황 요약 |

## 3. 섹션별 레이아웃·와이어프레임

### 3.1 ContentHeader (상단 바)

- **위치**: 메인 영역 최상단. 배경 `var(--mg-color-background-main)`, 하단 1px `var(--mg-color-border-main)`.
- **구성**:
  - 좌: 브레드크럼(예: ERP > 환불 관리)
  - 중/좌: 제목 **「환불 관리 시스템」** (20~24px, fontWeight 600, `var(--mg-color-text-main)`)
  - 부제(선택): **「상담 환불 현황 및 환불·결제 연동」** (14px, `var(--mg-color-text-secondary)`)
  - 우: **「운영 현황으로 돌아가기」** 아웃라인 버튼 — 배경 없음, 테두리 `var(--mg-color-border-main)`, height 40px, radius 10px.
- **클래스·토큰**: 기존 ContentHeader / `mg-v2-ad-b0kla__header` 등 레이아웃 공통 사용.

```
┌─────────────────────────────────────────────────────────────────────────┐
│ ERP > 환불 관리                    환불 관리 시스템    [운영 현황으로 돌아가기] │
│ 상담 환불 현황 및 환불·결제 연동                                          │
└─────────────────────────────────────────────────────────────────────────┘
```

### 3.2 환불 KPI 블록 (4열 카드)

- **역할**: 한눈에 환불 건수·금액·회기·연동 상태 파악.
- **구조**: 하나의 **섹션 블록** 안에 카드 4개를 가로 배치(그리드 4열). 데스크톱 4열, 태블릿 2열, 모바일 1열.
- **섹션 블록**:
  - 배경: `var(--mg-color-surface-main)` (#F5F3EF)
  - 테두리: 1px `var(--mg-color-border-main)`, border-radius 16px
  - 패딩: 24px(데스크톱), 내부 gap 24px
  - 제목(선택): 좌측 **세로 악센트 바** 4px `var(--mg-color-primary-main)`, radius 2px + 텍스트 "환불 현황 요약" 16px/600
- **카드 1장 (Molecule)**:
  - 배경: 동일 서페이스 또는 약간 밝게. 테두리 1px, radius 16px.
  - **좌측 세로 악센트** 4px — 카드별로 색 구분 권장(건수: primary, 금액: accent, 회기: secondary, 연동: primary/보조).
  - **숫자**: 24px, fontWeight 600, `var(--mg-color-text-main)`
  - **라벨**: 12px, `var(--mg-color-text-secondary)` (예: "환불 건수", "환불 금액(원)", "환불 회기", "ERP 연동 상태")
  - (선택) 보조 텍스트: 기간 범위 등 12px
- **토큰**: `var(--mg-color-surface-main)`, `var(--mg-color-border-main)`, `var(--mg-color-primary-main)`, `var(--mg-color-accent-main)`, `var(--mg-color-text-main)`, `var(--mg-color-text-secondary)`.  
- **클래스 제안**: `refund-kpi-block`, `refund-kpi-block__card`, `mg-v2-ad-b0kla__card` 등. KPI 카드는 기존 대시보드 StatCard/KPI 타일과 동일 패턴으로 재사용 권장.

```
┌─ 환불 현황 요약 (세로 악센트 4px + 제목) ─────────────────────────────────┐
│ ┌──────────────┐ ┌──────────────┐ ┌──────────────┐ ┌──────────────┐       │
│ │ ▌ 12         │ │ ▌ 1,250,000  │ │ ▌ 24         │ │ ▌ 연동 완료  │       │
│ │   환불 건수   │ │   환불 금액   │ │   환불 회기   │ │   ERP 상태   │       │
│ │   (기간 내)   │ │   (원)       │ │   (회)       │ │   N건         │       │
│ └──────────────┘ └──────────────┘ └──────────────┘ └──────────────┘       │
└────────────────────────────────────────────────────────────────────────────┘
```

### 3.3 필터·제어 영역

- **역할**: 기간·상태로 이력 필터, 새로고침, (선택) 엑셀 내보내기, (선택) ERP 환불 반영 수동 트리거 배치.
- **구조**: 하나의 **섹션 블록**. 내부는 1~2행.
  - **1행**: 필터 그룹 — 기간(시작일·종료일 또는 기간 프리셋), 상태(전체/연동완료/미연동/실패 등) 드롭다운. 우측에 버튼: 새로고침, 엑셀 내보내기(선택).
  - **2행(선택)**: "선택 건 ERP 환불 반영" 같은 **일괄 수동 트리거** 버튼. 또는 테이블 행 단위 액션만 둘 경우 이 행 생략 가능.
- **섹션 블록**: 동일 — 배경·테두리·radius 16px, 패딩 24px. 제목: 세로 악센트 4px + "조회 조건".
- **컨트롤**: select/date input은 디자인 토큰·`pencil-new.pen` 아토믹 컴포넌트 스타일. 버튼: 주조(새로고침 반영 시), 아웃라인(엑셀, ERP 반영).
- **ERP 환불 반영 배치 제안**:
  - **옵션 A**: 필터 영역 우측에 "선택 건 ERP 환불 반영" 버튼(테이블에서 선택한 행에 대해 일괄 호출).
  - **옵션 B**: 테이블 각 행에 "ERP 반영" 액션 버튼만 두고, 상단 일괄 버튼 없음.
  - **권장**: 둘 다 지원 — 필터 영역에 일괄 버튼 + 테이블 행에 개별 "ERP 반영" 버튼. (구현은 코더·API 연동 포인트에 따름)
- **토큰·클래스**: `var(--mg-color-border-main)`, `var(--mg-spacing-md)`, `var(--mg-spacing-lg)`. 클래스: `refund-filters`, `refund-filters__group` 등.

```
┌─ 조회 조건 (세로 악센트 4px + 제목) ──────────────────────────────────────┐
│ 기간 [시작일] ~ [종료일]  상태 [전체 ▼]  [새로고침] [엑셀 내보내기] [선택 건 ERP 환불 반영] │
└────────────────────────────────────────────────────────────────────────────┘
```

### 3.4 환불 이력 테이블

- **역할**: 환불일시, 내담자, 상담사, 패키지, 환불 회기, 환불 금액, 환불 사유, ERP 상태, (선택) 행 액션(ERP 반영).
- **구조**: 하나의 **섹션 블록** 안에 테이블. 제목: 세로 악센트 4px + "환불 이력".
- **테이블**:
  - 헤더: 12px, `var(--mg-color-text-secondary)`, 배경 약간 구분(선택). 컬럼: 환불일시 | 내담자 | 상담사 | 패키지 | 환불 회기 | 환불 금액 | 환불 사유 | ERP 상태 | (액션)
  - 본문: 14px, `var(--mg-color-text-main)`. ERP 상태는 배지(연동완료/미연동/실패 등) — 배지 색은 토큰(primary/success, warning, error 등)으로 구분.
  - **행 액션**: 각 행 "ERP 반영" 버튼(텍스트 또는 아이콘). 아웃라인·소형.
- **페이지네이션**: 테이블 하단. 기존 Admin 패턴·토큰 사용.
- **반응형**: 데스크톱 — 가로 스크롤 허용. 태블릿/모바일 — 가로 스크롤 또는 카드형 리스트(행을 카드로 세로 쌓기) 전환.
- **토큰**: `var(--mg-color-surface-main)`, `var(--mg-color-border-main)`, `var(--mg-color-text-main)`, `var(--mg-color-text-secondary)`.
- **클래스 제안**: `refund-history-table`, `refund-history-table__row`, `mg-v2-ad-b0kla__table` 등.

```
┌─ 환불 이력 (세로 악센트 4px + 제목) ─────────────────────────────────────┐
│ 환불일시    │ 내담자 │ 상담사 │ 패키지 │ 회기 │ 금액    │ 사유   │ ERP상태 │ 액션   │
```

## 5. 토큰·클래스 요약 (코더 구현용)
