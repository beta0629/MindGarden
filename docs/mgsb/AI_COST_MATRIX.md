## 3. 비용 산정 가이드

1. **토큰당 비용 모델**  
   - `(월간 토큰 수) × (토큰당 비용)` = 사용량 기반 요금  
   - 예: GPT-4o Mini 월 300만 토큰 → 300만 × $0.15/1K ≈ $450 (~60만원)  
   - 챗봇/콘텐츠 생성 등은 토큰 기반이 일반적
2. **호스티드 vs. 온프레미스**  
   - 호스티드: 초기 비용 낮고 운영 안정성 확보 (벤더 SLA)  
   - 온프레미스: 대규모 텐서, 민감 데이터 처리 시 장점 → GPU, 전력, 운영 인력 필요
3. **학습/파인튜닝 비용**  
   - Fine-tuning: 데이터 준비 + 학습 비용 → 보통 $2~4/시간, 수십~수백 시간 필요  
   - 지식 베이스(RAG) 구성 시 벡터 DB/스토리지 비용 추가
4. **추가 비용**  
   - 모니터링/로그 비용 (CloudWatch, ELK, APM)  
   - 보안/Compliance (PII 마스킹, 접근 제어)  
   - 개발/운영 인력

| 항목 | 단위 | 요금 산정 기준(예시) | 비고 |

| 맞춤형 AI 모델 | 프로젝트 단위 | 커스터마이징 비용 + 유지보수료 | 대형 고객(프랜차이즈 HQ) 대상 |
| 테넌트용 AI 마켓플레이스 | 선택 서비스 | MindGarden 수수료 10~20% + 벤더 비용 | 선택형(Opt-in), 테넌트가 활성화 시 과금 |

> *요금제는 2026 Q4 SaaS 상용화 단계에서 확정 예정. MVP 단계에서는 비용 데이터 수집 및 테스트 목적 무료/비공개 베타로 운영.*

### 4.1 테넌트 AI 서비스 선택 모듈
- **옵션**: 테넌트 온보딩 시 AI 서비스(챗봇, 웰니스, 로그 요약 등)를 카드 리스트로 제공  
- **정보 표시**: 모델별 용도, 장단점, 추정 비용, 데이터/보안 주의사항  
- **과금 방식**: 기본료 + 사용량(토큰/요청) + MindGarden 수수료(10~20%)  
- **한도 설정**: 월별 사용량 한도, 초과 시 알림/자동 중단 옵션  
- **보고서**: 관리자 포털에서 월별 AI 사용량/비용 리포트 제공, HQ용 집계 포함  
- **주의**: 선택 서비스(Opt-in)인 만큼, 테넌트가 동의한 경우에만 활성화; 추후 업셀링/추가 기능 제안 가능

### 4.2 테넌트별 AI 과금/정산 구조 (안)
- **데이터 모델**  
  - `tenant_ai_subscription`: 테넌트 ↔ 선택한 AI 상품/모델 매핑, 기본 요금·수수료율·한도 저장  
  - `tenant_ai_usage_daily`: 토큰/요청 단위 사용량 일별 집계(모델, 기능, 사용자)  
  - `tenant_ai_invoice_monthly`: 월별 청구 총액(벤더 원가, MindGarden 수수료, 할인/프로모션, 세금) 기록  
  - `ai_pricing_catalog`: 상품별 기본요금, 구간별 단가, MindGarden 마진(30~40% 기본, 고급 기능 50%까지) 정의
- **정산 프로세스**  
  1. 실시간 사용량 수집 → 큐/스트림 → 일별 배치로 `tenant_ai_usage_daily` 적재  
  2. 월말 PL/SQL 배치가 사용량 × 단가 계산, MindGarden 수수료(기본 30~40%, 고급 플랜 50%까지) 반영  
  3. `tenant_ai_invoice_monthly` 생성 후 HQ/지점 관리자에게 인보이스 및 알림 발송  
  4. 결제 상태(자동이체/카드)를 추적하고 미납 시 알림 및 서비스 제한 옵션 적용
- **대시보드/리포트**  
  - HQ/테넌트별 월간 사용량·과금 내역, 모델별 비용 비중, 예상 다음 달 비용 예측 제공  
  - 초과 사용/예산 한도 근접 시 실시간 알림 제공  
  - 업셀링: 고급 기능(보안 모니터링, 인사이트 리포트) 추천 카드 노출
- **보안/컴플라이언스**  
  - 청구 데이터는 멀티테넌시 분리 기준 적용, HQ 집계 시 권한 검증  
  - PII/결제 정보는 PersonalDataEncryption 및 키 로테이션 정책 준수  
  - 감사 로그와 연동해 AI 사용량·과금 변경 이력 추적

> **요금제 운영 가이드**  
> - 기본 요금제(Starter/Standard/Premium)는 위 “기본 제공” 범위에서 사용자 수, 지점 수, 월간 토큰량, 저장 공간 등 한도를 차등 적용  
> - 추가 요금제는 기능 패키지 단위(`AI Advanced Pack`, `HQ Analytics Pack`, `Security Compliance Pack` 등)로 묶어 업셀링 구조 마련  
> - 가격표는 분기별 업데이트, 테넌트 포털에서 실시간 업그레이드/다운그레이드 및 견적 요청 가능하도록 구성  
> - 모든 애드온 변경(활성화/해지)은 감사 로그에 기록하고 관리자 2차 인증(2FA)으로 확인

### 4.5 로그/AI 요금제 권한 구조 (초안)

| 역할 | 주요 책임 | 권한 범위 | 변경 승인 체계 |
| --- | --- | --- | --- |
| MindGarden HQ Master | 전체 요금 정책 수립, 수수료율 변경, 글로벌 로그 정책 승인 | 모든 테넌트 데이터 접근, 가격표/애드온 카탈로그 수정, 로그 보존 정책 | 이사회/대표 승인, 정책 변경 시 테넌트 공지 및 재동의 관리 |
| MindGarden HQ Admin | 요금제 배포, 애드온 승인/해지, 로그 감사 운영 | 테넌트별 구독 현황 열람, 애드온 활성화 처리, 감사 로그/보안 이벤트 열람 | HQ Master 사전 승인 또는 워크플로우(전자결재) |
| Tenant Owner (테넌트 대표) | 본인 테넌트 요금제 선택, AI 서비스 한도 설정, 로그 조회 범위 관리 | 테넌트 데이터/로그 조회, 요금제 업/다운그레이드 요청, 애드온 활성화 요청 | HQ Admin 워크플로우 승인 필요, 긴급 변경 시 2FA 및 이메일 확인 |
| Tenant Manager (지점 관리자) | 지점 단위 AI 사용량 확인, 로그 모니터링, 사용자 권한 배분 | 본인 지점 데이터, 지점별 알림/보고 설정 | Tenant Owner 승인 후 일부 자율 변경 가능 |
| Security Officer (공통) | 보안/컴플라이언스 감사, 로그 무결성 체크 | 로그/감사 데이터 전용 접근, 가격 변경 기록 검증 | 독립 감사 체계를 통해 HQ Master 보고 |

- **정책 메모:**  
  - 요금제/애드온 변경은 `tenant_subscription`/`subscription_addon`에 감사 로그를 남기고, 역할별 승인 플로우를 `approval_user_id`, `approval_status` 필드로 추적  
  - 로그 데이터 접근은 RBAC + ABAC 조합(테넌트/지점 범위, 목적 코드)을 적용하고, 민감 로그 열람 시 사유 기록 필수  
  - AI 사용 한도 초과/요금 변경 이벤트는 HQ Admin과 Tenant Owner 모두에게 실시간 알림을 발송

## 5. 참고 사항

- 벤더별 가격 정책은 수시로 변동하므로 분기마다 업데이트 필요
