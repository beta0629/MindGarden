# 프로시저 배포 문제 해결 방안

## 🔴 문제 요약
운영 환경에서도 동일한 배포 실패가 발생할 가능성이 매우 높습니다.

## 원인 분석
1. **LEAVE 문 오류**: 라벨 없이 `LEAVE;` 사용 → 수정 완료
2. **12번째 줄 구문 오류**: 지속적으로 발생, 원인 미확인
3. **파일 인코딩/줄바꿈**: 가능성 있음

## 해결 방안

### 방안 1: 기존 작동 프로시저 형식 정확히 따르기
- 개발 DB에 이미 존재하는 프로시저(`AddSessionsToMapping` 등)의 형식 분석
- 동일한 형식으로 재작성

### 방안 2: 단계별 배포
1. DROP PROCEDURE만 먼저 실행
2. CREATE PROCEDURE를 별도로 실행
3. 각 단계별 오류 확인

### 방안 3: MySQL 클라이언트 직접 사용
- 서버에 직접 접속하여 mysql 클라이언트로 실행
- DELIMITER 사용 가능

### 방안 4: Flyway 마이그레이션 사용
- Flyway를 통한 프로시저 배포
- 버전 관리 및 롤백 가능

## 권장 조치
1. ✅ **기존 작동 프로시저 형식 분석**: `AddSessionsToMapping` 등 분석
2. ✅ **단계별 배포 테스트**: DROP → CREATE 분리 실행
3. ✅ **배포 스크립트 개선**: 성공 사례 기반 재작성
4. ✅ **운영 배포 전 검증**: 개발 환경에서 반드시 성공 확인
