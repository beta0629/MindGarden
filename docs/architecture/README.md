## 📖 문서 작성 가이드

### 1. 표준 문서 참조
모든 개발은 `TENANT_ROLE_SYSTEM_STANDARD.md`를 기준으로 합니다.

### 2. 브랜치 개념 제거
- ❌ `branchCode`, `branchId` 사용 금지
- ✅ `tenantId` 사용
- ❌ 지점별 필터링 금지
- ✅ 테넌트별 필터링

### 3. 역할 관리
- 테넌트별 역할 생성/수정/삭제
- 커스텀 역할 무제한 생성 가능
- 기본 역할(ADMIN, CONSULTANT, CLIENT, STAFF) 삭제 불가

### 4. 비즈니스 타입
- 온보딩 시 선택 (CONSULTATION, ACADEMY, HOSPITAL)
- 생성 후 변경 불가
- 업종별 자동 대시보드 생성
