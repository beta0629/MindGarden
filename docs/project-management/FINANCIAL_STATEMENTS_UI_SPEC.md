# 재무제표(대차대조표·손익계산서) UI/UX 개선 스펙 — 데이터 없음·0·에러 상태

> **목적**: 대차대조표·손익계산서 탭에서 "데이터를 불러오는 중", "데이터 없음", "전부 0", "API 에러" 상태의 UI·문구·레이아웃을 B0KlA·unified-design-tokens 기준으로 일관되게 정의한다.  
> **기준 문서**: `docs/project-management/FINANCIAL_STATEMENTS_ZERO_FIX_PLAN.md`, `docs/design-system/PENCIL_DESIGN_GUIDE.md`, `frontend/src/styles/unified-design-tokens.css`  
> **산출물**: 코더가 추측 없이 반영할 수 있는 스펙·체크리스트. **코드 수정은 core-coder가 수행.**

---

## 1. 개요 및 배경

- **배경**: 재무제표 탭에서 (1) 로딩 시 문구만 있고 레이아웃 일관성 부족, (2) 데이터 없음/빈 응답 시 별도 안내 없이 0만 나열됨, (3) 전부 0인 경우 에러로 오인될 수 있음, (4) API 에러 시 토스트만 있고 재시도 유도 없음, (5) 대차대조표·손익계산서 탭 간 톤·레이아웃 불일치 가능성이 있음.
- **목표**: 위 상태별로 **동일한 비주얼 언어**(B0KlA·어드민 대시보드 샘플)와 **동일한 패턴**을 적용하여, 사용자가 "로딩 중 / 데이터 없음 / 정상(0 포함) / 에러"를 명확히 구분하고, 에러 시 재시도할 수 있도록 한다.

---

## 2. 적용 범위·일관성 원칙

| 적용 대상 | 비고 |
|-----------|------|
| **대차대조표 탭** | `IntegratedFinanceDashboard.js` 내 `BalanceSheetTab` |
| **손익계산서 탭** | 동일 파일 내 `IncomeStatementTab` |
| **공통** | 로딩·빈 데이터·전부 0·에러 상태의 **문구·레이아웃·토큰**은 두 탭에서 **동일한 패턴**으로 적용 |

- **레이아웃**: 두 탭 모두 기존 `DashboardSection` + 본문 영역 구조 유지. "상태 전용 블록"(로딩/빈 데이터/에러)은 **섹션 블록 안**에 배치하여 상단 바(기준일·기간 선택)와 시각적으로 같은 레벨을 유지한다.

## 3. 상태별 UI/UX 스펙

### 3.1 로딩 상태 — "데이터를 불러오는 중"

| 항목 | 스펙 |
|------|------|
| **표시 조건** | API 요청 중(`loading === true`) |
| **문구** | **대차대조표**: `데이터를 불러오는 중…` / **손익계산서**: `데이터를 불러오는 중…` (동일 문구 권장. 기존 "대차대조표를 불러오는 중..." 등도 허용하되, **탭 간 통일** 시 위 문구 사용) |
| **컴포넌트** | 기존 `UnifiedLoading` 재사용. `text="데이터를 불러오는 중…"`, `size="medium"`, `type="inline"` |
| **레이아웃** | 탭 전환 시 즉시 표시. **섹션 블록 내부**에 로딩을 넣어, 상단의 기준일자/기간 입력 UI는 그대로 두고, 그 아래 콘텐츠 영역만 로딩 스피너+문구로 채움. (전체 탭을 로딩으로만 채우지 않고, `DashboardSection` + 내부 로딩으로 구조 유지) |
| **토큰** | 로딩 컴포넌트는 기존 디자인 시스템 토큰 사용. 배경은 `var(--mg-color-surface-main)` 또는 `var(--mg-color-background-main)` 유지 |

**체크리스트**

- [ ] 로딩 시 문구가 "데이터를 불러오는 중…"으로 통일되어 있는가?
- [ ] 로딩이 섹션 블록(카드/패널) 안에 표시되어, 상단 날짜/기간 선택과 시각적으로 같은 레이아웃인가?
- [ ] 대차대조표·손익계산서 탭 모두 동일한 로딩 패턴을 사용하는가?

### 3.2 데이터 없음 — "해당 기간 데이터가 없습니다"

| 항목 | 스펙 |
|------|------|
| **판단 조건** | API 성공(`success === true`)이지만, **유효한 데이터가 없음**. (응답 `data`가 `null`/`undefined`이거나, 대차대조표는 summary/자산·부채·자본 총계가 모두 없음, 손익계산서는 revenue/expenses/netIncome 등이 없거나 전부 0인 "빈 집계"로 정의할 수 있음. 구현 시 "데이터 없음"과 "전부 0" 구분은 §3.3와 협의) |
| **문구** | **제목(강조)**: `데이터 없음` 또는 `해당 기간 데이터가 없습니다` (둘 중 하나로 통일 권장: **`해당 기간 데이터가 없습니다`**) |
| **보조 문구** | **대차대조표**: `선택한 기준일자에 등록된 원장 데이터가 없습니다.` **손익계산서**: `선택한 기간에 등록된 원장 데이터가 없습니다.` (선택) |
| **레이아웃** | **섹션 블록** 안에 배치. 배경 `var(--mg-color-surface-main)`, 테두리 `1px solid var(--mg-color-border-main)`, `border-radius: 16px`, 패딩 `var(--mg-spacing-24)`. 세로 중앙 정렬, 텍스트 중앙 정렬. 아이콘(선택): 회계/문서 관련 아이콘(예: FileText, PieChart) — 색상 `var(--mg-color-text-secondary)` |
| **토큰** | 제목: `font-size: 16px`, `font-weight: 600`, `color: var(--mg-color-text-main)`. 보조 문구: `font-size: 14px`, `color: var(--mg-color-text-secondary)`. 배경/테두리: 위와 동일 |
| **아토믹** | 기존 `mg-v2-empty-state` 또는 `erp-empty-state` 패턴 재사용. 클래스명 예: `mg-v2-empty-state`, `finance-statement-empty` (프로젝트 네이밍 규칙에 맞게 결정) |

**체크리스트**

- [ ] "데이터 없음" 상태가 테이블/카드 그리드 대신 **단일 빈 상태 블록**으로 표시되는가?
- [ ] 문구가 "해당 기간 데이터가 없습니다"(또는 "데이터 없음")로 통일되어 있는가?
- [ ] 대차대조표는 기준일자, 손익계산서는 기간 기준 문구로 구분했는가?(선택)
- [ ] 빈 상태 블록이 B0KlA 섹션 블록(배경·테두리·radius·패딩)을 따르는가?

### 3.3 전부 0인 상태 — 에러처럼 보이지 않게 하기

| 항목 | 스펙 |
|------|------|
| **판단 조건** | API 성공, 데이터 구조는 있으나 **모든 금액이 0** (대차대조표: 자산·부채·자본 총계 0; 손익계산서: 수익·비용·순이익 0) |
| **표시 방식** | **옵션 A(권장)**: 기존과 동일하게 숫자 0으로 표시하되, **상단에 짧은 안내 문구**를 넣어 "해당 기간에 발생한 거래가 없어 0으로 표시됩니다"처럼 명시. 에러 메시지 톤이 아니라 **정보성 문구**. |
| **안내 문구** | **대차대조표**: `기준일자까지 등록된 거래가 없어 금액이 0으로 표시됩니다.` **손익계산서**: `선택한 기간에 등록된 거래가 없어 금액이 0으로 표시됩니다.` (한 줄, 캡션 스타일) |
| **레이아웃** | 안내 문구는 **섹션 블록 상단**(날짜/기간 입력 바로 아래, 카드 그리드 위)에 배치. 배경 `var(--mg-color-surface-main)` 또는 연한 포인트 배경(선택), 테두리 없거나 `1px var(--mg-color-border-main)`, `border-radius: 10px`, 패딩 `12px 16px`. 텍스트: `font-size: 12px`, `color: var(--mg-color-text-secondary)`. |
| **옵션 B** | "전부 0"을 "데이터 없음"과 동일한 빈 상태 블록으로 처리하고, 문구만 "해당 기간 거래가 없어 모든 금액이 0입니다"로 할 수 있음. 이 경우 §3.2와 동일한 블록 스타일 사용. |

**체크리스트**

- [ ] 전부 0인 경우 에러 메시지(빨간색·경고 아이콘)가 아닌 **정보성 안내**로 표시하는가?
- [ ] 안내 문구가 "거래가 없어 0으로 표시됩니다" 계열로 통일되어 있는가?
- [ ] 대차대조표·손익계산서 탭 모두 동일한 톤(캡션/인포 메시지)을 사용하는가?

### 3.4 API 에러 — 메시지·재시도 유도

| 항목 | 스펙 |
|------|------|
| **판단 조건** | API 실패(네트워크 오류, 4xx/5xx, `success === false` 등). 현재는 토스트만 띄우고 탭 내부에 에러 UI가 없음. |
| **문구** | **제목**: `데이터를 불러오지 못했습니다` **설명**: `일시적인 오류일 수 있습니다. 아래 버튼으로 다시 시도해 주세요.` (또는 서버 메시지가 있으면 그대로 표시 + 재시도 유도) |
| **재시도** | **버튼 문구**: `다시 불러오기` 또는 `재시도`. 주조 버튼: 배경 `var(--mg-color-primary-main)`, 텍스트 밝은색, padding 10–20px, height 40px, border-radius 10px. 클릭 시 동일 API 재호출. |
| **레이아웃** | **섹션 블록** 안에 에러 메시지 + 버튼. 배경 `var(--mg-color-surface-main)`, 테두리 `1px solid var(--mg-color-border-main)`, border-radius 16px, 패딩 24px. 세로 중앙 정렬. 에러 아이콘(선택): `var(--mg-color-error-500)` 또는 `var(--mg-error-500)` 계열, 크기 24–32px. |
| **토큰** | 제목: 16px, 600, `var(--mg-color-text-main)`. 설명: 14px, `var(--mg-color-text-secondary)`. 버튼: 기존 `mg-v2-button-primary` 또는 동일 스타일 토큰 |

**체크리스트**

- [ ] API 에러 시 탭 **내부**에 에러 블록(메시지 + 재시도 버튼)이 표시되는가?
- [ ] "다시 불러오기"(또는 "재시도") 버튼이 있고, 클릭 시 해당 탭의 fetch 함수가 다시 호출되는가?
- [ ] 에러 블록이 섹션 블록과 동일한 스타일(배경·테두리·radius)을 사용하는가?
- [ ] 토스트와 병행해도 되나, **탭 내부 에러 UI**가 반드시 있는가?

## 4. 대차대조표·손익계산서 탭 간 일관된 톤·레이아웃

| 항목 | 스펙 |
|------|------|
| **공통 구조** | 상단: 날짜/기간 입력(기존 유지) → 그 아래: (로딩 \| 빈 데이터 \| 에러 \| 정상 콘텐츠) 중 하나. **한 탭에서만** 다른 구조를 쓰지 않음. |
| **공통 문구 톤** | 로딩: "데이터를 불러오는 중…". 빈 데이터: "해당 기간 데이터가 없습니다". 0 안내: "…거래가 없어 0으로 표시됩니다". 에러: "데이터를 불러오지 못했습니다" + "다시 불러오기". |
| **공통 블록** | 로딩/빈 데이터/에러 모두 **같은 섹션 블록** 컨테이너 스타일: 배경 `var(--mg-color-surface-main)`, 테두리 `1px solid var(--mg-color-border-main)`, border-radius 16px, 패딩 24px. |
| **색상** | B0KlA·PENCIL_DESIGN_GUIDE 팔레트만 사용. 에러 강조 시 `var(--mg-error-500)` 또는 프로젝트 에러 토큰. |

**체크리스트**

- [ ] 두 탭 모두 "로딩 → (빈 데이터 | 전부 0 안내 | 에러 | 정상)" 플로우가 동일한가?
- [ ] 같은 상태에서 같은 문구 톤(친절·정보성)을 사용하는가?
- [ ] 섹션 블록(카드/패널) 스타일이 두 탭에서 동일한가?
