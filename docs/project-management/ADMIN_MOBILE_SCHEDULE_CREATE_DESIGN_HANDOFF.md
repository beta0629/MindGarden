# Admin Mobile — 일정 등록 Step 3 (시간 선택) UX 디자인 핸드오프

## 1. 개요
본 문서는 관리자 모바일 앱(Admin Mobile)의 **일정 등록 Step 3 (시간 선택)** 화면에 대한 UI/UX 및 레이아웃 스펙을 정의합니다. 
사용자 요구사항에 따라 **기존 스케줄을 시각적으로 확인**할 수 있도록 하고, **시간 직접 입력을 금지**하며, **빈 슬롯 탭 선택 방식**으로 UX를 개선합니다.

---

## 2. 화면 레이아웃 및 와이어프레임 (텍스트 계층)

전체 화면은 스크롤 가능한 메인 영역과 하단 고정 요약/액션 영역으로 구성됩니다.

### 2.1. 날짜 네비게이션 (Date Navigation)
- **구조**: `[◀] YYYY년 MM월 DD일 (요일) [▶]`
- **스타일**: 중앙 정렬, 좌우 화살표 아이콘 버튼. 텍스트는 16px, fontWeight 600 (`#2C2C2C`).

### 2.2. 「이날 기존 일정」 섹션 (Existing Schedules)
- **구조**: 
  - 섹션 타이틀: "이날 기존 일정" (좌측 악센트 바 적용)
  - 가로 스크롤 또는 세로 리스트(`FlashList`) 형태의 일정 카드 목록
- **컴포넌트 (`ScheduleCard` 재사용 제안)**:
  - **내용**: 시간 (HH:mm - HH:mm), 내담자명, 상태 Badge (`BOOKED`, `COMPLETED` 등)
  - **EmptyState**: 일정이 없을 경우 "등록된 일정이 없습니다." 텍스트 표시 (14px, `#5C6B61`, 중앙 정렬, 패딩 24px)

### 2.3. 상담 시간(분) 선택 (Duration Chips)
- **구조**: 
  - 섹션 타이틀: "상담 시간"
  - 가로 스크롤 칩 리스트: `[30분] [50분] [60분] [90분] [120분]`
- **스타일**: 선택된 칩은 Primary 색상 배경, 미선택 칩은 Outline 스타일.

### 2.4. 시작 시간 슬롯 그리드 (Start Time Slot Grid)
- **구조**:
  - 범례(Legend): `[■ 가용] [■ 예약됨] [■ 지난 시간]` (우측 상단 또는 그리드 상단 배치, 12px 텍스트)
  - 시간 슬롯 그리드 (3열 또는 4열 배치)
- **컴포넌트 (`TimeSlotChip` 재사용 제안)**:
  - **가용 (Available)**: 배경 `#FAF9F7`, 테두리 `#D4CFC8`, 텍스트 `#2C2C2C`
  - **예약됨 (Booked/Occupied)**: 배경 `#F5F3EF`, 텍스트 `#A39E98` (Disabled 처리)
  - **지난 시간 (Past)**: 배경 `#E8E6E1`, 텍스트 `#A39E98` (Disabled 처리)
  - **선택됨 (Selected)**: 배경 `#3D5246`, 텍스트 `#FFFFFF`, 테두리 `#3D5246`

### 2.5. 선택 요약 및 완료 카드 (Summary Card)
- **구조**: 하단 고정 (Bottom Fixed)
- **내용**: 
  - 선택된 날짜 및 시작 시간
  - **종료 시간 요약**: "종료 예정 시간: HH:mm" (시작 시간 + 상담 시간 자동 계산)
  - `[다음]` 또는 `[완료]` 버튼

## 3. 디자인 시스템 및 토큰 스펙 (B0KlA 기준)

본 화면은 `mindgarden-design-system.pen` 및 Expo Theme 기반의 토큰을 사용합니다.

### 3.1. 색상 (Colors)
- **Background**: `#FAF9F7` (화면 전체 배경)
- **Surface**: `#F5F3EF` (섹션 블록 및 카드 배경)
- **Primary**: `#3D5246` (선택된 칩, 주요 버튼, 좌측 악센트 바)
- **Text (Main)**: `#2C2C2C` (제목, 본문 텍스트)
- **Text (Sub)**: `#5C6B61` (설명, EmptyState, 범례 텍스트)
- **Border**: `#D4CFC8` (카드 테두리, 미선택 칩 테두리)

### 3.2. 타이포그래피 (Typography)
- **섹션 타이틀**: 16px, fontWeight 600, `#2C2C2C`
- **본문/카드 내용**: 14px, fontWeight 400, `#2C2C2C`
- **캡션/라벨**: 12px, fontWeight 400, `#5C6B61`

### 3.3. 레이아웃 및 간격 (Spacing)
- **화면 패딩**: 좌우 20px
- **섹션 간 간격**: 32px
- **섹션 내 요소 간격**: 16px
- **카드/블록 Corner Radius**: 12px ~ 16px

## 4. core-coder 구현 체크리스트

core-coder는 본 스펙을 바탕으로 다음 항목을 구현해야 합니다. (코드 작성 시 UI/UX 로직에 집중)

- [ ] **날짜 네비게이션 구현**: 좌우 화살표 탭 시 날짜 변경 로직 연동
- [ ] **기존 일정 FlashList 구현**: 선택된 날짜의 기존 스케줄 데이터를 불러와 `ScheduleCard` 리스트로 렌더링 (데이터 없을 시 EmptyState 표시)
- [ ] **상담 시간 Chip 구현**: 기본값(예: 50분) 설정 및 탭 시 상태 업데이트
- [ ] **시간 슬롯 그리드 구현**: 
  - 키보드를 통한 **시간 직접 입력 필드 제거**
  - `TimeSlotChip`을 활용하여 빈 슬롯만 탭 가능하도록 구현
  - 기존 일정과 겹치는 시간(점유 슬롯) 및 과거 시간은 `disabled` 처리
  - 범례(Legend) UI 추가
- [ ] **종료 시간 자동 계산 및 요약 카드 구현**: 시작 시간 탭 시, 선택된 '상담 시간'을 더해 "종료 HH:mm"을 하단 요약 카드에 노출
- [ ] **디자인 토큰 적용**: 하드코딩된 색상을 지양하고, Expo Theme에 정의된 B0KlA 팔레트 토큰(`theme.colors.*`)을 매핑하여 사용
- [ ] **컴포넌트 재사용**: 기존 `TimeSlotChip`, `ScheduleCard`, `Badge` 컴포넌트를 최대한 재사용 및 확장하여 구현
