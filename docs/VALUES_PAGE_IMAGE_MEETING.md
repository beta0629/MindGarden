# 가치관(`/values`) 페이지 이미지 — 서브에이전트 회의록 & 스펙

**문서 역할**: `/values`에 **획일적이지 않은(에디토리얼) 크기·배치**의 이미지를 넣기 위한 합의 스펙.  
**회의 가정**: 기획 · 디자이너 · 퍼블리셔 · 코더가 동일 목표로 정렬한 결과를 문서화함.  
**관련 코드**: `app/values/page.tsx`, `app/globals.css` (`.values-page`, `.value-section-card`).  
**다음 단계**: 본 문서 하단 **「오케스트레이터 핸드오프」** 를 입력으로 **오케스트레이션 → 기획**이 최종 AC·역할 분해를 확정한 뒤, 디자인 → 퍼블 → 코더 순 실행.

---

## 1. 회의 메타

| 항목 | 내용 |
|------|------|
| 목표 | 가치관 페이지에 AI 생성(예: Gemini 등) 또는 동일 톤의 정적 이미지를 두되, **디자인 요소에 맞춰 사이즈·비율·배치를 블록마다 다르게** 가져가 획일감을 줄인다. |
| 합의 | 모든 이미지를 **동일 썸네일 크기**로 맞추지 않는다. **레이아웃 역할**(히어로형 / 좌·우 스플릿 / 내러티브 간 여백용 악센트)을 먼저 정하고, 그에 맞춰 에셋 비율·export 크기를 정한다. |
| 자산 위치 | **`public/images/values/`** (영문 파일명·슬러그). 레포에 커밋하는 **정적 자산**을 1차 기준으로 한다. |
| 정책 정합 | 공개 UI **유니코드 이모지 금지** — 분위기는 **실제 이미지**로 처리 (`.cursor/skills/homepage-planning/SKILL.md`). 문의·예약 CTA는 **GNB/바텀시트 전용**; 이미지 위 **중복 CTA·전화번호 오버레이** 금지 (`.cursor/skills/gnb-inquiry-bottom-sheet/SKILL.md`). |

## 2. 역할별 발언 요약 (회의록)

- **기획**: 인트로(명제·두 문단)와 세 가치 카드가 **서로 다른 시선 리듬**을 갖도록, 이미지 **장수는 4~5장**을 상한으로 두고 우선순위만 고정한다. 카드마다 “반드시 동일 구조”를 강제하지 않는다.
- **디자이너**: 홈 `HomeSectionVisual`·`home-section-images` 계열과 **색·조도(따뜻한 실내·자연광·정원 은유)** 를 맞추되, **매 블록 동일 프레이밍 금지**. 인물이 나오면 식별 가능한 초상·특정 내담자 연상 지양; AI 사용 시 **상용 조건·고지** 여부는 운영과 별도 합의.
- **퍼블리셔**: `value-section-card`와 이미지 래퍼의 **반응형 규칙**(모바일에서는 스택, 데스크톱에서만 스플릿 등)을 CSS 변수·클래스로 정리해 **인라인 스타일 난립을 줄인다**. `object-fit`·최대 높이로 **CLS**를 막는다.
- **코더**: `next/image`의 **`sizes` / `priority`(LCP 후보 1곳만)** / `alt`를 스펙표와 1:1 대응. 구현 시 기존 페이지 구조를 최소 침해하되, 이미지 전용 서브컴포넌트 분리를 허용한다.

## 3. 블록별 레이아웃·사이즈 스펙 (합의안)

**원칙**: 아래 수치는 **디자인 시안·퍼블 구현 시 조정 가능**하나, **역할(히어로 vs 악센트 vs 와이드)** 은 유지한다.

| 블록 ID | 페이지 위치 | 레이아웃 역할 | 목표 비율(가이드) | 데스크톱 최대 너비(가이드) | 모바일 |
|---------|-------------|----------------|-------------------|---------------------------|--------|
| `V-hero` | h1·이탤릭 명제 **아래**, 두 문단 **위** | 인트로 **와이드 히어로** — 시선 고정 | **21:9 ~ 16:9** | 컨텐츠 영역 **100%** (max `min(100%, 900px)` 부근) | 전폭, 높이 상한(예: `min(40vh, 280px)`)으로 과다 점유 방지 |
| `V-pro` | **전문성** 카드 | **좌 텍스트 / 우 이미지** 스플릿 (RTL 아님) | **4:3** | 스플릿 시 이미지 컬럼 **38~42%** | 이미지 **카드 상단** 전폭, 비율 4:3 유지 |
| `V-human` | **인류애** 카드 | **작은 악센트** — 본문 흐름을 끊지 않음 | **1:1** 또는 **5:4** | 카드 내 **max 280~320px** 정렬(우측 플로트 또는 하단 중앙) | 중앙 **정사각** 작게, 본문 다음 삽입 |
| `V-growth` | **회복과 성장** 카드 | **풀블리드 느낌의 서브히어로** (카드 내 하단 밴드) | **2:1** 또는 **16:9** | 카드 패딩 안 **전폭** | 상단 또는 하단 **전폭 밴드**, 높이는 Pro보다 낮게 유지해 계층 표현 |
| `V-intro` (선택) | 이탤릭 명제 **옆** 또는 **배경 아주 약한 워터마크** | *선택 과제* — 구현 난이도 높으면 **생략** | — | — | 기본 **미적용** 권장 |

**파일명 예시 (슬러그)**  
`values-hero.webp`, `values-professionalism.webp`, `values-humanity.webp`, `values-growth.webp`  
(실제 확장자·압축은 디자인/퍼블이 용량 기준으로 확정.)

## 4. 접근성·카피

- **alt**: 제목(h3) 문자열을 **복붙하지 않고**, 장면·분위기를 **한 문장**으로 (예: “따뜻한 조도의 상담 공간과 책이 놓인 테이블”). 순수 장식이면 팀 합의 하에 `alt=""` 및 시맨틱 구조 검토.
- **LCP**: `V-hero` 한 곳에만 `priority` 검토; 나머지는 lazy.

---

## 5. 비범위 (본 라운드)

- DB·CMS 연동 이미지, 요청마다 API 생성.
- 가치 카드 **3개 모두 동일한 썸네일 그리드**로 맞추기.
- 이미지 내 **문의/예약 CTA** 또는 `#contact` 유도.

---

## 6. 구현 순서 (오케스트레이션 참고)

1. **기획**: 본 문서를 입력으로 AC 문장화·범위 확정(선택 블록 `V-intro` 포함 여부).  
2. **디자이너**: Pencil 등으로 블록별 와이어·크롭 가이드 확정.  
3. **퍼블리셔**: `globals.css` + 마크업 패턴(스플릿/악센트/밴드).  
4. **코더**: `app/values/page.tsx`에 이미지·에셋 경로 반영. 원격(Unsplash 등)은 `HomeSectionVisual`과 동일하게 **`<img>` 직접 로드** 권장(`/_next/image` 프록시 이슈 회피). 로컬 `public/images/values/`만 쓸 때는 `next/image` 검토 가능.  
5. **verify-changes** (선택): `/values` 반응형·LCP·alt 스팟 체크.

## 7. 오케스트레이터 핸드오프 (붙여넣기 블록)

아래 블록을 **오케스트레이션 요청** 또는 **기획 에이전트** 입력으로 사용한다.

```
[사용자 요청 요약]
가치관(/values) 페이지에 AI 생성 이미지를 public/images/values/ 등에 두고 사용한다. 이미지는 획일적이지 않게, 디자인 요소에 맞춰 블록마다 사이즈·비율·배치를 다르게 한다.

[오케스트 단계 요약]
docs/VALUES_PAGE_IMAGE_MEETING.md에 서브에이전트 회의 합의가 정리됨. 블록 V-hero / V-pro / V-human / V-growth (+선택 V-intro)별 레이아웃 역할·비율 가이드·파일 슬러그 예시·정책(GNB/이모지/CTA)이 명시됨.

[기획에게 요청할 일]
1) 본 문서의 선택 과제(V-intro) 포함 여부와 최종 AC를 한 페이지로 확정한다.
2) 디자이너·퍼블·코더에게 넘길 역할별 핸드오프 프롬프트를 쪼갠다(스킬: homepage-planning).
3) AI 이미지 상용·고지 필요 시 운영 정책 한 줄을 추가한다.

[선택 과제]
홈 home-section-images와 톤 보드 1장을 디자이너가 먼저 고정하면 에셋 생성 일관성이 올라간다.
```
