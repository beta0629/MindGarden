# MindGarden Design & UX Roadmap

> **작성일:** 2025-11-12  
> **목적:** MindGarden 플랫폼(소상공인 통합) 전반의 디자인/UX 전략과 산출물, 일정, 책임 체계를 문서화한다.

## 1. 디자인 비전 & 원칙

- **일관성:** 소비자·관리자·HQ 포털에서 동일한 사용자 경험, 컴포넌트, 언어 체계를 유지한다.
- **유연성:** 업종(학원, 미용, 서비스 등)마다 필요한 UI 변형은 “테마/설정” 형태로 제공한다.
- **접근성:** WCAG 2.1 AA 기준 준수, 키보드 네비게이션, 다국어/다문화 고려(UI 텍스트/아이콘).
- **데이터 기반:** UX 개선 사항은 로그·A/B 테스트·사용자 인터뷰 결과에 근거해 우선순위화한다.

## 2. 디자인 시스템 구성 요소

| 층위 | 설명 | 산출물 |
| --- | --- | --- |
| **기초(Foundations)** | 컬러, 타이포그래피, 아이콘, 간격, 그림자, 그리드 등 | 토큰 정의, Figma 라이브러리, CSS 변수 |
| **컴포넌트(Component)** | 버튼, 폼, 카드, 모달, 테이블, 그래프 등 재사용 가능한 UI 블록 | Storybook, Figma 컴포넌트, React 컴포넌트 |
| **패턴(Pattern)** | 예약/결제/정산/출결 등 80/20 플로우 패턴 | 사용자 흐름 다이어그램, 패턴 가이드 |
| **페이지/레이아웃** | 소비자/관리자/HQ 화면 템플릿 | Figma 화면 설계, 반응형 레이아웃 가이드 |
| **콘텐츠 가이드** | UI 텍스트, 에러 메시지, 알림, 메일 템플릿 | UX Writing 가이드, 다국어 전략 |

## 3. 산출물 및 일정 (2025 Q4 ~ 2026 Q4)

| 분기 | 산출물 | 책임 | 비고 |
| --- | --- | --- | --- |

| **2026 Q3** | 미용/서비스 업종 5대 흐름 UI 템플릿, 업종 테마(색/아이콘) 추가 | 디자인팀 + 업종별 PM | 업종 확장 T3 |
| **2026 Q4** | 마켓플레이스/HQ 포털 고도화, 파트너 온보딩 UI, 다국어/다문화 가이드 | 디자인팀 + 파트너팀 | SaaS 상용화 준비 |

## 4. 협업 프로세스

1. **기획 단계**  
   - PM/PO가 요구사항 → 디자인팀과 IA(Information Architecture) 정리 → 주요 사용자/시나리오 정의  
2. **디자인 단계**  
   - Figma 와이어프레임 → UI 디자인 → 프로토타입 및 사용자 테스트  
   - 디자인 리뷰 (PM/개발/QA 동시 참여)  
3. **개발 인계**  
   - Storybook / 프런트엔드 컴포넌트 동기화  
   - 디자인 QA 체크리스트 제공 → 개발 QA → 사용자 UAT  
4. **검증 및 배포 후**  
   - Analytics/Logging 기반 개선 항목 기록  
   - 고객 VOC, 온보딩 TF 회의 결과 반영  


## 5. UX 리서치 전략

| 활동 | 대상 | 주기 | 목적 |
| --- | --- | --- | --- |
| 정성 인터뷰 | 테넌트(학원 원장, 지점 스태프) 3~5명 | 월 1회 | 업무 흐름 이해, 페인 포인트 확인 |
| 사용성 테스트 | 신규 기능(출결 앱, 예약 UX) | 주요 릴리스마다 | UI/UX 개선, 오류 방지 |
| 설문/피드백 | 소비자(학부모/회원) | 분기 | 만족도, 재등록율 분석 |
| 데이터 분석 | 클릭 로그, Heatmap | 지속 | A/B 테스트, UX 개선 우선순위 |

## 6. 접근성 & 다국어 전략

- 키보드 및 스크린 리더 지원, 콘트라스트 기준 준수
- 한글/영문 텍스트 길이 고려, 번역 관리 시스템 도입 (Phase 2 이후)
- 장애물 없는 알림 UX (색상 외 아이콘/텍스트 동시 제공)

## 7. 협업 도구

- **디자인:** Figma, FigJam, Zeplin (필요 시)  
- **컴포넌트:** Storybook, Chromatic, Design Token CLI  
- **커뮤니케이션:** Notion, Slack, Linear/Jira, Miro  
- **리서치/분석:** Looker, GA4, Hotjar, Firebase Analytics

## 8. 차후 과제

- 디자인 시스템 거버넌스 운영 (버전 관리, 리뷰 프로세스)
- 업종별 테마/브랜드 템플릿 개발
- AI 기반 UI 생성/추천(예: 텍스트 자동 작성, KPI 대시보드 자동 디자인)
- PWA/모바일 네이티브 디자인 가이드 표준화
