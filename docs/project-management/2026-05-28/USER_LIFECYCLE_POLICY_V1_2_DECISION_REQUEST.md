## §1. v1.1 → v1.2 변경 사항 요약 (5종)

1. **Q9 근거 재기술**: 휴면 정책(§39의6)의 근거를 "법령 강제"에서 "사업자 자율 정책 + 동의서 명시"로 변경 (세법·전금법 5년 분리보관 의무는 별도 유지).
2. **Q10 비즈니스 정체성 분기**: 상담 기록 보존 기간을 비의료(3년) / 의료기관 연계(10년) / 하이브리드로 분기하여 사용자 결재 변수로 신설.
3. **Q11 AI PII 스크러빙 로드맵 명시**: 정규식(즉시) → KcBERT(6개월 내) → ko-pii-public-v1(12개월 내) 3단계 로드맵 명시 및 외부 API 도입 검토 조건 추가.
4. **보존·파기 기간 표 갱신**: 데이터 유형별 법령 근거 및 보존 기간을 명확히 한 표(§6.3) 신설.
5. **법령 매트릭스 신설**: 개인정보보호법, 의료법, 전자상거래법, 국세기본법, 전자금융거래법, 상법, 학회 윤리강령을 종합한 법령 매트릭스(§11) 신설.

## §2. 사용자 결재 필요 변수 (13 옵션)

정책서 §10에 명시된 사용자 결재 변수입니다.

### §10.1 비즈니스 정체성 (Q10 핵심 변수)
- **A. 비의료 상담 플랫폼 (default)**: 3년 보존 (학회 윤리강령 + 동의서 명시)
- **B. 의료기관 연계 (의원 제휴)**: 10년 보존 (의료법 §22)
- **C. 하이브리드**: 상담 유형별 분기 처리
- **D. 판단 보류**: 안전하게 10년 유지

### §10.2 휴면 정책 일수 (Q9 — 자율)
- **A. 1년 DORMANT + 4년 anonymize (v1.1 유지, default)**
- **B. 6개월 DORMANT (적극)**
- **C. 2년 DORMANT (보수)**

### §10.3 AI PII 스크러빙 도입 시점 (Q11)
- **A. 1단계 정규식만 + 향후 단계적 (default)**
- **B. 1+2단계 일괄 도입 (KcBERT 즉시)**
- **C. 외부 GPT-4o (mcp-pii-tools) PIPA §17 검토 후 도입**

## §3. 권장안

다음의 Default 셋을 권장합니다.
- **비즈니스 정체성**: A. 비의료 상담 플랫폼 (3년 보존)
- **휴면 정책 일수**: A. 1년 DORMANT + 4년 anonymize
- **AI PII 스크러빙**: A. 1단계 정규식만 + 향후 단계적 도입

**사유**: 현재 MindGarden의 주요 서비스 모델에 가장 적합하며, 법적 리스크를 최소화하면서도 즉각적인 시스템 반영이 가능합니다.

## §4. 권장안 채택 시 후속 작업 일정

권장안 채택 시 `core-coder`에게 Phase 5 위임 분량으로 다음 작업이 예상됩니다. (PR 1건으로 처리 가능)
- **마이그레이션**: 보존 기간 변경에 따른 마이그레이션 1~2건
- **Service 로직 분기**: 상담 기록 보존 기간(3년/10년)에 따른 분기 처리 구현
- **Cron Job**: 만료 데이터 삭제를 위한 cron 1건 추가
- **테스트**: 관련 단위 테스트 작성

---

## §5. 권장안과 다른 선택 시 영향 분석

- **10년 유지 (B 또는 D) 선택 시**: 
  - 마이그레이션 및 법령 인용 차이가 발생합니다.
  - 데이터 보존 비용이 장기적으로 증가할 수 있습니다.
  - 의료법 §22를 준수하는 강력한 보존 정책이 적용됩니다.
- **하이브리드 (C) 선택 시**:
  - 상담 유형별로 보존 기간을 다르게 적용해야 하므로 로직 복잡도가 증가합니다.
  - 추가적인 설계 및 구현 시간이 필요합니다.
- **AI 스크러빙 즉시 도입 (B 또는 C) 선택 시**:
  - 모델 통합 및 검증을 위한 추가 공수가 발생합니다.
  - 외부 API 사용 시 개인정보 국외 이전 및 PIPA §17 검토가 선행되어야 합니다.
