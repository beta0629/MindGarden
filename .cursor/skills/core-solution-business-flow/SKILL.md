# 핵심 비즈니스 흐름 및 등록 UI 자동화 규칙

이 스킬은 MindGarden의 핵심 비즈니스 흐름을 구현하거나, 관리자 대시보드 및 입력 폼을 구성할 때 반드시 적용해야 하는 지침입니다.

## 1. 5단계 핵심 비즈니스 파이프라인

프론트엔드 및 백엔드 개발 시 다음 5단계를 명확히 구분하고 연동해야 합니다.

1. **내담자/상담사 매칭 (관리자 전담)**: 관리자가 직접 내담자와 상담사를 선택(드롭다운)하여 수동 배정 및 매칭 확정.
2. **입금 확인 (ERP 연동)**: 결제 완료 후 ERP 시스템과 연동되어 입금 확인 대기/완료 상태 동기화.
3. **회기(세션) 권한 부여**: 입금 확인이 완료되면 해당 내담자-상담사 매칭에 적용된 회기수(세션)만큼 스케줄 권한 부여.
4. **스케줄 등록 (관리자 전담)**: 부여된 세션 내에서 관리자가 사전 협의된 일정을 스케줄에 등록. (상담사는 정규직이 아니므로 스케줄을 직접 관리하지 않음)
5. **자동 회기차감 / 회계처리 (ERP)**: 스케줄에 등록된 일정이 지나거나(배치), 상담사가 상담 일지를 작성하면 중앙 회기관리 시스템에서 회기수가 자동 차감되고 ERP에서 자동 회계처리.

## 2. 사용자 입력 최소화 & 자동화 규칙 (UI/UX)

서브에이전트(`core-coder`, `core-designer`)는 화면 개발/디자인 시 다음을 준수해야 합니다.

- **필수 입력값 최소화**: 사용자(특히 관리자)가 직접 타이핑해야 하는 텍스트 입력을 최소한으로 줄입니다.
- **선택형(드롭다운) 우선**: 카테고리, 상태, 소속, 상담사 선택, 내담자 선택 등은 반드시 `CustomSelect`나 드롭다운(Dropdown) 형식으로 제공합니다.
- **자동화 및 기본값 채우기**: 시스템적으로 추론 가능한 데이터(날짜, 기본 상태, 생성자 등)는 자동화하여 사용자에게 입력을 요구하지 않습니다.
- **중복 클릭 방지 (안전성)**: 폼 제출 및 등록 시 `MGButton`의 로딩/비활성화 상태를 필수 적용하여 중복 데이터 생성을 막습니다 (`preventDoubleClick=true`).

## 3. 서브에이전트 역할 지침

- **`core-coder`**: 이 파이프라인의 5단계 로직(매칭 생성 API, ERP 연동 로직, 세션 차감 배치 등)을 구현할 때 상태(Status) 값과 연관 Entity를 정확히 매핑하세요. 입력 폼 구현 시 `Input` 대신 `Select` 컴포넌트 사용을 우선 검토하세요.
- **`core-designer`**: 등록 모달이나 매칭 대기열 퍼블리싱 시, 타이포그래피 규칙과 둥근 모서리, 소프트 섀도우를 적용하고 시각적으로 번잡한 텍스트 필드보다는 깔끔한 선택형 UI(드롭다운)를 적용하세요.
- **`core-tester`**: 5단계 파이프라인의 각 상태 전이(State Transition)와 입력 자동화 로직(예: 필수 필드 누락 시 에러, 드롭다운 선택 후 제출)에 대한 E2E 및 단위 테스트를 작성하세요.
