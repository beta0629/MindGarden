# 업종별 스토리보드

**작성일**: 2025-11-20  
**버전**: 1.0.0  
**상태**: 완료

---

## 📋 개요

업종별 주요 사용자 시나리오와 화면 흐름을 정리한 문서입니다. 각 업종은 다른 역할 구조와 화면 흐름을 가지지만, 공통적인 패턴을 공유합니다.

---

## 🎓 ACADEMY (학원) 스토리보드

### 학생 (STUDENT) 스토리보드

#### 시나리오 1: 강좌 신청

**화면 흐름:**
```
1. 로그인 화면
   ↓
2. 학생 대시보드
   - 내 강좌 요약
   - 빠른 신청 버튼
   - 최근 공지사항
   ↓
3. 강좌 목록
   - 강좌 카테고리
   - 강좌 검색
   - 강좌 필터
   ↓
4. 강좌 상세
   - 강좌 정보
   - 시간표
   - 선생님 정보
   - 수강료 정보
   ↓
5. 수강 신청
   - 수강 정보 확인
   - 결제 정보 입력
   ↓
6. 결제
   - 결제 수단 선택
   - 결제 완료
   ↓
7. 수강 신청 완료
   - 수강 확인 메시지
   - 시간표 표시
```

---

#### 시나리오 2: 출석 확인 및 성적 조회

**화면 흐름:**
```
1. 학생 대시보드
   ↓
2. 내 강좌 메뉴
   ↓
3. 강좌 목록
   - 수강 중인 강좌
   - 완료된 강좌
   ↓
4. 강좌 상세
   - 출석 현황
   - 성적 조회
   - 과제 제출
   ↓
5. 출석 상세
   - 출석 일자별 현황
   - 출석률 통계
   ↓
6. 성적 상세
   - 시험 성적
   - 과제 성적
   - 총점
```

### 선생님 (TEACHER) 스토리보드

#### 시나리오 1: 반 관리 및 출석 체크

**화면 흐름:**
```
1. 로그인 화면
   ↓
2. 선생님 대시보드
   - 담당 반 요약
   - 오늘의 수업
   - 출석 체크 알림
   ↓
3. 반 관리 메뉴
   ↓
4. 반 목록
   - 담당 반 목록
   ↓
5. 반 상세
   - 학생 목록
   - 시간표
   - 출석 현황
   ↓
6. 출석 체크
   - 수업 일자 선택
   - 학생별 출석 체크
   - 출석 저장
   ↓
7. 출석 현황 업데이트
```

---

#### 시나리오 2: 성적 입력

**화면 흐름:**
```
1. 선생님 대시보드
   ↓
2. 성적 관리 메뉴
   ↓
3. 성적 입력
   - 반 선택
   - 시험/과제 선택
   - 학생별 성적 입력
   ↓
4. 성적 저장
   ↓
5. 성적 현황 확인
   - 평균, 최고점, 최저점
   - 성적 분포
```

## 🔄 공통 화면 전환 패턴

### 범용 네비게이션 구조

```
대시보드 (홈)
  ├─ 서비스/강좌 목록
  ├─ 예약/수강 내역
  ├─ 일정 관리
  ├─ 사용자 관리 (관리자)
  ├─ 통계 (관리자)
  ├─ 설정
  └─ 프로필
```

### 모달 사용 패턴

- **사용자 추가/수정**: 모달
- **역할 할당**: 모달
- **예약/신청 정보 입력**: 모달
- **설정 변경**: 모달
- **확인 다이얼로그**: 모달

### 페이지 전환 패턴

- **목록 → 상세**: 페이지 전환
- **상세 → 수정**: 페이지 전환
- **통계 조회**: 페이지 전환
- **설정 관리**: 페이지 전환

## 🎨 UI 컴포넌트 패턴

### 공통 컴포넌트

- **버튼**: Primary, Secondary, Danger, Link
- **입력 필드**: Text, Email, Phone, Number, Date, Time
- **선택 필드**: Select, Radio, Checkbox, Toggle
- **테이블**: 정렬, 필터링, 페이징, 액션 버튼
- **모달**: 확인, 입력, 정보 표시
- **알림**: 성공, 경고, 오류, 정보

### 대시보드 위젯

- **통계 카드**: 숫자, 아이콘, 변화율, 차트
- **목록 위젯**: 최근 항목, 빠른 액션
- **캘린더 위젯**: 일정 표시, 날짜 선택
- **차트 위젯**: 막대 그래프, 선 그래프, 파이 차트

## ⚠️ 중요 사항

1. **업종별 차이**
   - 각 업종은 다른 역할 구조와 화면 흐름을 가짐
   - 공통 패턴을 기반으로 업종별 커스터마이징

2. **동적 역할 시스템**
   - 역할은 동적으로 생성되므로 화면도 동적으로 구성
   - 테넌트 관리자가 역할별 화면 커스터마이징 가능

3. **MindGarden은 예시**
   - MindGarden의 화면 흐름은 CONSULTATION 업종의 예시
   - 다른 상담소는 다른 화면 구조를 가질 수 있음
