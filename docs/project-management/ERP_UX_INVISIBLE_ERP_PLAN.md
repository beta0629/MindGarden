# ERP 인지 최소화 UX 기획 — "이게 ERP인 줄 모를 정도로"

**작성일**: 2026-02-12  
**목표**: 사용자 입장에서 최대한 편하게, **ERP라는 시스템을 의식하지 않고** 상담소 운영·수입·지출·정산을 쓰는 경험 제공.  
**원칙**: 업무/결과 중심 언어 사용, 전문 용어(분개·원장·전기 등) 최소화.

---

## 1. 핵심 원칙

| 원칙 | 설명 | 예시 |
|------|------|------|
| **업무 중심** | 사용자가 하는 일(입금 확인, 지출 등록, 정산 보기)으로 표현 | "분개 관리" ❌ → "거래 정리" ✅ |
| **결과 중심** | 화면이 보여주는 결과(수입·지출 요약, 손익 현황)로 표현 | "원장 조회" ❌ → "계정별 내역" ✅ |
| **ERP 노출 최소화** | 메뉴·타이틀에서 "ERP" 단어 제거, 필요 시 "운영·재무" 수준 | "ERP 대시보드" ❌ → "운영 현황" ✅ |
| **전문 용어 완화** | 회계 용어는 유지하되 사용자 친화 라벨 병기 또는 대체 | "분개" → "거래 정리", "전기" → "반영 완료" |

## 2. 메뉴·화면 라벨 매핑 (적용 기준)

### 2.1 LNB(사이드 메뉴)

| 현재 라벨 | 변경 라벨 | 비고 |
|-----------|-----------|------|
| ERP 관리 | **운영·재무** | 상위 메뉴, ERP 미노출 |
| ERP 대시보드 | **운영 현황** | 수입·지출·구매 등 한눈에 |
| 구매 관리 | 구매 관리 | 유지 |
| 재무 관리 | **수입·지출** | 거래·달력·요약 |
| 예산 관리 | 예산 관리 | 유지 |
| 세무 관리 | 세무 관리 | 유지 |

### 2.2 ERP 대시보드(운영 현황) 페이지

| 현재 | 변경 |
|------|------|
| 페이지 타이틀: ERP 대시보드 | **운영 현황** |
| 부제: 통합 자원 관리 및 회계 시스템 | **수입·지출·구매를 한눈에** |
| 통합 회계 시스템 (카드) | **수입·지출 한눈에** |
| 분개 관리 (카드) | **거래 정리** |

| 현재 | 변경 |

| 페이지 타이틀: 통합 회계 관리 시스템 | **수입·지출 관리** |

| 탭: 손익계산서 | **손익 현황** (또는 손익계산서 유지) |
| 탭: 현금흐름표 | **현금 흐름** |
| 탭: 정산 관리 | **정산** |

| 화면 | 현재 | 변경 |
|------|------|------|

## 5. 제외·유지

- **URL 경로**: `/erp/*`, `/admin/erp/*` 변경 없음 (북마크·권한 매핑 유지).
- **권한 코드**: `ERP_ACCESS`, `INTEGRATED_FINANCE_VIEW` 등 내부 코드 유지, 화면에 노출되는 **라벨만** 변경.
- **대차대조표·손익계산서**: 법·회계에서 쓰는 정식 명칭이므로, 필요 시 부제로 "자산·부채 현황", "손익 현황"만 병기 가능.

---

## 6. 완료 기준

- [ ] 메뉴에서 "ERP" 문구가 보이지 않음 (운영·재무, 운영 현황 등으로 대체).
- [ ] 통합 재무 첫 화면이 "수입·지출 관리"로 인식됨.
- [ ] "분개", "원장", "전기"가 일반 사용자 노출 문구에서 최소화됨.
- [ ] 입금 확인·거래 등록 후 메시지가 업무 결과 중심("반영되었습니다")으로 통일됨.
