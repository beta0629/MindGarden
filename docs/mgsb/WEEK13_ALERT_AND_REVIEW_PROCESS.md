```markdown
## 변경 사항
- [ ] 기능 추가
- [ ] 버그 수정
- [ ] 리팩토링
- [ ] 문서화

## 변경 내용 설명
<!-- 변경 사항을 자세히 설명해주세요 -->

## 테스트
- [ ] 단위 테스트 작성/수정
- [ ] 통합 테스트 작성/수정
- [ ] 수동 테스트 완료

## 코드 품질 검증
- [ ] ESLint/Checkstyle 통과
- [ ] 하드코딩 검사 통과
- [ ] 테스트 커버리지 유지

## 체크리스트
- [ ] 상수 사용 확인 (하드코딩 없음)
- [ ] 에러 처리 확인
- [ ] 보안 검토 완료
- [ ] 성능 영향 검토 완료
```

### 코드 리뷰 체크리스트

#### 필수 체크 항목

1. **하드코딩 확인**
   - [ ] 문자열 하드코딩 없음
   - [ ] 숫자 하드코딩 없음 (매직 넘버)
   - [ ] URL/경로 하드코딩 없음
   - [ ] 모든 값이 상수 파일 또는 환경 변수에서 관리됨

2. **코드 품질**
   - [ ] ESLint/Checkstyle 규칙 준수
   - [ ] 코드 복잡도 적절함
   - [ ] 중복 코드 없음
   - [ ] 적절한 네이밍

3. **보안**
   - [ ] SQL Injection 방지
   - [ ] XSS 방지
   - [ ] 인증/권한 검증
   - [ ] 민감 정보 암호화

4. **성능**
   - [ ] 불필요한 쿼리 없음
   - [ ] 적절한 인덱스 사용
   - [ ] 캐싱 활용 (필요 시)
   - [ ] N+1 문제 없음

5. **테스트**
   - [ ] 단위 테스트 작성
   - [ ] 통합 테스트 작성 (필요 시)
   - [ ] 테스트 커버리지 유지

### 자동 검증 통과 후 리뷰 요청 프로세스

```
1. 코드 작성
   ↓
2. 로컬 테스트 실행
   - mvn test
   - npm test
   ↓
3. 하드코딩 검사
   - node scripts/check-hardcoding-enhanced.js
   ↓
4. Git Commit
   ↓
5. Pull Request 생성
   ↓
6. CI/CD 자동 검증
   - 문법 검사 (ESLint, Checkstyle)
   - 테스트 실행
   - 하드코딩 검사
   ↓
7. 자동 검증 통과 확인
   ↓
8. 코드 리뷰 요청
   - 리뷰어 지정
   - 체크리스트 확인
   ↓
9. 코드 리뷰 진행
   - 리뷰어 피드백
   - 수정 사항 반영
   ↓
10. 리뷰 승인
   ↓
11. Merge
```

### 코드 리뷰 가이드라인

#### 리뷰어 관점

1. **기능 정확성**
   - 요구사항 충족 여부
   - 엣지 케이스 처리
   - 에러 처리

2. **코드 품질**
   - 가독성
   - 유지보수성
   - 확장성

3. **아키텍처**
   - 설계 패턴 준수
   - 의존성 관리
   - 모듈화

#### 작성자 관점

1. **명확한 설명**
   - PR 설명 작성
   - 변경 이유 명시
   - 테스트 방법 안내

2. **피드백 수용**
   - 건설적 피드백 수용
   - 질문에 명확히 답변
   - 수정 사항 빠르게 반영
