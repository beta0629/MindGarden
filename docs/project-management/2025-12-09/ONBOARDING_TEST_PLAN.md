**작성일**: 2025-12-09

## 테스트 범위

### 1. 온보딩 요청 생성 (`/onboarding/request`)

#### 필수 입력 항목
- [ ] 회사명 (tenantName) - 필수
- [ ] 업종 카테고리 선택
- [ ] 세부 업종 선택 (businessType) - 필수
- [ ] 연락처 전화번호 (contactPhone)
- [ ] 위험도 수준 (riskLevel) - 선택

#### 기능 테스트
- [ ] 업종 카테고리 로드 확인
- [ ] 카테고리 선택 시 세부 업종 목록 업데이트
- [ ] 필수 항목 미입력 시 제출 방지
- [ ] 폼 제출 성공 시 `/onboarding/status`로 이동
- [ ] 에러 발생 시 에러 메시지 표시

## 테스트 시나리오

### 시나리오 1: 정상 플로우
1. `/onboarding/request` 접속
2. 회사명 입력: "테스트 회사"
3. 업종 카테고리 선택: "상담"
4. 세부 업종 선택
5. 연락처 입력
6. 위험도 수준 선택
7. 제출 버튼 클릭
8. 성공 메시지 확인
9. `/onboarding/status`로 자동 이동
10. 생성된 요청 확인

### 시나리오 2: 필수 항목 누락
1. `/onboarding/request` 접속
2. 필수 항목 일부만 입력
3. 제출 버튼 클릭
4. 필수 항목 에러 메시지 확인
5. 제출 실패 확인

### 시나리오 3: 상태별 필터링
1. `/onboarding/status` 접속
2. 상태 필터 선택 (예: PENDING)
3. 해당 상태의 요청만 표시 확인
4. 다른 상태 필터 선택
5. 목록 업데이트 확인

### 시나리오 4: 상세 정보 확인
1. `/onboarding/status` 접속
2. 요청 목록에서 특정 요청의 "상세" 버튼 클릭
3. 모달에 상세 정보 표시 확인
4. 모든 필드 정보 확인
5. 모달 닫기 확인

### 시나리오 5: 에러 처리
1. 네트워크 연결 끊기
2. 온보딩 요청 제출 시도
3. 네트워크 오류 메시지 확인
4. 네트워크 복구 후 재시도
5. 정상 동작 확인
