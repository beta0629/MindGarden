# MindGarden Platform Roadmap

> **기준일:** 2025-11-12  
> **범위:** MindGarden 2.0 (소상공인 통합 플랫폼) 구축을 위한 18개월 로드맵  
> **단계 정의:** `T` = 출시 시점 기준 분기(Quarter)

## 1. 로드맵 개요

| 단계 | 기간 | 핵심 목표 | 주요 성과 |
| --- | --- | --- | --- |
| **T0** | 2025 Q4 | 학원 Phase 1 MVP 안정화 | 멀티테넌시, 학원 80/20 흐름, 정산 자동화 |
| **T1** | 2026 Q1 | 운영 자동화 & 품질 강화 | 무중단 배포, 모니터링, 테스트 자동화 |
| **T1.5** | 2026 Q1~Q2 | **외부 온보딩 + 자동결제 상용화** | 공개 신청 페이지, 요금제 선택, PG 정기결제 |
| **T2** | 2026 Q2 | 학원 Phase 2 + AI 기반 운영 인사이트 | 출결 모바일 앱, AI 이탈 예측, HQ 정책 엔진 |
| **T3** | 2026 Q3 | 업종 확장 (미용/서비스) | 공통 5대 흐름 패턴화, POS/ERP 확장 |
| **T3.5** | 2026 Q3~Q4 | **컴포넌트 기반 모듈화 & 애드온 과금** | 컴포넌트 카탈로그, 테넌트별 선택 활성화, 컴포넌트 단위 과금 |
| **T4** | 2026 Q4 | 플랫폼 고도화 & SaaS 상용화 | 파트너 API, 마켓플레이스, Data-as-a-Service |

## 2. 상세 로드맵

### T0 (2025 Q4) — Phase 1 안정화
- [ ] 멀티테넌시 도입 (TenantContext, Repository, 마이그레이션)
- [ ] 학원 80/20 흐름 표준 구현
  - 상담 → 반 배정 → 출결 → 청구 → 정산
- [ ] 정산/통계 PL/SQL 패키지 및 배치 스케줄
- [ ] 공통 코드 한글 필드 적용 + 코드 관리 UI
- [ ] QA 전략 (E2E + 성능 테스트) 및 PoC 10곳 실증

### T1 (2026 Q1) — 운영 자동화 & 품질
- [ ] 무중단 배포 도입 (Blue/Green, Canary) + 배포 헬스체크
- [ ] 세션/토큰 모니터링 대시보드, 이상 탐지 알림
- [ ] PL/SQL 배치 KPI 모니터링 및 장애 알림
- [ ] Spin-up 환경 자동화 테스트 (E2E/성능/회귀)
- [ ] API 문서(OpenAPI) 자동 배포, 형상 관리

### T1.5 (2026 Q1~Q2) — 외부 온보딩 + 자동결제
- [ ] 외부 공개 온보딩 신청 페이지 구축 (정보 입력 + 요금제 선택)
- [ ] 결제 게이트웨이(PG) 연동 및 정기결제(Subscription) 도메인 설계
- [ ] 신청 → 결제 → 테넌트 생성 자동화 트랜잭션 구현
- [ ] 구독 상태/결제 실패 알림, 결제 내역 조회 기능
- [ ] 운영 포털과 결제/승인 흐름 통합 (심사 후 활성화)

### T2 (2026 Q2) — 학원 Phase 2 + AI 인사이트
- [ ] 출결/스케줄 모바일 앱 UX 개선, 푸시/오프라인 모드 지원
- [ ] AI 이탈 예측/추천 MVP (학원 데이터 기반)
- [ ] HQ 정책 엔진 고도화 (로열티, 프로모션, 공지)
- [ ] 소비자 포털 예약/결제 UX AB 테스트, 접근성 개선
- [ ] 운영 로그 요약/알림: GPT 기반 로그 요약→Slack/대시보드 통합  
- [ ] 내부 운영 지원 챗봇 PoC: 테넌트/운영 관련 Q&A
- [ ] **커뮤니티 DM 기능 구현** (WebSocket 기반 실시간 메시징) ⭐ **우선순위: 높음**
  - [ ] DM 대화방 및 메시지 기능 (Phase 1: WebSocket 직접 구현)
  - [ ] 실시간 알림 및 읽음 상태 표시
  - [ ] 입력 중 표시(Typing Indicator) - 인스타그램/카카오톡 스타일
  - [ ] 사용자 차단 기능
  - [ ] 서버 과부하 모니터링 및 최적화
  - **비즈니스 가치**: 시작 단계 기능이라도 없는 것보다는 있는 것이 학원 선택 시 차별화 포인트
- [ ] **학원 Quick Win 기능 구현** ⭐ **우선순위: 높음** (간단하면서도 비즈니스 가치 높음)
  - [ ] **공지사항 게시판** (2-3일, 우선순위: 높음) - 학원-학부모 소통 채널 강화
  - [ ] **출석 체크 QR 코드** (3-4일, 우선순위: 높음) - 출석 체크 자동화
  - [ ] **학부모 일일 리포트** (2-3일, 우선순위: 높음) - 매일 저녁 자동 발송
  - [ ] **수강권 잔여 횟수 표시** (1-2일, 우선순위: 중간) - 투명한 수강권 관리
  - [ ] **생일 축하 알림** (1일, 우선순위: 낮음) - 인간적인 서비스
  - [ ] **과제/숙제 관리** (3-4일, 우선순위: 중간) - 과제 관리 자동화
  - [ ] **간단한 설문조사** (2-3일, 우선순위: 중간) - 피드백 수집

### T3 (2026 Q3) — 업종 확장 (미용/서비스)
- [ ] 미용/서비스 업종 5대 흐름 정의 및 모듈 설계
- [ ] POS/현장 결제 데이터와 ERP 정산 통합
- [ ] 업종별 공통 컴포넌트 및 디자인 시스템 확장
- [ ] 프랜차이즈 본사 정책(로열티, 이벤트) 자동 적용 기능
- [ ] 고객 FAQ 챗봇 (예약/결제/정산 FAQ, 역할별 응답)  
- [ ] 웰니스/콘텐츠 AI 고도화: 업종별 맞춤 컨텐츠 생성

### T3.5 (2026 Q3~Q4) — 컴포넌트 기반 모듈화 & 애드온 과금 시스템
- [ ] 컴포넌트 카탈로그 DB 설계 (`component_catalog`, `tenant_component`, `component_feature`)
- [ ] 섹션별 컴포넌트 메타데이터 등록 (상담, 예약, 결제, 정산, 알림, CRM 등)
- [ ] 테넌트 온보딩 시 컴포넌트 선택 UI 구현 (기본 세팅 + 선택 애드온)
- [ ] 컴포넌트 활성화/비활성화 API 및 Feature Flag 연동
- [ ] 컴포넌트별 사용량 추적 및 과금 계산 로직 구현
- [ ] 애드온 과금 시스템 통합 (컴포넌트 단위 월정액/Usage 기반)
- [ ] 컴포넌트 의존성 관리 (필수 컴포넌트, 선택 컴포넌트, 충돌 방지)
- [ ] 운영 포털 컴포넌트 관리 UI (카탈로그 관리, 테넌트별 활성화 현황)
- [ ] 테넌트 대시보드에 "내 컴포넌트" 관리 페이지 추가

### T4 (2026 Q4) — 플랫폼 고도화 & 상용화
- [ ] 파트너 API/마켓플레이스 공개, OAuth Client 발급 로직
- [ ] Data-as-a-Service (정산/운영 데이터 외부 제공)
- [ ] SaaS 요금제/청구(Usage 기반) 적용
- [ ] SSO/OIDC Provider 모드로 외부 HQ 연동
- [ ] 네이버 클라우드(NCP) 기반 멀티 리전 배포 설계
- [ ] AI 운영 인사이트 대시보드 (정산/예약 요약, 리포트 자동 생성)
- [ ] On-premise/Fine-tuned LLM 검토 및 비용/보안 정책 수립
- [ ] AI 요금제/비용 모델 정립 (사용량 기반 vs. 구독형, 테넌트별 과금 전략)

## 3. 의존성 & 리스크

| 항목 | 내용 | 대응 |
| --- | --- | --- |
| 인력 | DevOps/보안/데이터 인력 충원 필요 | 외주 → 채용 전환 로드맵 |
| 운영 | 무중단 배포/장애 대응 역량 미비 | 자동화 + 온콜 체계 구축 |
| 데이터 | 업종 확장 시 데이터 모델 변경 | 공통/업종 모듈 분리, 중앙화 정책 유지 |
| AI | 학습 데이터/윤리 이슈 | PoC 데이터 확보, 개인정보 처리 방침 강화 |
