# 업종별 역할 예시

**작성일**: 2025-11-20  
**버전**: 1.0.0  
**상태**: 완료

---

## 📋 개요

업종별 역할 예시를 정리한 문서입니다. **MindGarden은 CONSULTATION 업종의 예시**이며, 실제 입점사는 자신만의 역할 구조를 가질 수 있습니다.

---

## 🎓 ACADEMY (학원) 예시

### 기본 역할 구조

| 레벨 | 역할명 | 템플릿 코드 | 설명 |
|------|--------|------------|------|
| 4 | 원장 | ACADEMY_DIRECTOR | 학원 최고 관리자 |
| 2 | 교사 | ACADEMY_TEACHER | 수업 담당 교사 |
| 1 | 학생 | ACADEMY_STUDENT | 수강생 |
| - | 학부모 | ACADEMY_PARENT | 학생의 학부모 |
| - | 사무원 | ACADEMY_STAFF | 학원 사무 담당 |

### 역할별 주요 기능

**원장 (ACADEMY_DIRECTOR)**
- 학원 전체 관리
- 교사 관리
- 학생 관리
- 통계 조회
- 설정 관리

**교사 (ACADEMY_TEACHER)**
- 수업 일정 관리
- 출석 체크
- 성적 입력
- 학생 관리 (담당 반)

**학생 (ACADEMY_STUDENT)**
- 수강 신청
- 출석 확인
- 성적 조회
- 일정 조회

**학부모 (ACADEMY_PARENT)**
- 자녀 출석 확인
- 자녀 성적 조회
- 학원 공지 확인

**사무원 (ACADEMY_STAFF)**
- 수강 등록 관리
- 결제 관리
- 통계 조회

## 🏥 CONSULTATION (상담소) 예시

### 기본 역할 구조

| 레벨 | 역할명 | 템플릿 코드 | 설명 |
|------|--------|------------|------|
| 4 | 원장 | CONSULTATION_DIRECTOR | 상담소 최고 관리자 |
| 2 | 상담사 | CONSULTATION_COUNSELOR | 상담 제공자 |
| 1 | 내담자 | CONSULTATION_CLIENT | 상담 받는 사람 |
| - | 사무원 | CONSULTATION_STAFF | 상담소 사무 담당 |

### 역할별 주요 기능

**원장 (CONSULTATION_DIRECTOR)**
- 상담소 전체 관리
- 상담사 관리
- 내담자 관리
- 통계 조회
- 설정 관리

**상담사 (CONSULTATION_COUNSELOR)**
- 상담 일정 관리
- 상담 기록 작성
- 내담자 관리 (할당된 내담자)
- 상담 내역 조회

**내담자 (CONSULTATION_CLIENT)**
- 상담 예약
- 상담 내역 조회
- 결제
- 힐링 컨텐츠 조회

**사무원 (CONSULTATION_STAFF)**
- 상담 예약 관리
- 결제 관리
- 통계 조회

### MindGarden 예시

**참고**: MindGarden은 CONSULTATION 업종의 실제 구현 예시입니다.

**현재 MindGarden 역할:**
- HQ_MASTER (본사 총관리자)
- HQ_ADMIN (본사 관리자)
- BRANCH_SUPER_ADMIN (지점 수퍼 관리자)
- BRANCH_ADMIN (지점 관리자)
- ADMIN (관리자)
- CONSULTANT (상담사)
- CLIENT (내담자)

**동적 역할 시스템으로 마이그레이션 시:**
- CONSULTATION_DIRECTOR → 원장
- CONSULTATION_COUNSELOR → 상담사
- CONSULTATION_CLIENT → 내담자
- CONSULTATION_STAFF → 사무원

## 🏢 기타 업종 예시

### 일반 사업체

| 레벨 | 역할명 | 설명 |
|------|--------|------|
| 4 | 사장 | 최고 관리자 |
| 3 | 관리자 | 일반 관리자 |
| 2 | 직원 | 서비스 제공자 |
| 1 | 고객 | 서비스 이용자 |

### 커스텀 역할 예시

입점사가 필요에 따라 커스텀 역할을 생성할 수 있습니다:

**예시 1: 학원 - 보조강사**
- 템플릿 없이 직접 생성
- 제한된 권한 (특정 반만 관리)

**예시 2: 상담소 - 수퍼바이저**
- 템플릿 없이 직접 생성
- 상담사 관리 권한

**예시 3: 일반 사업체 - 매니저**
- 템플릿 없이 직접 생성
- 팀 관리 권한

## 🔄 역할 커스터마이징 예시

### 예시 1: 역할명 변경

**템플릿**: ACADEMY_TEACHER (교사)
**커스터마이징**: "선생님" → "강사"

### 예시 2: 권한 추가

**템플릿**: CONSULTATION_COUNSELOR (상담사)
**커스터마이징**: 
- 기본 권한: 상담 일정 관리, 상담 기록 작성
- 추가 권한: 통계 조회 (자신의 상담 내역만)

### 예시 3: 커스텀 역할 생성

**역할명**: "보조강사"
**권한**: 
- 특정 반만 관리
- 출석 체크만 가능
- 성적 입력 불가

## ⚠️ 중요 사항

1. **MindGarden은 예시일 뿐**
   - MindGarden의 역할은 CONSULTATION 업종의 예시
   - 실제 입점사는 다른 역할 구조를 가질 수 있음

2. **역할은 동적으로 생성됨**
   - 템플릿 기반 또는 커스텀 생성
   - 테넌트 관리자가 역할 커스터마이징 가능

3. **업종별 기본 템플릿**
   - ACADEMY: 학원용 기본 템플릿
   - CONSULTATION: 상담소용 기본 템플릿
   - 기타 업종: 필요에 따라 추가
