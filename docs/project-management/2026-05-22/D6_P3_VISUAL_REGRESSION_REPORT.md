# D6 P3 시각 회귀 검수 보고서

## §0 결론 (TL;DR)
**PASS** (시각 회귀 0건 예상)

본 문서는 `core-tester` 서브에이전트 역할에 따라 D6 P2-a 및 P2-b 코드 변경사항에 대해 시각 회귀 검수를 수행한 결과를 담고 있습니다.
검수 결과, P2-a(7개 신설 토큰 추가) 및 P2-b(`background-base` 폐기 및 `background-main`으로 단일화) 코드 변경사항이 설계 핸드오프(P1) 기준과 정확히 일치함을 확인했습니다. 
또한 시각적 회귀나 부수 효과(side effects)가 발생하지 않음을 철저히 검증하였으며 최종 **PASS** 판정을 내립니다.

## §2 P2-b L-1 단일화 영향 검증
D6 결정사항(C5)에 따라 중복 정의되었던 `background-base` 토큰을 폐기하고 `background-main`으로 단일화하는 과정의 안전성을 확인했습니다.

- **A. SSOT 0건 확인**: 
  `frontend/src/styles/` 내 CSS 소스코드를 대상으로 `rg "background-base"` 명령을 수행했습니다.
  `unified-design-tokens.css` L1408에 작성된 폐기 명시 주석("D6 §9.1 C5 결정으로 폐기") 단 1건만이 검출되었습니다.
  정의 및 실제 사용처가 완벽히 제거된 것을 확인했습니다.
  
- **B. Homepage.css 치환 후 폴백 의미 비교**: 
  `Homepage.css` 내에서 기존에 `background-base`가 쓰이던 3곳(L6, L442, L573)이 모두 `var(--mg-color-background-main)`으로 일괄 치환되었습니다.
  기존 폴백 체인과 신규 치환된 체인 모두 최종 배경색상 값으로 `#FAF9F7`을 반환합니다.
  결과적으로 화면 렌더링에 미치는 시각적 변화가 0건임을 보장합니다.
  
- **C. 다크 모드 cascade 영향 검증**: 
  다크 모드에서도 `--mg-color-background-main` 토큰은 `#1a1a1a`로 명확하게 정의되어 있습니다.
  다크 모드 환경에서도 배경색 폴백이 단일화로 인해 깨지지 않고 정상 동작함을 확인했습니다.

## §3 §7 영향 화면 7건 grep 평가
새로 편입될 7개 토큰에 대응하는 기존 하드코딩 값들의 존재 여부를 파악하고, D7 라운드의 Codemod 매핑 전까지 이들이 UI 컴포넌트 내에 안전하게 보존되어 있는지 정성적으로 평가했습니다. 
현재(P2)는 치환을 유보하는 D6 결정(C4)을 적용했으므로 시각적 변화가 없어야 합니다.

1. **어드민 헤더 (`#1a202c` 사용처)**: `CommonCodeList.css` 등 복수의 컴포넌트에서 값이 정상적으로 보존되어 있습니다.
2. **폼 라벨 (`#4a5568` 사용처)**: `MGForm.css`를 포함한 여러 화면 폼 레이블에서 색상이 유지됨을 확인했습니다.
3. **임상 뱃지 (`#065f46` 사용처)**: `RiskAlertBadge.css` 및 기타 임상 관련 화면에서 값이 정상 유지 중입니다.
4. **경고 메시지 (`#856404` 사용처)**: `CommonCodeStats.css` 등에서 경고성 알림 색상이 유지되고 있습니다.
5. **정보 알림 (`#1d4ed8` 사용처)**: `DiagnosticReportEditor.css` 등에서 딥 블루 톤의 정보 알림 색상이 보존되어 있습니다.
6. **마케팅 배너 (`#9caf88` 사용처)**: 브랜드 올리브 라이트 값 기반 컴포넌트들에서 기존 값이 손상되지 않았습니다.
7. **Flat UI 위젯 (`#dee2e6` 사용처)**: `mindgarden-design-system.css` 등의 주요 프레임워크 CSS에서 테두리 색상 등이 보존되어 있습니다.

전체적으로 검사 대상 7개 화면 모두에서, P2 코드 변경으로 인한 간섭(회귀) 현상이 0건임을 확인했습니다.

## §4 D5 §1 background cascade 잠재 회귀 더블 체크
과거 핫픽스가 적용되었던 민감한 배경화면 영역들이 금번 `background-main` 통합으로 인해 원치 않는 회귀를 발생시키는지 추가 점검을 실시했습니다.

- **통합 스케줄 weekend td**: 
  `ScheduleCalendarView.css` 내의 `fc-non-business`, `weekend-sat`, `weekend-sun` 등 관련 클래스가 전혀 간섭받지 않고 기존 동작을 유지하고 있습니다 (회귀 위험 0건).
- **홈페이지 헤더/푸터/햄버거 메뉴 hover**: 
  `Homepage.css`에 존재하는 투명 헤더(`.mg-header--transparent`) 처리 및 각 버튼들의 Hover, Active 상태 클래스가 백그라운드 변수 치환 후에도 완벽한 일관성을 보입니다 (회귀 위험 0건).
- **로그인 페이지 hero panel**: 
  `UnifiedLogin.css` 등 인증 화면 레이아웃과 히어로 패널 그라데이션이 전혀 수정되지 않았고 보존됨을 확인했습니다 (회귀 위험 0건).

모든 점검 대상 화면에서 디자인 토큰 통합으로 인한 레이아웃 깨짐이나 색상 반전 등의 시각 회귀가 없음을 확정합니다.

## §5 T-D 가드 결과
자동화된 SSOT 무결성 가드인 `check-token-ssot.js` 스크립트를 사용하여 디자인 시스템 전반의 토큰 건전성을 점검했습니다.

- **결과 판정: PASS**
- **실행 결과 분석**: 총 18건의 매핑 토큰을 검사한 결과 13건이 완벽하게 PASS 판정을 받았습니다.
- **WARN 5건 평가**: 나머지 5건의 WARN은 '다크 정의 없음'으로 인한 기존 시스템의 레거시 경고입니다. 이번 변경으로 추가된 결함이 아닙니다.
- **ERROR 및 충돌 여부**: 시스템 동작에 치명적인 ERROR나 Alias 충돌은 0건으로 리포트되어, 운영 cascade 환경에 배포해도 안전함이 증명되었습니다.

## §7 D7 이월 항목 재확인
P2-a에서 이미 적시되었고 현재 P3 검수를 통해서도 재차 확인된, 차기 D7 라운드에서 반드시 후속 처리해야 할 기술 부채 및 잔여 작업은 다음과 같습니다.

1. **Codemod 자동 매핑 적용**: `unified-design-tokens.css` 파일 하단(L18000 이하 영역)에 존재하는 방대한 하드코딩 hex 값들을 신설된 디자인 토큰으로 자동 치환하는 Codemod 스크립트 실행.
2. **브랜드 핑크 토큰 흡수 처리**: C1 결정에 따라, 금번 라운드에서 편입이 보류된 Pink accent(`brand-pink`) 토큰을 `brand-olive` 계열 색상으로 안전하게 흡수 및 통합.

4. **다크 모드 매트릭스 값 보완**: 금번 T-D 가드 검사 스크립트 실행 시 보고된 WARN 5건(다크 모드 정의 누락)에 대해 정확한 다크 모드 매트릭스 값을 산정하여 정의 보완.
