# Psych AI 분석이 기초적으로 나오는 원인 분석

**배경**: 상담사 피드백 — "AI 분석 결과를 물어보니 너무 기초적인 대답만 했다. 실제 상담에 도움이 될 만한 자료가 필요해 보인다."

**참조 코드**: `OpenAIPsychAiServiceImpl`, `PsychAssessmentExtractionServiceImpl`, `PsychAssessmentReportServiceImpl.buildRuleBasedMarkdown`, `Mmpi2ExtractionParser`

---

## 1. 원인 후보 분석

### (A) 스캔/추출 문제 — **가능성 높음**

- **현재 추출 구조**
  - `PsychAssessmentExtractionServiceImpl`: PDF는 PDFTextStripper, 이미지는 Tesseract OCR로 텍스트 추출 후 `Mmpi2ExtractionParser.parse()` 호출.
  - `Mmpi2ExtractionParser`: **MMPI-2 고정 19개 척도**에 대해 `scaleCode`, `rawScore`, `tScore`만 출력. `percentile`, `scaleLabel`, `cutoffTag`는 **파서에서 생성하지 않음**.
- **DB/리포트로 넘어가는 값**
  - `ensureMetricsFromExtraction`에서 JSON의 `scaleLabel`, `percentile`, `cutoffTag`를 읽지만, 현재 파서 출력에는 없으므로 **대부분 null**.
- **결과**
  - AI에게 넘어가는 `MetricInput`은 `scaleCode`, `rawScore`, `tScore` 위주이고, percentile/라벨/컷오프가 비어 있으면 **해석에 쓸 수 있는 정보가 적음**.
  - 스캔 품질이 나쁘거나 PDF 레이아웃이 다르면 `extractNumberSequence`가 잘못 매칭되거나 누락될 수 있어, **입력 자체가 빈약하거나 왜곡**될 수 있음.

**정리**: (A)는 **입력이 적거나 잘못되는 경우**에 해당하며, 특히 percentile/라벨/타당도 해석용 메타데이터가 없어 “기초적인 대답만” 나오는 데 기여할 수 있음.

---

### (B) AI/프롬프트 문제 — **가능성 있음**

- **시스템 프롬프트** (`buildSystemPrompt`)
  - 역할: "임상심리 평가 보조 AI", "상담에 도움이 되도록 해석·소견·질문 제안".
  - 제약: 확정 진단/법적 결론 금지, 입력 수치만 사용, 위험 신호는 가능성 수준, evidence 구조화.
  - **“얼마나 깊이 있게” 해석할지는 구체적으로 명시되어 있지 않음** — "보강하라" 수준.
- **유저 프롬프트** (`buildUserPrompt`)
  - "규칙 기반 요약(초안)을 바탕으로 해석을 보강하라" + baseMarkdown + 지표 JSON.
  - 요구사항에 "## 주요 소견: T점수 상승 척도별로 **임상적 의미를 해석**, **상담 시 주목할 점 구체 작성**", "## 추적 질문: **구체적 질문 3~5개**"가 있으나, **예시·심층 해석 기준·상담 맥락**은 없음.
- **결과**
  - 모델이 “안전하게” 표면적 해석만 할 수 있음. 프롬프트가 **깊이보다 형식(섹션, evidence)**에 더 초점이 맞춰져 있음.

**정리**: (B)는 **같은 데이터로도 더 심층적으로 쓰라고 유도하지 못하는** 요인. “구체적으로”라고만 하고 수준/예시가 없어 기초적 문장이 나올 수 있음.

---

### (C) 모델 한계 — **가능성 있음**

- `SystemConfigService.getModelForProvider(providerId)`로 모델이 결정되며, **gpt-3.5-turbo** 등 저사양 모델 사용 시:
  - 복잡한 임상 해석·프로파일 해석·맥락 추론보다 **일반적 문장** 생성에 치우치기 쉬움.
- `MAX_TOKENS=4096`, `TEMPERATURE=0.3`으로 길이·일관성은 확보되나, **모델 자체의 추론 깊이**는 설정으로 보완하기 어려움.

**정리**: (C)는 **입력·프롬프트가 충분해도** 모델이 심층 해석을 잘 못하는 경우에 해당함.

---

### (D) 기타 — **출력 구조는 맞지만 내용 깊이만 부족**

- 검증 로직(`hasRequiredSections`, `validateModelOutput`)은 **"## 요약", "## 권고"** 등 **섹션 존재 여부**와 evidence 구조를 보는 데 집중되어 있음.
- **내용 품질(깊이, 구체성, 상담 활용도)**에 대한 검증은 없음. 따라서 **형식은 통과하지만 문장이 피상적**인 결과가 나와도 걸리지 않음.

**정리**: (D)는 **품질 기준이 “구조”에만 있어** “기초적인 대답”이 그대로 통과하는 경우를 말함.

## 2. 데이터 관점 정리

### 현재 Psych AI에 들어가는 입력

| 구분 | 내용 |
|------|------|
| **추출** | `PsychAssessmentExtractionServiceImpl` → `Mmpi2ExtractionParser`: MMPI-2 기준 **scaleCode, rawScore, tScore** (19척도). percentile/scaleLabel/cutoffTag는 파서에서 미제공. |
| **지표** | `PsychAssessmentReportServiceImpl`: extraction JSON에서 **scaleCode, scaleLabel, rawScore, tScore, percentile, cutoffTag**를 읽어 `MetricInput`으로 전달. 현재 파서 출력 특성상 **scaleLabel·percentile·cutoffTag는 대부분 null**. |
| **baseMarkdown** | `buildRuleBasedMarkdown`: "요약", "주요 지표"(최대 15개, `scaleCode (scaleLabel): T=…, P=…`), "권고" 2문장. **규칙 기반 초안일 뿐, 임상 해석·프로파일·척도 설명 없음**. |

→ **실질적으로 AI는 “척도 코드 + T점수(및 일부 원점수)”와 매우 단순한 문장형 초안만 보고 해석을 확장하는 구조**라, “심층적인 데이터”가 부족한 상태임.

### 심층 데이터 필요 vs 같은 데이터로 심층 해석

- **“심층적인 데이터”가 더 필요한가?**  
  - **예.** 현재는 **프로파일(코드 타입, 2점/3점 프로파일)**, **척도별 임상적 의미(설명문)**, **나이/성별 규준 보정**, **percentile/cutoff 일관된 채우기** 등이 AI 입력에 없음. 이런 정보가 있으면 “같은 T점수”라도 해석 깊이가 달라질 수 있음.
- **같은 데이터로 더 심층적인 해석이 나오게 할 수 있는가?**  
  - **어느 정도는 가능.** 프롬프트에서 **해석 깊이·상담 활용·구체적 예시(소견/추적 질문)**를 명시하고, 필요 시 **few-shot 예시**를 넣으면, 현재 수준의 지표만으로도 문장이 덜 기초적으로 바뀔 수 있음. 다만 **입력이 빈약하면 한계**가 있음.

**정리**:  
- **데이터 부족(추출/메타데이터)** → “심층 데이터 추가”가 필요.  
- **데이터는 있는데 문장이 피상적** → “같은 데이터로 더 심층 해석”은 **프롬프트/모델 개선**으로 보완 가능.  
- 현재는 **둘 다 해당**: 추출은 percentile·라벨 등이 비어 있고, baseMarkdown도 단순하며, 프롬프트는 “구체적으로” 수준만 있어 **데이터와 지시 둘 다 개선 여지**가 있음.

## 3. 결론 및 제안

### (1)(2)를 바탕으로 한 답변

- **“AI를 바꿔야 하는가?”**  
  - **모델만 바꿀 필요는 없음.** 우선 **들어가는 데이터(추출·baseMarkdown)와 프롬프트(해석 깊이·상담 활용 명시)**를 보강하는 것이 효과적임. 그 후에도 품질이 부족하면 **모델 업그레이드(gpt-4 등)**를 검토하면 됨.
- **“스캔을 잘못한 것인가?”**  
  - **완전히 잘못했다고 단정할 수는 없음.** 다만 **현재 스캔/파서는 “숫자 나열” 위주**이고, percentile·척도 설명·프로파일 등 **해석에 필요한 메타데이터는 거의 넣지 않고 있어**, “입력이 빈약해서 기초적인 대답만 나온다”는 설명이 성립함. 스캔 품질 이슈(OCR 실패, 레이아웃 이탈)가 있으면 추가로 입력 왜곡 가능성도 있음.

### 실제 상담에 도움이 되려면 — 우선 개선 순서 제안

1. **추출·입력 보강(우선 권장)**  
   - **추출**: `Mmpi2ExtractionParser`(또는 동일 출력)에서 **percentile, scaleLabel, cutoffTag**를 채울 수 있으면 채워서 `extractedJson`/`PsychAssessmentMetric`에 넣기. (규준표 매핑, 또는 외부 메타데이터 연동.)
   - **baseMarkdown**: `buildRuleBasedMarkdown`에서 **타당도 척도(VRIN, TRIN, F 등) 구분**, **상승 척도 강조**, 필요 시 **짧은 척도 설명**을 포함하면 AI가 “무엇을 해석할지” 더 명확히 알 수 있음.
   - 효과: **같은 프롬프트/모델이라도 입력이 풍부해져** 해석이 덜 기초적으로 나올 가능성이 큼.

2. **프롬프트/지시 개선(병행 권장)**  
   - **시스템/유저 프롬프트**에 “상담에서 바로 쓸 수 있는 **구체적 소견·추적 질문**을 제시하라”, “**척도 조합(프로파일)과 임상적 함의**를 간단히 서술하라” 등 **깊이·구체성 기준**을 추가.
   - 가능하면 **소견/추적 질문 1~2개 예시**를 프롬프트에 포함해, “이 정도 구체성”을 보여 주는 것이 좋음.
   - 효과: **입력은 그대로라도** 출력이 더 심층·실무적으로 나오게 유도할 수 있음.

3. **모델 선택(필요 시)**  
   - 위 1·2를 적용한 뒤에도 “기초적”이라는 피드백이 남으면, **gpt-4 등 상위 모델**로 변경해 추론 깊이를 높이는 것을 검토.

**한 줄 요약**:  
- **원인**: 스캔/추출이 “숫자 위주”이고 percentile·라벨 등이 비어 있어 입력이 빈약한 것(A), 프롬프트가 “깊이”보다 형식에 치우친 것(B), 그리고 필요 시 모델 한계(C)가 겹쳐 있을 수 있음.  
- **제안**: **추출·baseMarkdown 보강**과 **프롬프트(해석 깊이·상담 활용·예시) 개선**을 **둘 다** 우선 진행하고, 그래도 부족하면 모델 업그레이드를 검토하는 것이 좋음.
