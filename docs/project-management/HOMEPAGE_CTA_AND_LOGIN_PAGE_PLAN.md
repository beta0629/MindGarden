# 홈페이지 CTA 및 로그인 페이지 기획 요약

## 1. 제목·목표

- **제목**: Hero CTA 플로우 정리 및 로그인 페이지 비주얼 방향성
- **목표**: "무료로 시작하기" 버튼의 비로그인 시 이동 대상(로그인 vs 회원가입)을 결론 내리고, 로그인 페이지 좌측 이미지 영역의 필수·선택 여부를 기획 관점에서 정리한다. 코드·디자인 수정은 본 문서에서 수행하지 않는다.

---

## 2. CTA 플로우 결론: "무료로 시작하기" → **로그인(/login)** 권장

### 2.1 현재 동작

- Hero "무료로 시작하기" 클릭 시 비로그인 사용자는 **회원가입(/register)** 으로 이동함.
- 사용자 기대: **로그인 페이지(/login)** 로 이동하는 것.

### 2.2 권장: 로그인(/login)으로 이동

**결론**: 비로그인 사용자 클릭 시 **로그인(/login)** 으로 보내는 것을 권장한다.

| 근거 | 설명 |
|------|------|
| **전형적인 랜딩 플로우** | 대부분의 서비스는 "시작하기" → 로그인 화면 → 미가입 사용자는 "회원가입" 링크로 이동하는 흐름을 따른다. 이미 계정이 있는 사용자가 더 많거나, 재방문·이탈 후 복귀 시나리오를 고려하면 로그인 진입이 자연스럽다. |
| **사용자 기대와 일관성** | "시작하기"는 "서비스를 쓰기 위해 들어간다"는 의미에 가깝고, 기존 사용자는 로그인을 기대한다. 회원가입은 "아직 계정이 없을 때만" 선택하는 경로로 두는 것이 인지 부담을 줄인다. |
| **전환·이탈 관리** | 로그인 페이지에서 "회원가입" 링크를 명확히 두면, 신규 사용자도 한 단계만 더 가면 가입할 수 있다. 반대로 CTA가 곧바로 회원가입으로 가면, 이미 가입한 사용자는 불필요한 화면을 거치게 되어 이탈 가능성이 커진다. |

### 2.3 정리

- **권장 동작**: Hero "무료로 시작하기" 클릭 시 비로그인 사용자 → **/login** 이동.
- **회원가입 경로**: 로그인 페이지에 "계정이 없으신가요? 회원가입" 등 링크를 두어 **/register** 로 유도.
- 구현 시에는 홈 Hero CTA의 `handleRegister()` 호출을 로그인 페이지 이동으로 변경하는 방향을 적용하면 된다(구체 수정은 구현 Phase에서 수행).

## 3. 로그인 페이지 비주얼: 좌측 이미지 영역

### 3.1 현재 인상

- 스펙상 좌측에 브랜딩·이미지 배경 영역이 있으나, 사용자 체감은 "이미지가 없고 텍스트만 있다"는 반응이 있음. 이미지 미노출 또는 우측 폼만 강조된 인상일 수 있다.

### 3.2 기획 관점 정리

**좌측 이미지 영역은 필수가 아니며, 선택(권장)으로 두는 것을 제안한다.**  
데스크톱에서는 브랜딩·이미지가 있으면 서비스 정체성과 신뢰감을 높일 수 있어 **권장**한다. 다만 로그인의 핵심 목표는 "계정으로 진입"이므로, **좌측 영역이 없거나 텍스트만 있어도 기능적으로는 문제 없다.**  
**모바일**에서는 공간 제약과 터치·가독성 우선으로, 좌측 이미지 영역을 **생략하고 상단 로고·짧은 문구 + 폼만 노출**하는 구성이 일반적이며, 이 경우에도 좌측 이미지를 필수로 두지 않아도 된다.  
따라서 "좌측 이미지 필수"로 규정하기보다는, **데스크톱에서는 이미지·브랜딩 영역 유지(또는 개선)를 권장하고, 모바일에서는 생략 가능**하다고 보는 것이 적절하다. 이미지가 현재 미노출된다면 원인(경로·로딩·반응형 분기) 확인 후 데스크톱에서만이라도 노출되도록 하는 개선을 검토할 수 있다.

## 4. 요약

| 항목 | 결론·권장 |
|------|-----------|
| **Hero "무료로 시작하기"** | 비로그인 시 **/login** 으로 이동. 로그인 페이지에서 회원가입(/register) 링크 제공. |
| **로그인 페이지 좌측 이미지** | 필수 아님. 데스크톱에서는 권장, 모바일에서는 생략 가능. 이미지 미노출 시 원인 확인 후 데스크톱 노출 개선 검토. |

본 문서는 기획·결론 및 권장사항만 기술하며, 실제 코드·디자인 수정은 별도 Phase(구현·디자인)에서 수행한다.
