# Phase 1: 국내 B2B/SaaS 디자인 스타일 벤치마킹 (2026.06.17)

> **Status**: Active  
> **Scope**: Phase 1 **`/` 메인 허브 시안** 참고 자료. public landing 1순위 아님.

본 문서는 채널톡, 토스페이먼츠, 센드버드, 원티드비즈 등 국내 Top-tier B2B SaaS의 랜딩페이지 및 UI/UX 트렌드를 분석하고, MindGarden (Core Solution) **내부 앱 허브** 및 UI 시안에 적용할 디자인 원칙을 도출합니다.

## 1. 벤치마크 대상 및 주요 트렌드 분석

### 1.1. 타이포그래피 (Typography)
- **서체 구성**: 영문은 Inter, 국문은 Noto Sans KR 또는 Pretendard를 조합하여 사용.
- **크기 및 행간**: Hero 섹션의 주요 타이틀은 54~64px 이상의 과감한 크기와 1.3~1.4 수준의 타이트한 행간을 적용. 본문은 15~16px.
- **자간 (Tracking)**: 타이틀 텍스트는 자간을 좁혀(-1px ~ -1.5px) 단단하고 명료한 인상을 주며, 본문은 기본 자간을 유지하여 가독성 확보.
- **색상**: 순수 검정(#000000) 대신 Off-black(`rgba(0,0,0,0.85)` 또는 `#2C2C2C`)을 사용하여 눈의 피로도를 낮추고 세련된 느낌 전달.

### 1.2. 여백 및 레이아웃 (Spacing & Layout)
- **섹션 간격 (Section Spacing)**: 콘텐츠가 숨 쉴 수 있도록 섹션 사이에 80~120px의 넉넉한 수직 패딩(Vertical Padding)을 부여. 여백 자체가 콘텐츠 간의 시각적 계층을 나눔.
- **그리드 구조**: 맥스 위드(Max-width)를 둔 중앙 정렬 컨테이너를 기본으로 하되, 강조할 섹션만 전체 너비(Full-width) 배경 사용.
- **컴포넌트 여백**: 카드 내부는 24~32px, 인라인 요소 간은 8~12px 등 토큰화된 간격 규칙을 엄격하게 적용.

### 1.3. 색상: 채도 및 대비 (Color & Contrast)
- **브랜드 컬러의 절제된 활용**: 화이트 또는 미색 캔버스(#FAF9F7 등)를 배경으로 사용하고, Primary Color는 CTA 버튼이나 핵심 강조 요소(악센트)에만 사용하여 집중도를 극대화.
- **제약**: 전형적인 B2B SaaS의 쨍한 블루/퍼플 그라데이션은 배제되고, 브랜드의 개성을 담은 단색 기반 솔리드 컬러나 은은한 모노톤 조합이 대세.

### 1.4. 카드 및 버튼 곡률 (Radius)
- **계층적 곡률 적용**: 임의의 값을 쓰지 않고 크기와 위계에 따라 토큰화된 Radius 적용.
  - 작은 칩/라벨: 8px
  - 폼/인풋: 8~10px
  - CTA 버튼: 10px ~ 18px (완전한 캡슐형보다는 모서리가 둥근 직사각형 선호)
  - 주요 콘텐츠 카드/섹션 블록: 16px ~ 24px

### 1.5. Hero 섹션 구성 (Hero Composition)
- **구조**: 좌측 텍스트(문제 해결 중심의 명확한 카피 + 보조 설명)와 우측 직관적 시각 자료, 또는 중앙 정렬된 타이틀 아래에 거대한 대시보드 화면 배치.
- **CTA 버튼**: 2개 이하로 제한하여 사용자 여정 분산 방지 (예: [무료 시작하기] [도입 문의]).

### 1.6. 일러스트 vs 실제 UI (Illustration vs Real UI)
- **Real UI 우세**: 추상적인 인물 일러스트보다는 **실제 동작하는 프로덕트 UI 목업(대시보드 화면, 코드 스니펫, 인터랙션 요소)**을 노출하는 것이 대세. 고객은 자신이 구매할 소프트웨어가 어떻게 생겼고 작동하는지 바로 확인하고 싶어함.

### 1.7. 신뢰 섹션 (Social Proof)
- **배치**: Hero 섹션 직후인 'Above the fold' 끄트머리나 바로 아래에 배치.
- **표현 방식**: 고객사 로고를 흑백 또는 단색 처리하고, 투명도를 70% 수준으로 낮춰 UI 시선을 뺏지 않으면서도 신뢰를 각인.

### 1.8. 다크 모드 및 섹션 반전 (Dark Mode & Inversion)
- 전체 다크 모드 지원보다는, 제품의 특정 강점(예: 보안, AI, 인프라)을 설명하는 **특정 섹션의 배경을 어둡게 반전**시켜 시각적 환기를 주고 메시지를 분리하는 기법(Dark Section)을 선호.

## 2. 목업 적용 10원칙 (MindGarden V2 원칙)

위의 벤치마크 결과와 **MindGarden (Core Solution)의 Calm Forest 브랜드 토큰 제약사항**을 결합하여 랜딩페이지 및 목업 설계에 적용할 10가지 원칙을 정의합니다.

1. **로고 및 에셋 보존**: 새 로고 발명을 엄격히 금지하며, `core-solution-logo-primary.svg` 등 기존 Trinity 에셋만 사용한다.
2. **블루/퍼플 그라데이션 금지**: 흔한 SaaS식 화려한 그라데이션 대신, Calm Forest 테마(Primary `#3D5246`, Background `#FAF9F7`) 기반의 절제되고 신뢰감 있는 색상만 사용한다.
3. **Off-Black 텍스트**: 완전한 검은색이 아닌 다크 그레이(`#2C2C2C`)를 기본 텍스트 색상으로 사용하여 부드럽고 가독성 높은 한국형 B2B UX를 구현한다.
4. **대담한 Hero 타이포그래피**: 국문 타이틀(Noto Sans KR)은 크기를 키우고 자간을 타이트하게 조절(-1px)하여 묵직하고 명료하게 메시지를 전달한다.
5. **Real UI 중심의 시각화**: Hero 및 주요 기능 설명 시, 추상적 일러스트를 배제하고 어드민 대시보드 샘플 기반의 실제 UI 목업 이미지를 전면 배치한다.
6. **섹션 간 넉넉한 여백**: 각 콘텐츠 블록(섹션) 사이에 80~120px의 과감한 수직 패딩을 주어 정보 과부하를 방지하고 고급스러운 밀도를 만든다.
7. **토큰화된 Radius 위계**: 버튼은 10px, 콘텐츠 카드 및 섹션 블록은 16px로 Radius를 고정하여 전체 레이아웃의 형태적 일관성을 확보한다.
8. **단색 고객사 로고 (Social Proof)**: Hero 직후에 배치되는 신뢰 요소(고객사 로고)는 무채색 톤과 투명도를 적용해 시각적 노이즈를 줄인다.
9. **시각적 앵커 (좌측 악센트 바)**: 주요 콘텐츠 카드나 섹션 제목 옆에 브랜드 Primary Color(`#3D5246`)를 활용한 4px 세로 악센트 바를 사용하여 구조적 명확성을 높인다.
10. **다크 섹션 활용**: 강조가 필요한 특정 영역(예: 보안, 엔터프라이즈 기능)은 사이드바와 동일한 다크 배경(`#2C2C2C`)으로 반전시켜 캔버스에 리듬감을 준다.
