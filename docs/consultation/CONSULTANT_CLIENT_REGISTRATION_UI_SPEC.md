# 상담사·내담자 등록 및 상담일지 상단 UI/UX 스펙

> MindGarden — 주민번호·주소·상담사 자격/경력 추가 및 상담일지 내담자 정보·심리검사 리포트 영역  
> @author core-designer | @since 2026-03-02  
> **참조**: [CONSULTANT_CLIENT_REGISTRATION_ADDITIONS_PLAN.md](./CONSULTANT_CLIENT_REGISTRATION_ADDITIONS_PLAN.md) §8 디자이너 전달 사양

---

## 1. 개요 및 배경

- **목적**: 상담사·내담자 등록 폼에 주민번호(앞 6자리 + 뒤 1자리), 주소(카카오 API), 상담사 전용 자격증·경력사항을 추가하고, 상담일지 상단에 내담자 정보(나이·성별 보강) 및 심리검사 리포트 영역을 명확히 노출하여 상담 활용도를 높인다.
- **적용 범위**: (1) 상담사 등록 폼, 내담자 등록 폼 (2) 상담일지 작성/조회 화면(ConsultationRecordScreen) 상단.
- **디자인 기준**: AdminCommonLayout, B0KlA(mindgarden-design-system), `unified-design-tokens.css` 및 `docs/design-system/PENCIL_DESIGN_GUIDE.md` 단일 소스. 코드는 작성하지 않고 시안·스펙만 산출한다.

구현 시 아래 토큰만 사용한다. hex/px 하드코딩 금지.

| 용도 | 토큰 (우선) | 참고 값 |
|------|-------------|---------|
| 메인 배경 | `var(--mg-color-background-main)` | #FAF9F7 |
| 서페이스/카드 | `var(--mg-color-surface-main)` | #F5F3EF |
| 테두리 | `var(--mg-color-border-main)` | #D4CFC8 |
| 주조 | `var(--mg-color-primary-main)` | #3D5246 |
| 주조 밝음 | `var(--mg-color-primary-light)` | #4A6354 |
| 본문 텍스트 | `var(--mg-color-text-main)` | #2C2C2C |
| 보조 텍스트/라벨 | `var(--mg-color-text-secondary)` | #5C6B61 |
| 오류 강조 | `var(--mg-error-500)` 또는 프로젝트 오류 토큰 | — |
| 섹션 패딩 | `var(--mg-spacing-24)` (24px) | 24px |
| 필드 간 gap | 16px ~ 24px | `var(--mg-spacing-16)` 등 |
| 카드 radius | 16px | — |

- **레이아웃**: AdminCommonLayout — 좌측 LNB 260px, 메인 영역 상단 바(브레드크럼·제목·액션) + 본문 패딩 24~32px. 본문 내 콘텐츠는 **섹션 블록** 단위(배경 surface, 테두리, radius 16px, 패딩 24px, 좌측 세로 악센트 바 4px)로 구분.

## 3. (1) 상담사/내담자 등록 폼 — 추가 항목 스펙

등록 폼은 기존 필드(아이디, 이메일, 비밀번호, 이름, 전화 등) 아래에 **새 섹션**으로 구분하여 추가한다. 데스크톱에서는 2단 그리드, 모바일에서는 1단으로 배치한다.

### 3.1 주민번호 (공통: 상담사·내담자)

- **정책**: 앞 6자리(YYMMDD) + 뒤 1자리(성별·세대 구분)만 입력·저장. 전체 13자리 또는 원문 노출 금지.
- **필수/선택**: 기획 정책에 따라 필수 또는 선택. 필수일 경우 라벨 옆에 `*` 표시.

| 항목 | 스펙 |
|------|------|
| **배치** | **한 줄 두 칸**: 왼쪽 칸 "주민번호 앞 6자리", 오른쪽 칸 "주민번호 뒤 1자리". 두 입력란은 같은 행에 나란히 배치하며, 시각적으로 한 묶음(테두리/배경 없이 라벨+입력만)으로 처리. |
| **라벨** | 왼쪽: "주민번호 앞 6자리" (선택 시 "(선택)" 또는 필수 시 빨간 `*`). 오른쪽: "주민번호 뒤 1자리". |
| **placeholder** | 앞 6자리: `900101`. 뒤 1자리: `1`. (숫자만 입력되므로 placeholder는 예시용.) |
| **입력 제한** | 숫자만 입력. 앞 6자리 maxLength 6, 뒤 1자리 maxLength 1. (자동 포맷 하이픈은 사용하지 않음.) |
| **검증 오류 메시지** | **형식 오류**: "주민번호 앞 6자리는 6자리 숫자, 뒤 1자리는 1자리 숫자로 입력해 주세요." **체크섬 오류**: "주민번호가 올바르지 않습니다. 다시 확인해 주세요." (뒤 1자리만 받을 경우 체크섬 검증이 제한적이면, 기획에 따라 형식 오류만 사용 가능.) |
| **오류 표시** | 해당 입력란 하단 또는 오른쪽에 오류 메시지 표시. 색상 `var(--mg-error-500)` 또는 프로젝트 오류 토큰. 폰트 12px. |

- **사용성**: 라벨과 입력란은 기존 폼과 동일한 라벨/인풋 스타일(라벨 12px, 보조색; 인풋 border 1px, radius 8~10px, padding 10~12px)을 적용한다.

### 3.2 주소 (공통: 상담사·내담자)

- **정책**: 카카오 주소 API로 "주소" 검색 후, 상세 주소·우편번호는 사용자 입력 또는 API 반환값으로 채운다.

| 항목 | 스펙 |
|------|------|
| **배치** | **주소를 하나의 블록**으로 묶어 배치. 순서: ① 주소 검색(버튼 + 검색 결과로 채워지는 "주소" 읽기 전용 또는 입력란) ② 상세 주소 ③ 우편번호. 상세 주소와 우편번호는 한 줄 두 칸(우편번호 오른쪽, 상세 주소 왼쪽·더 넓게) 또는 두 줄(주소 한 줄, 그 다음 줄에 우편번호 | 상세 주소)로 배치. 권장: 첫 줄 "주소 검색" 버튼 + 주소 표시 영역, 둘째 줄 "상세 주소" 전체 너비, 셋째 줄 "우편번호" 한 칸. |
| **라벨** | "주소 검색", "상세 주소", "우편번호". 필수/선택 정책에 따라 `*` 표시. |
| **주소 검색** | 버튼 문구: "주소 검색" 또는 "주소 찾기". 버튼 스타일: 아웃라인(배경 없음, 테두리 `var(--mg-color-border-main)`), height 40px, radius 10px. 클릭 시 카카오 주소 API(새 창/팝업 또는 임베드) 호출. 검색 결과로 "주소" 필드와 "우편번호" 필드가 채워지면, 사용자는 "상세 주소"만 추가 입력. |
| **placeholder** | 주소(검색 전): "주소 검색 버튼을 눌러 주소를 입력하세요." 상세 주소: "동, 호수, 상세 주소를 입력하세요." 우편번호: "00000" (숫자 5자리). |
| **사용성** | 주소는 검색 후 상세 주소만 직접 입력. 우편번호는 API 반환값으로 자동 채움 가능하면 읽기 전용 표시. |

- **섹션 구분**: "주소" 블록은 기존 "연락처" 등과 동일한 폼 섹션 내에서 구분선 또는 소제목 없이, 라벨로만 구분해도 되며, 필요 시 "주소 정보" 같은 소제목(12px, 보조색)을 위에 둘 수 있다.

### 3.3 상담사만 — 자격증·경력사항

- **적용 대상**: 상담사 등록 폼에만 노출. 내담자 등록 폼에는 없음.
- **기존 폼과의 구분**: 기존 "전문 분야", "메모" 등과 **섹션 구분** — "자격·경력" 블록으로 묶고, **섹션 제목**을 둔다.

| 항목 | 스펙 |
|------|------|
| **섹션 제목** | "자격·경력". 좌측 세로 악센트 바(폭 4px, `var(--mg-color-primary-main)`), 제목 텍스트 16px, fontWeight 600, `var(--mg-color-text-main)`. 기존 폼의 다른 섹션(예: 기본 정보, 연락처)과 동일한 스타일. |
| **자격증** | 라벨: "자격증" (선택 시 "(선택)" 표시). placeholder: "보유 자격증을 입력하세요. 예) 정신건강임상심리사 1급, 상담심리사 1급". 입력: 한 줄 텍스트(input) 또는 여러 개 입력 시 textarea. 기존 `certification`(100자) 매핑. |
| **경력사항** | 라벨: "경력사항" (선택). placeholder: "관련 경력 및 근무 이력을 입력하세요.". 입력: textarea, 여러 줄. 기존 `work_history`(1000자) 매핑. |
| **배치** | 자격증 한 줄, 그 아래 경력사항 textarea. 데스크톱에서도 두 필드 모두 전체 너비(한 칸)로 배치. |

- **필수/선택**: 둘 다 선택 항목으로 표시(라벨 옆 "(선택)" 또는 필수 시 `*`만).

### 3.4 등록 폼 공통 사용성 요약

| 요구사항 | 적용 |
|----------|------|
| 필수/선택 표시 | 모든 필드 라벨 옆에 필수는 `*`, 선택은 "(선택)" 또는 생략. |
| 주민번호 숫자만 | 앞 6자리·뒤 1자리 input type="text" + pattern/inputFilter로 숫자만 허용. |
| 주소 검색 후 상세만 | "주소 검색" 버튼 → 카카오 API → 주소·우편번호 자동 채움 → 상세 주소만 사용자 입력. |

## 4. (2) 상담일지 상단 — 내담자 정보 카드·심리검사 리포트 영역

상담일지 화면(ConsultationRecordScreen)은 AdminCommonLayout 안에서, **상단 바 아래**에 (A) 내담자 정보 카드, (B) 심리검사 리포트 영역을 배치한다. 기존 "상담일지 작성" 폼은 그 아래에 유지한다.

### 4.1 내담자 정보 카드 (보강)

- **현재 노출 필드**: 이름, 이메일, 전화번호, 주소, 상태, 등급, 가입일.
- **추가 노출 필드**: **나이**, **성별**. (주민번호는 **노출하지 않음**.)
- **주소 표시**: "주소" 한 필드에 **기본 주소 + 상세 주소**를 함께 표시. 우편번호 포함 여부는 정책에 따라: 포함 시 "우편번호 + 기본 주소 + 상세 주소" 형태로 한 줄 또는 두 줄로 표시.

| 항목 | 스펙 |
|------|------|
| **카드 컨테이너** | 배경 `var(--mg-color-surface-main)`, 테두리 `1px solid var(--mg-color-border-main)`, border-radius 16px, 패딩 `var(--mg-spacing-24)`. |
| **카드 제목** | "내담자 정보". 좌측 세로 악센트 바 4px `var(--mg-color-primary-main)` + 제목 16px, fontWeight 600, `var(--mg-color-text-main)`. |
| **그리드** | 기존과 동일하게 2열 그리드(데스크톱). 각 항목: 라벨 12px `var(--mg-color-text-secondary)`, 값 14~16px `var(--mg-color-text-main)`. |
| **필드 순서(권장)** | 이름, 나이, 성별, 이메일, 전화번호, 주소(상세/우편 포함 여부 정책 반영), 상태, 등급, 가입일. (나이·성별은 이름 바로 아래 또는 이름 오른쪽에 배치해 가독성 확보.) |
| **나이·성별 없을 때** | 데이터 없으면 "—" 또는 "정보 없음" 표시. |

### 4.2 심리검사 리포트 영역

- **위치**: **내담자 정보 카드 바로 아래** 권장. (옆에 배치할 경우: 데스크톱에서 2단 레이아웃 — 왼쪽 내담자 정보 카드, 오른쪽 심리검사 리포트. 단, 상담일지 본문 폼이 넓게 쓰이므로 "카드 아래"가 기본.)
- **표시 방식**: "링크 + 요약 1줄" 수준. 각 심리검사 문서/리포트에 대해: **제목 또는 문서명(링크)** + **요약 텍스트 1줄**(있을 경우). 클릭 시 해당 심리검사 상세 페이지로 이동.
- **없을 때**: 등록된 심리검사 리포트가 없을 때 문구: "등록된 심리검사 리포트가 없습니다." 또는 "아직 등록된 심리검사가 없습니다." 배경은 카드와 동일하게 surface, 테두리·radius 유지, 패딩 24px, 텍스트 14px `var(--mg-color-text-secondary)`, 중앙 또는 왼쪽 정렬.

| 항목 | 스펙 |
|------|------|
| **영역 제목** | "심리검사 리포트". 좌측 세로 악센트 바 4px(보조색 `var(--mg-color-secondary-main)` 또는 포인트색으로 구분 가능) + 제목 16px, 600, `var(--mg-color-text-main)`. |
| **컨테이너** | 배경 `var(--mg-color-surface-main)`, 테두리 `1px solid var(--mg-color-border-main)`, border-radius 16px, 패딩 `var(--mg-spacing-24)`. |
| **목록 항목** | 각 리포트: 제목(또는 문서명)은 링크 스타일(색 `var(--mg-color-primary-main)`, 밑줄 hover). 그 아래 또는 옆에 요약 1줄(14px, `var(--mg-color-text-secondary)`). 여러 건일 경우 리스트 형태, gap 12~16px. |
| **빈 상태** | "등록된 심리검사 리포트가 없습니다." 14px, `var(--mg-color-text-secondary)`. |

## 5. 상호작용·상태

- **등록 폼**
  - 주민번호: blur 또는 제출 시 검증. 형식/체크섬 오류 시 해당 필드 하단에 오류 메시지 표시.
  - 주소 검색: 버튼 클릭 → 카카오 API 팝업/레이어 → 선택 시 주소·우편번호 반영.
  - 자격증·경력: 비어 있어도 제출 가능(선택 항목).
- **상담일지**
  - 내담자 정보 카드: 읽기 전용. 나이·성별·주소는 API에서 내려주는 값 그대로 표시.
  - 심리검사 영역: 목록만 표시. 클릭 시 상세 페이지 이동. 로딩 중에는 해당 영역에 스피너 또는 스켈레톤 적용.

## 7. 코더 전달 체크리스트

- [ ] 주민번호: 한 줄 두 칸(앞 6자리 / 뒤 1자리), 라벨·placeholder·숫자만 입력·검증 메시지(형식/체크섬) 반영.
- [ ] 주소: "주소 검색" 버튼(카카오 API), "상세 주소", "우편번호" 라벨·placeholder·배치(검색 후 상세만 입력).
- [ ] 상담사만: "자격·경력" 섹션 제목 + 자격증·경력사항 라벨·placeholder·배치.
- [ ] 필수/선택 표시 일관 적용.
- [ ] 상담일지 상단: 내담자 정보 카드에 나이·성별 추가, 주소는 상세/우편 포함 여부 정책 반영.
- [ ] 상담일지 상단: 심리검사 리포트 영역 카드 아래(또는 옆) 배치, 링크+요약 1줄, 없을 때 문구.
- [ ] 모든 색·간격·radius는 지정된 `var(--mg-*)` 토큰 사용.
