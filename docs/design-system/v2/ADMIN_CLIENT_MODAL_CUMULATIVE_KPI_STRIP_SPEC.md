# ADMIN ClientModal 상단 KPI 스트립 스펙 (core-designer 정리)

본 문서는 **관리자(ADMIN) ClientModal** 상단에 배치하는 **누적·등급 관련 KPI 스트립**에 대한 디자인 시스템 관점 스펙이다. 구현은 `ContentSection` + `ContentKpiRow` 등 공통 레이아웃 컴포넌트와 디자인 토큰을 따른다.

## 1. 정보 위계

1. **1순위**: 내담자 식별·컨텍스트(이름 등) — 모달 헤더 영역과 구분되되, KPI와 시각적 충돌 없게 유지.
2. **2순위**: **누적 KPI** (세션·이용 관련 수치) — 한 눈에 스캔 가능한 숫자·라벨 쌍.
3. **3순위**: **회원등급** — 읽기 전용 배지/텍스트로 표시, KPI 숫자보다 보조적 위계.
4. **4순위(선택)**: **다음 등급까지 진행** — API에 해당 필드가 있을 때만 노출; 없으면 영역 자체를 렌더하지 않음(빈 상태·플레이스홀더 금지).

## 2. 레이아웃: ContentSection + ContentKpiRow

- **섹션 래퍼**: `ContentSection`으로 KPI 블록을 그룹화한다. 제목(예: 누적 현황)은 짧고 동일 용어를 다른 화면과 공유한다.
- **KPI 행**: `ContentKpiRow`(또는 동등한 분자/분모 패턴 컴포넌트)로 지표를 가로 배치한다.
  - 각 셀: **라벨(상단 또는 좌측, 토큰 타이포)** + **값(강조 타이포)**.
  - 3개 이상 지표 시 **반응형**으로 줄바꿈 또는 스크롤 가능한 행(모바일 폭에서는 세로 스택 우선).

## 3. 디자인 토큰 (`var(--mg-*)`)

- 배경·테두리·간격·타이포는 **디자인 시스템 v2 토큰**만 사용한다. (`var(--mg-*)` 계열)
- 색상: KPI 값은 `text` 계열 강조, 라벨은 `muted` 계열. 등급 배지는 기존 `Badge`·시맨틱 토큰과 정합.
- 간격: 섹션 내부 `padding`/`gap`은 토큰 스케일에 맞추고 임의 `px` 금지.

## 4. 반응형

- **데스크톱**: KPI를 한 행 또는 2행 그리드로 균등 배분.
- **태블릿/모바일**: KPI를 세로 스택 또는 2열 그리드로 줄어들게 하고, 터치 타깃 최소 높이 준수.
- 모달 최대 너비 내에서 **가로 스크롤**이 필요하면 스크롤바·포커스 순서가 접근성 기준을 만족해야 한다.

## 5. 등급 표시

- **읽기 전용**: 관리자 모달 KPI 스트립 내 등급은 표시만 하며, 이 스펙 범위에서 인라인 편집·드롭다운 변경 UI를 두지 않는다. (등급 변경은 별도 플로우·모달.)
- 등급 라벨은 API/도메인에서 내려준 코드·이름을 `safeDisplay` 등 표시 경계 유틸과 함께 사용한다.

## 6. 다음 등급 진행 표시

- **조건부 노출**: 백엔드/API에 **다음 등급명·잔여 조건·진행률** 등 명시 필드가 있을 때만 프로그레스 또는 보조 텍스트를 표시한다.
- 필드 부재 시: 해당 서브블록을 **렌더하지 않음**(0% 바, "준비 중" 등 추측 문구 금지).

## 7. 구현 체크리스트

- [ ] `ContentSection`으로 KPI 그룹 경계가 명확한가?
- [ ] `ContentKpiRow`(또는 동등 패턴)로 라벨·값 정렬이 일관적인가?
- [ ] 색·간격·폰트가 `var(--mg-*)` 토큰만 사용하는가?
- [ ] 좁은 뷰포트에서 레이아웃이 깨지지 않고 스택/그리드 전환이 자연스러운가?
- [ ] 등급이 읽기 전용으로만 보이는가?
- [ ] 다음 등급 진행 UI는 API 필드가 있을 때만 나타나는가?
- [ ] 하드코딩된 팔레트·문자열이 없고 표준 컴포넌트를 재사용하는가?
