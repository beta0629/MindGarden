# 세션 유휴 경고 모달 — 백그라운드 탭·401 선행 이슈 (기획 주관 1장 오케스트레이션)

**작성일**: 2026-04-25  
**저장소**: `/Users/mind/mindGarden`  
**역할**: `core-planner` 주관 · 구현·원인 단정은 서브에이전트 위임

---

## 1) 문제·사용자 영향

30분 유휴 시나리오에서 사용자는 **만료 직전 경고 모달**을 통해 연장·안내를 받아야 하나, 탭이 백그라운드에 있거나 타이머·세션 점검이 경쟁하는 상황에서는 **모달 없이 `current-user` 401 후 로그인(document 200)으로 이동**하는 것으로 관찰된다. 결과적으로 **예고 없는 세션 종료**로 작업 중단·재인증 비용이 발생하고, 운영 문의(“경고 없이 끊김”)로 이어질 수 있다.

---

## 2) 가설 표 (H1~H4)

| ID | 가설 요약 | 검증 방향(위임 시 태스크에 포함) |
|----|------------|----------------------------------|
| **H1** | 브라우저가 백그라운드 탭의 `setTimeout`/타이머를 **스로틀**해 유휴 카운트다운·모달 표시 타이밍이 밀리거나 건너뛴다. | 탭 전환·최소화 재현, 타이머 구현 위치·대안(Visibility API, `document.hidden`, 서버 기준 보조) 인벤토리 |
| **H2** | 주기적 `checkSession`(또는 동등 호출)과 **유휴 모달 타이밍이 경쟁**해, 만료·401이 먼저 발생한다. | 호출 순서·간격·401 처리 분기와 모달 게이트 관계 추적 |
| **H3** | `session-info` 등 **보조 데이터가 UI/타이머와 동기되지 않음**으로, 클라이언트 표시 만료와 서버 세션 불일치가 난다. | API 응답 필드·캐시·스토어와 모달 조건의 단일 소스 여부 확인 |
| **H4** | (복합) H1~H3 중 둘 이상 + **401 시 공통 리다이렉트**가 모달보다 우선 실행된다. | 401 핸들러·ajax 인터셉터·라우트 가드와 세션 UX 플로우 순서 정리 |

## 3) Phase 분배 (A~D)

| Phase | 담당 | 목표 | 비고 |
|-------|------|------|------|
| **A** | **explore** (선택) | `SessionManagement`, 유휴 모달, `checkSession`, `current-user`/401 처리, 타이머·Visibility 사용 여부 **인벤토리·호출 그래프** 1페이지 요약 | 범위가 이미 명확하면 생략 가능; 생략 시 **B 프롬프트에 파일 후보·증상 재현 절차**를 코더가 자체 탐색하도록 명시 |
| **B** | **core-coder** | 백그라운드 탭에서도 **만료 1분 전 모달 또는 동등 UX**(사용자가 연장·인지 가능) 달성을 위한 **구현 범위 확정 및 패치** | 표준: `/core-solution-frontend`, `/core-solution-api`, `/core-solution-multi-tenant`; 필요 시 **core-debugger**에 원인 분석 의뢰 후 B 재실행 |
| **C** | **core-tester** | 수동: 30분·백그라운드·탭 복귀 시나리오 체크리스트; **eslint**·관련 스모크(세션·ajax 경로) | 코드 변경 시 [검증 게이트](docs/project-management/CORE_PLANNER_DELEGATION_ORDER.md) 준수 |
| **D** | **사용자 확인 (운영)** | 스테이징/운영에서 동일 시나리오 재현 후 **모달 선행·401 없음(또는 모달 후 정상 흐름)** 확인 | 배포 절차는 `core-deployer`·운영 체크리스트와 별도 합의 |

**의존성**: A(선택) → B → C → D. A 생략 시 B가 탐색·구현을 일괄.

## 4) 완료 정의 (Definition of Done)

- **백그라운드 탭**에서도(크롬 기본 스로틀 조건 하에서) **세션 만료 약 1분 전**에 유휴 경고 **모달 또는 동등 UX**(동일 정보·동일 행동 가능: 연장/안내)가 **401·로그인 리다이렉트보다 선행**한다.  
- `current-user` 401이 **모달 없이** 최초 사용자 대면으로 이어지는 재현 경로가 **Phase C·D에서 재현 불가**로 확인된다(또는 의도된 예외가 문서화됨).
