## 1. 범위·전제

- **대상**: 재무 관리 화면의 **기간(MONTH / CUSTOM)**, **거래 유형(`INCOME` / `EXPENSE`)**, **카테고리**, **검색어**, **페이지 이동** 조합과, 필터 변경 직후 **연속 API 호출 순서(레이스)** 회귀.
- **검증 축**: 수정 후 유형·카테고리 등이 **서버 요청에 반영**되면, **네트워크 탭의 요청 쿼리(또는 바디)**에 해당 파라미터가 포함되고, **응답 목록이 동일 필터 조건과 일치**하는지(클라만의 착시 제거).
- **통합·E2E**: `Authorization: Bearer`, `X-Tenant-ID` 필수 ([TESTING_STANDARD.md](../../standards/TESTING_STANDARD.md)).
- **테스트 데이터**: 동적 생성·테넌트 격리; 프로덕션 ID·하드코딩 테넌트 금지.

## 2. 검색어·클라이언트 필터 범위(명시)

검색 E2E·수동 스모크 시 **아래 두 층을 문서·코드와 동기화**해 한 번 고정한다(core-coder가 실제 파라미터 키·필드명으로 본 표를 보강).

| 구분 | 설명 | 검증 방법(요지) |
| --- | --- | --- |
| **서버 필터** | API 요청에 실려 백엔드가 WHERE/검색에 쓰는 조건(예: 거래 유형, 카테고리 ID, 기간, **서버 키워드 컬럼** 등) | 요청 URL/바디에 파라미터 존재 + 응답 각 행이 조건을 만족 |
| **클라이언트 필터 범위** | 응답 배열을 받은 뒤 **브라우저에서만** 추가로 좁히는 필드(예: 표시용 메모, 참여자 표시명, 태그 UI만 — **제품 스펙에 따라 확정**) | 동일 API 응답으로 **서버 조건만 만족하는 행**이 화면에 남는지 vs **클라 문자열**로 추가 제거되는지 네트워크+DOM 비교 |

**기대(수정 반영 후)**: “유형·카테고리가 서버에 반영된 상태”에서는 **반드시** 네트워크 쿼리에 해당 조건이 포함되고, 응답의 각 항목이 **서버 계약 필터**와 일치한다. 검색어가 서버만 타면 쿼리에 키워드 파라미터가 보여야 하고, 클라만 타면 요청은 불변이거나 키워드 미포함이어도 **행 수·내용이 클라 규칙과 일치**함을 별도 단언한다.

## 3. 필터·기간·유형·카테고리 시나리오

| ID | 시나리오 | Given | When | Then(기대) |
| --- | --- | --- | --- | --- |
| **FM-01** | MONTH 월 이동(다음 달) | 기본 MONTH 뷰, 해당 월에 거래 N건 존재 | 헤더/캘린더에서 **다음 달**로 이동 | 목록 API(또는 집계 API) 요청에 **이동 후 월에 해당하는 기간 파라미터**가 반영; 응답 거래일이 그 월 범위 내 |
| **FM-02** | MONTH 월 이동(이전 달) | FM-01 이후 또는 중간 월 | **이전 달**로 이동 | 동일하게 기간 파라미터·응답 범위 일치 |
| **FM-03** | `transactionType` = **INCOME** | 수입·지출 혼재 데이터 | 유형 필터에서 **수입** 선택 | 요청에 `INCOME`(또는 제품 enum 동등값) 포함; 응답 행이 모두 수입(불일치 행 0) |
| **FM-04** | `transactionType` = **EXPENSE** | 동일 | **지출** 선택 | 요청에 `EXPENSE` 포함; 응답 전부 지출 |
| **FM-05** | **카테고리** 단일 선택 | 복수 카테고리 거래 존재 | 특정 카테고리 선택 | 요청에 카테고리 식별자 포함; 응답의 카테고리가 선택값과 일치 |
| **FM-06** | **CUSTOM** 날짜 범위(정상) | MONTH가 아닌 사용자 지정 기간 UI | 시작일 ≤ 종료일인 범위 선택·적용 | `from`/`to`(또는 제품 키)가 요청에 포함; 응답 거래일 ∈ [시작, 종료] |
| **FM-07** | **검색어**(서버·클라 범위 §2 참조) | 검색어가 일부 행에만 매칭 | 검색어 입력 후 검색/디바운스 완료 | §2 표에 따라 **쿼리 포함 여부**와 **응답·화면 행**이 스펙과 일치 |
| **FM-08** | **페이지 2** 이동 | `totalCount`가 페이지 크기 초과 | 필터 고정 상태에서 **2페이지**로 이동 | 요청에 페이지 인덱스/오프셋(또는 cursor)이 **2페이지에 해당**; 응답 행이 1페이지와 중복 없음(정렬 동일 전제) |

## 4. 메타데이터·화면 일치(`totalPages` / `totalCount`)

| ID | 시나리오 | 기대 |
| --- | --- | --- |
| **FM-M-01** | 필터 없음 또는 기본 월 | API 응답의 `totalCount`·`totalPages`(또는 동등 메타)가 **페이지네이션 UI 숫자**·**현재 페이지 행 수 합산 가능 범위**와 일치 |
| **FM-M-02** | **검색·복합 필터** 적용 후 | `totalCount`가 **서버가 반환한 필터 적용 후 총건**과 같고, 현재 페이지 **실제 행 수** ≤ 페이지 크기; 마지막 페이지에서 `(page-1)*size + rows === totalCount` 관계가 성립(0건이면 `totalPages` 0 또는 1 — **제품 규약에 따름**) |

> E2E: 네트워크 응답 JSON의 메타와 화면의 “총 N건”·페이지 표시를 동시에 캡처해 비교. 단위·통합은 MockMvc/서비스 반환 DTO 필드 단언.
