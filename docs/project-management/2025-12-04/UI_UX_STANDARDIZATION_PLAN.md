# UI/UX 표준화 계획

**작성일**: 2025-12-04  
**작업**: Priority 4.3 - UI/UX 표준화  
**상태**: 진행 중

## 📋 개요

모든 리스트를 카드 형태로 전환하고, 반응형 레이아웃을 적용하며, 호버 효과를 표준화하여 UI 일관성을 향상시킵니다.

- [반응형 레이아웃 표준](../../standards/RESPONSIVE_LAYOUT_STANDARD.md)
- [버튼 디자인 표준](../../standards/BUTTON_DESIGN_STANDARD.md)

## 🎯 목표

1. **리스트 → 카드 형태 전환**
   - 테이블 형태 리스트 → 카드 그리드로 전환
   - 버튼 세로 배치 방지
   - 일관된 카드 디자인 적용

2. **반응형 레이아웃 적용**
   - 모바일/태블릿/데스크탑 대응
   - 반응형 그리드 시스템 적용
   - 브레이크포인트 표준 준수

3. **호버 효과 표준화**
   - 버튼 호버 효과 통일
   - 카드 호버 효과 적용
   - 일관된 인터랙션 제공

## 📊 현재 상태 분석

### 테이블 형태 리스트 사용 파일 (3개)

1. **`AccountTable.js`** - 계정 테이블
   - 테이블 형태로 계정 목록 표시
   - 카드 형태로 전환 필요

2. **`MenuPermissionManagementUI.js`** - 메뉴 권한 관리
   - 테이블 형태로 메뉴 권한 표시
   - 카드 형태로 전환 필요

3. **`TenantCommonCodeManagerUI.js`** - 테넌트 공통코드 관리
   - 테이블 형태로 공통코드 표시
   - 카드 형태로 전환 필요

### 이미 카드 형태로 전환된 파일

- ✅ `MappingManagement.js` - `MappingCard` 사용
- ✅ `ConsultantComprehensiveManagement.js` - 카드 형태 사용
- ✅ `ClientComprehensiveManagement.js` - 카드 형태 사용

- [ ] 버튼 가로 배치 확인

#### 2. MenuPermissionManagementUI.js

- [ ] 카드 컴포넌트 생성 또는 기존 카드 활용
- [ ] 테이블 → 카드 그리드 전환
- [ ] 버튼 가로 배치 확인

#### 3. TenantCommonCodeManagerUI.js
- [ ] 테이블 구조 분석
- [ ] 카드 컴포넌트 생성 또는 기존 카드 활용
- [ ] 테이블 → 카드 그리드 전환
- [ ] 버튼 가로 배치 확인

### Day 2: 반응형 레이아웃 적용

#### 모든 페이지 반응형 확인
- [ ] 브레이크포인트 표준 확인
- [ ] 모바일 레이아웃 검증
- [ ] 태블릿 레이아웃 검증
- [ ] 데스크탑 레이아웃 검증

#### 반응형 그리드 시스템 적용
- [ ] 카드 그리드 반응형 적용
- [ ] 모바일: 1열, 태블릿: 2열, 데스크탑: 3-4열
- [ ] 간격 및 패딩 반응형 조정

### Day 3: 호버 효과 표준화

- [ ] CSS 변수 사용 확인

- [ ] 카드 호버 효과 적용
- [ ] 그림자 증가, 약간 위로 이동
