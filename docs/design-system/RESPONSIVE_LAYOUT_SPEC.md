# MindGarden 반응형 레이아웃 스펙 (4K 지원)

**버전**: 1.0.0  
**최종 업데이트**: 2025-02-21  
**상태**: 디자인 스펙 (Design Spec)  
**참조**: 어드민 대시보드 샘플 (https://mindgarden.dev.core-solution.co.kr/admin-dashboard-sample)

---

## 📌 개요

MindGarden 아토믹 디자인 시스템의 **화면 레이아웃 기준**을 정의합니다. 4K(3840px)부터 모바일(375px)까지 모든 해상도를 지원하는 반응형 레이아웃 토큰과 컴포넌트 스펙을 포함합니다.

---

## 1. 해상도 브레이크포인트

| 브레이크포인트 | 최소 너비 | 설명 | 주요 디바이스 |
|---------------|-----------|------|---------------|
| **4K** | 3840px | 울트라와이드/4K 디스플레이 | 4K 모니터, 대형 TV |
| **2K (QHD)** | 2560px | 2K/WQHD 해상도 | 27"~34" 와이드 모니터 |
| **Full HD** | 1920px | FHD 해상도 | 24" 일반 모니터 |
| **데스크톱** | 1280px | 소형 데스크톱 | 노트북, 소형 모니터 |
| **태블릿** | 768px | 태블릿 세로/가로 | iPad, 태블릿 |
| **모바일** | 375px | 스마트폰 기준 | iPhone SE ~ 15 Pro Max |

### 브레이크포인트 토큰 (참조용)

```
--mg-breakpoint-mobile:    375px
--mg-breakpoint-tablet:    768px
--mg-breakpoint-desktop:   1280px
--mg-breakpoint-fhd:       1920px
--mg-breakpoint-2k:        2560px
--mg-breakpoint-4k:        3840px
```

## 3. 레이아웃 컴포넌트 스펙

### 3.1 사이드바 (Sidebar)

어드민 대시보드 샘플 기준: 고정 좌측, 다크 톤.

| 브레이크포인트 | 너비 | 동작 | 비고 |
|---------------|------|------|------|
| 4K, 2K, FHD, 데스크톱 | **260px** (고정) | 항상 표시 | 다크 #2C2C2C |
| 태블릿 | 260px | 오버레이/드로어 | 햄버거 토글 |
| 모바일 | 280px (전체) | 드로어 | 햄버거 토글 |

- **배경**: #2C2C2C (다크 그레이)
- **메뉴 항목 높이**: 44px
- **패딩**: 12~14px (좌우), gap 8px
- **활성 메뉴 배경**: #3D5246 (주조)
- **비활성 텍스트**: rgba(255,255,255,0.7)
- **border-radius**: 10px (메뉴 버튼)

### 3.2 상단 바 (Header / Top Bar)

| 브레이크포인트 | 높이 | 패딩 | 구성 |
|---------------|------|------|------|
| 모바일 | 56px | 16px | 햄버거 + 제목 + 액션 |
| 태블릿~4K | 64px | 24~32px | 브레드크럼 + 제목 + 액션 버튼 |

- **배경**: #FAF9F7
- **하단 구분선**: 1px #D4CFC8
- **액션 버튼 위치**: 우측 정렬

### 3.3 메인 콘텐츠 영역 (Main Content)

| 브레이크포인트 | 좌측 오프셋 | 패딩 |
|---------------|-------------|------|
| 데스크톱+ | 사이드바 260px 이후 | 24~40px |
| 태블릿/모바일 | 0 | 16~20px |

- **배경**: #FAF9F7 → #F2EDE8 (그라데이션)
- **스크롤**: 메인 영역만 스크롤, 헤더·사이드바 고정

### 3.4 섹션 블록 (Section Block)

모든 해상도 공통:

- **배경**: #F5F3EF 또는 #FAF9F7
- **테두리**: 1px #D4CFC8
- **corner-radius**: 16px
- **패딩**: 24px (기준), 브레이크포인트별 상단 표 참조
- **좌측 악센트 바**: 4px × 높이, #3D5246, radius 2px

## 4. 반응형 규칙 요약

### 4.1 브레이크포인트별 컬럼 수

| 화면 타입 | KPI/메트릭 카드 | 리스트/테이블 | 대시보드 그리드 |
|-----------|-----------------|---------------|-----------------|
| 모바일 | 1 | 1 | 1 |
| 태블릿 | 2 | 1 | 2 |
| 데스크톱 | 3~4 | 1 | 3~4 |
| Full HD | 4~6 | 1 | 4~6 |
| 2K | 6~8 | 1~2 | 6~8 |
| 4K | 8~12 | 1~2 | 8~12 |

### 4.2 폰트 스케일

| 브레이크포인트 | 본문(base) | 제목(h1) | 캡션 |
|---------------|------------|----------|------|
| 모바일 | 14px | 20px | 11px |
| 태블릿 | 14px | 22px | 12px |
| 데스크톱 | 15px | 24px | 12px |
| Full HD+ | 16px | 24px | 12px |
| 4K | 16px (1.05x) | 26px | 13px |

*4K에서 가독성을 위해 소폭 스케일 업 가능.*

### 4.3 여백 조정

- **모바일**: 최소 터치 영역 44×44px 유지
- **태블릿**: 터치/마우스 혼용, 40px 이상 간격 권장
- **데스크톱+**: 24px 이상 여백으로 시각적 호흡 확보
- **4K**: 콘텐츠 밀집 방지를 위해 gap·padding 20~30% 확대

## 5. Pencil(.pen) 레이아웃 프레임 명세

`mindgarden-design-system.pen`에 추가할 **레이아웃 스펙 프레임** 구조입니다.

### 5.1 추가할 최상위 섹션: "Layout & Responsive"

기존 "MindGarden Atomic Design System" 프레임의 `children` 배열에 다음 프레임을 추가합니다.

#### 5.1.1 브레이크포인트 표 프레임

- **이름**: `breakpointsTable`
- **구성**: 6행 (4K, 2K, FHD, 데스크톱, 태블릿, 모바일) × 4열 (브레이크포인트명, 최소 너비, 컨테이너 max-width, 패딩)
- **스타일**: 기존 atoms 섹션과 동일 (배경 #ffffff, cornerRadius 24, stroke #e2e8f0)
- **타이포**: Noto Sans KR, 제목 24px/600, 본문 14px

#### 5.1.2 레이아웃 구조 다이어그램 프레임

- **이름**: `layoutStructure`
- **구성**:
  - 좌측: 사이드바 블록 (260px × 600px, #2C2C2C)
  - 우측: 메인 영역
    - 상단 바 (full width × 64px, #FAF9F7)
    - 본문 (flex, #F5F3EF, 섹션 블록 2개 예시)
- **라벨**: "Sidebar 260px" | "Header 64px" | "Main Content"

#### 5.1.3 토큰 요약 프레임

- **이름**: `layoutTokens`
- **구성**:
  - `Container`: max-width 375~1920
  - `Padding`: 16~40
  - `Gap`: 12~40
  - `Sidebar`: 260
  - `Header`: 56~64

### 5.2 .pen 내 위치

```
MindGarden Atomic Design System
├── titleRow
├── atoms (Colors, Icons, …)
├── molecules
├── organisms
├── templates
└── [NEW] layout-responsive     ← 여기에 추가
    ├── breakpointsTable
    ├── layoutStructure
    └── layoutTokens
```
