# 온보딩 vs 코어 솔루션 역할 분담

**작성일**: 2025-11-24  
**목적**: 온보딩 플로우와 코어 솔루션의 역할을 명확히 구분

---

## 🎯 핵심 원칙

### 온보딩 (Trinity 홈페이지)
**역할**: **초기 설정 및 구성** (Setup & Configuration)
- 테넌트 최초 생성 시 필요한 모든 설정을 한 번에 완료
- 사용자가 처음 접하는 설정 화면
- 간편하고 직관적인 UI로 빠르게 완료 가능
- **애니메이션과 안내 메시지로 사용자 경험 최적화**

### 코어 솔루션 (MindGarden 앱)
**역할**: **컨텐츠 및 비즈니스 운영** (Content & Business Operations)
- 실제 비즈니스 컨텐츠 관리 (상담, 수업, 결제 등)
- 일상적인 업무 처리
- 설정 변경은 최소화, 컨텐츠 중심

---

### 1. 대시보드 위젯 설정

- 템플릿 기반 빠른 설정
- 역할별 기본 위젯 자동 구성

#### 코어 솔루션에서 처리 ✅
- **대시보드 위젯 수정/추가/삭제**
- 드래그 앤 드롭으로 레이아웃 변경
- 위젯 설정 상세 조정

**구현 위치**:

## 💡 구현 전략

### 1. Step 4: 결제 및 PG 설정 확장

**현재 상태**:
- `Step4Payment.tsx`는 카드 등록만 지원
- PG 사 등록 기능 없음
- 현재 숨김 처리 (`false && step === 4`)

**확장 필요 사항**:
1. **PG 사 선택 UI**
   - 드롭다운 또는 카드 형태로 PG 사 선택
   - 지원 PG: TOSS, IAMPORT, KAKAO, NAVER, PAYPAL, STRIPE 등

2. **PG 설정 정보 입력**
   - API Key 입력
   - Secret Key 입력 (암호화)
   - Merchant ID 입력
   - 기타 PG별 필수 정보

3. **PG 설정 관리**
   - 등록된 PG 설정 목록 표시
   - PG 설정 변경
   - PG 설정 삭제
   - 기본 PG 설정 선택

4. **결제 수단 관리**
   - 결제 수단 등록 (기존 기능)
   - 결제 수단 변경
   - 결제 수단 삭제

**파일 수정**:
- `frontend-trinity/components/onboarding/Step4Payment.tsx` (확장)
- `frontend-trinity/app/onboarding/page.tsx` (Step 4 활성화)

### 3. 코어 솔루션 역할 재정의

**코어 솔루션은 컨텐츠 위주로 운영**:
- 대시보드 위젯 편집은 있지만, **주 목적은 컨텐츠 관리**
- 설정 변경은 최소화
- 실제 비즈니스 업무에 집중

**코어 솔루션에서 제공하는 설정 기능**:
1. **대시보드 위젯 편집** (고급)
   - 드래그 앤 드롭 레이아웃 편집
   - 위젯 상세 설정
   - 운영 중 필요에 따른 변경

2. **결제 및 PG 관리**
   - 결제 수단 변경/삭제
   - PG 설정 변경/삭제
   - 결제 내역 조회

3. **요금제 관리**
   - 요금제 변경
   - 구독 관리

4. **기본 정보 수정** (제한적)
   - 회사 정보 수정
   - 담당자 정보 수정

## 🎨 사용자 경험 설계

### 온보딩 플로우
**목표**: **3-5분 내 모든 초기 설정 완료**

1. **간편함**
   - 템플릿 기반 빠른 선택
   - 최소한의 입력만 요구
   - 복잡한 설정은 나중으로

2. **안내 메시지**
   - 각 단계별 명확한 설명
   - "이것이 무엇인가요?" 도움말
   - 진행 상태 표시

3. **애니메이션**
   - 부드러운 전환 효과
   - 시각적 피드백
   - 설정 완료 시 축하 애니메이션

4. **유연성**
   - 건너뛰기 옵션 (나중에 설정)
   - 변경 가능 (온보딩 완료 전까지)

### 코어 솔루션
**목표**: **컨텐츠 중심의 효율적인 업무 처리**

1. **컨텐츠 우선**
   - 대시보드에 컨텐츠 중심 위젯
   - 설정 메뉴는 최소화
   - 빠른 접근성

2. **설정은 필요 시에만**
   - 고급 설정은 숨김 처리
   - 자주 사용하는 설정만 노출
   - 설정 변경은 최소화

## 🎯 결론

### 온보딩의 역할
- **초기 설정의 모든 것**: 기본 정보, 업종, 요금제, 결제/PG, 대시보드 위젯
- **간편하고 직관적인 UI**: 템플릿 기반, 애니메이션, 안내 메시지
- **빠른 완료**: 3-5분 내 모든 설정 완료

### 코어 솔루션의 역할
- **컨텐츠 및 비즈니스 운영**: 실제 업무 처리에 집중
- **설정은 최소화**: 필요 시에만 변경 가능
- **고급 기능**: 드래그 앤 드롭 위젯 편집 등

### 사용자 플로우
```
온보딩 (Trinity 홈페이지)
  → 초기 설정 완료 (3-5분)
  → 온보딩 승인 대기
  → 승인 완료
  ↓
코어 솔루션 (MindGarden 앱)
  → 컨텐츠 관리 및 업무 처리
  → 필요 시 설정 변경
```
