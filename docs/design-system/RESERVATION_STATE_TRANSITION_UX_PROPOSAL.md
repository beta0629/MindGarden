# 예약·결제 상태 전환 UX 제안 (1페이지)

**버전**: 1.0  
**작성**: core-designer (디자인 제안만, 코드 미포함)  
**참조**: B0KlA·어드민 대시보드 샘플, `PENCIL_DESIGN_GUIDE.md`, `unified-design-tokens.css`, core-solution-design-handoff

---

## 1. 개요·비즈니스 맥락

- **예약금 50% 수령** 후 → **결재확인(예약대기)** →  
  **경로 A**: 상담완료 → 전액 수령 → ERP 등록  
  **경로 B**: 미참석·취소 → 예약금 환불
- **상태 후보**: 예약대기, 결재확인, 상담완료, 전액수령, ERP등록완료, 환불완료 (논의 중).
- **대상 사용자**: 관리자·상담사 — 상담/예약 상세(또는 카드·리스트)에서 **상태를 전환**할 때의 버튼·액션 배치 및 사용자 안내 문구.

## 2. 상태 전환 흐름 요약

| 현재 상태     | 가능한 다음 상태     | 관리자/상담사 액션       |
|--------------|----------------------|--------------------------|
| 예약대기     | 결재확인             | "결재 확인"              |
| 결재확인     | 상담완료, 환불완료   | "상담 완료", "예약금 환불" |
| 상담완료     | 전액수령             | "전액 수령 확인"         |
| 전액수령     | ERP등록완료          | "ERP 등록"              |
| ERP등록완료  | —                    | (전환 없음)              |
| 환불완료     | —                    | (전환 없음)              |

## 3. 버튼·액션 배치 제안 (1화면 분량)

### 3.1 배치 원칙

- **위치**: 예약/상담 **상세 영역** 또는 **카드 푸터**에, 현재 상태에 따라 **가능한 전환 액션만** 노출.
- **레이아웃**: 어드민 대시보드 샘플·B0KlA 기준 — **섹션 블록** 내부, 블록 하단 또는 카드 액션 영역.  
  - 배경 `var(--mg-color-surface-main)`, 테두리 `var(--mg-color-border-main)`, radius 16px, 패딩 24px.  
  - 섹션 제목: 좌측 세로 악센트 4px `var(--mg-color-primary-main)` + 제목 16px 600.
- **버튼 스타일** (토큰만 명시, 구현은 코더):  
  - 주 액션(결재 확인, 상담 완료, 전액 수령 확인, ERP 등록): `mg-v2-button mg-v2-button-primary` (배경 `var(--mg-color-primary-main)`).  
  - 보조/취소성(닫기 등): `mg-v2-button mg-v2-button-secondary`.  
  - 위험(환불): `mg-v2-button mg-v2-button-danger`.  
- **순서**: 보조(취소/닫기) → 주 액션; **환불**은 별도 줄 또는 우측 끝에 배치해 실수 방지.

### 3.2 상태별 노출 버튼 (한 화면에 하나의 예약/상담 기준)

| 현재 상태     | 노출 버튼 (라벨)                    | 비고 |
|--------------|-------------------------------------|------|
| 예약대기     | [결재 확인]                         | 1개  |
| 결재확인     | [상담 완료] [예약금 환불]           | 2개, 환불은 danger |
| 상담완료     | [전액 수령 확인]                    | 1개  |
| 전액수령     | [ERP 등록]                          | 1개  |
| ERP등록완료  | —                                   | 상태 뱃지만 표시  |
| 환불완료     | —                                   | 상태 뱃지만 표시  |

- **상태 표시**: 카드 상단 또는 제목 옆에 **상태 뱃지** (예: 예약대기, 결재확인, …).  
  토큰: `var(--mg-color-text-secondary)` 배경/테두리, 주조 상태는 `var(--mg-color-primary-main)` 계열 사용 가능.

## 4. 상태별 안내 문구 제안 (사용자에게 보여줄 메시지)

- **예약대기**  
  - 요약: "예약금이 입금되면 결재 확인을 진행해 주세요."  
  - (선택) 상세: "입금 확인 후 [결재 확인] 버튼을 눌러 예약을 확정할 수 있습니다."

- **결재확인**  
  - 요약: "상담을 진행하시거나, 취소 시 예약금 환불을 진행할 수 있습니다."  
  - (선택) 상세: "상담 완료 후 전액 수령·ERP 등록으로 이어지거나, 미참석·취소 시 [예약금 환불]을 선택하세요."

- **상담완료**  
  - 요약: "잔금 수령 후 [전액 수령 확인]을 눌러 주세요."  
  - (선택) 상세: "전액 수령 확인 후 ERP 등록이 가능합니다."

- **전액수령**  
  - 요약: "ERP에 매출을 반영하려면 [ERP 등록]을 진행해 주세요."  
  - (선택) 상세: "등록 완료 후 정산·보고서에 반영됩니다."

- **ERP등록완료**  
  - 요약: "이 예약의 정산 처리가 완료되었습니다."  
  - (선택) 상세: "추가 전환은 없습니다. ERP에서 확인하세요."

- **환불완료**  
  - 요약: "예약금 환불이 완료되었습니다."  
  - (선택) 상세: "취소·환불 처리된 예약입니다."
