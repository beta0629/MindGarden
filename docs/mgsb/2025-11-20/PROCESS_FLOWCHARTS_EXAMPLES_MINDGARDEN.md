# MindGarden 예시 프로세스 플로우차트

**작성일**: 2025-11-20  
**버전**: 1.0.0  
**상태**: 완료

---

## 📋 개요

MindGarden은 CONSULTATION 업종의 실제 구현 예시입니다. 이 문서는 MindGarden의 주요 프로세스를 플로우차트로 정리한 것입니다.

**⚠️ 참고**: MindGarden의 프로세스는 CONSULTATION 업종의 예시일 뿐이며, 다른 상담소는 다른 프로세스를 가질 수 있습니다.

---

## 👤 역할별 프로세스

### 1. 내담자 (CLIENT) 프로세스

#### 상담 예약 프로세스

```mermaid
flowchart TD
    A[대시보드 접근] --> B[상담 예약 메뉴]
    B --> C[상담사 목록 조회]
    C --> D[상담사 선택]
    D --> E[상담사 상세 확인]
    E --> F[일정 선택]
    F --> G{일정 가능?}
    G -->|아니오| H[다른 일정 선택]
    G -->|예| I[예약 정보 입력]
    H --> F
    I --> J[상담 목적 입력]
    J --> K[특이사항 입력]
    K --> L[예약 확인]
    L --> M[결제 필요?]
    M -->|예| N[결제 처리]
    M -->|아니오| O[예약 저장]
    N --> P{결제 성공?}
    P -->|실패| Q[결제 실패 처리]
    P -->|성공| O
    O --> R[예약 확인 알림]
    Q --> I
```

#### 상담 내역 조회 프로세스

```mermaid
flowchart TD
    A[대시보드 접근] --> B[상담 내역 메뉴]
    B --> C{내역 타입}
    C -->|예정된 상담| D[예정된 상담 목록]
    C -->|완료된 상담| E[완료된 상담 목록]
    C -->|취소된 상담| F[취소된 상담 목록]
    D --> G[상담 상세 조회]
    E --> G
    F --> G
    G --> H[상담 정보 표시]
    H --> I[상담 메시지 조회]
    I --> J[메시지 목록 표시]
```

### 2. 상담사 (CONSULTANT) 프로세스

#### 일정 관리 프로세스

```mermaid
flowchart TD
    A[대시보드 접근] --> B[일정 관리 메뉴]
    B --> C{작업 타입}
    C -->|조회| D[일정 목록 조회]
    C -->|추가| E[일정 추가]
    C -->|수정| F[일정 수정]
    C -->|삭제| G[일정 삭제]
    D --> H[일정 목록 표시]
    E --> I[일정 정보 입력]
    I --> J[날짜/시간 선택]
    J --> K[일정 유형 선택]
    K --> L[반복 일정 설정]
    L --> M[시간 충돌 검사]
    M --> N{충돌 있음?}
    N -->|예| O[에러 메시지]
    N -->|아니오| P[일정 저장]
    F --> Q[일정 정보 수정]
    Q --> M
    G --> R[일정 삭제 확인]
    R --> S{확인}
    S -->|예| T[일정 삭제]
    S -->|아니오| U[취소]
    P --> V[알림 발송]
    T --> V
    O --> I
```

#### 상담 기록 작성 프로세스

```mermaid
flowchart TD
    A[대시보드 접근] --> B[오늘의 일정]
    B --> C[상담 선택]
    C --> D[상담 시작]
    D --> E[상담 진행]
    E --> F[상담 메시지 입력]
    F --> G[상담 기록 작성]
    G --> H[상담 요약 입력]
    H --> I[상담 기록 저장]
    I --> J[공개 여부 설정]
    J --> K[상담 완료]
    K --> L[알림 발송]
```

### 3. 관리자 (ADMIN) 프로세스

#### 사용자 관리 프로세스

```mermaid
flowchart TD
    A[대시보드 접근] --> B[사용자 관리 메뉴]
    B --> C{사용자 타입}
    C -->|상담사| D[상담사 관리]
    C -->|내담자| E[내담자 관리]
    C -->|전체| F[전체 사용자 관리]
    D --> G{작업 타입}
    E --> G
    F --> G
    G -->|조회| H[사용자 목록 조회]
    G -->|추가| I[사용자 추가]
    G -->|수정| J[사용자 수정]
    G -->|삭제| K[사용자 삭제]
    H --> L[사용자 목록 표시]
    I --> M[사용자 정보 입력]
    M --> N[역할 할당]
    N --> O[지점 할당]
    O --> P[사용자 저장]
    J --> Q[사용자 정보 수정]
    Q --> N
    K --> R[사용자 삭제 확인]
    R --> S{확인}
    S -->|예| T[사용자 삭제]
    S -->|아니오| U[취소]
    P --> V[알림 발송]
    T --> W[관련 데이터 정리]
```

#### 통계 조회 프로세스

```mermaid
flowchart TD
    A[대시보드 접근] --> B[통계 메뉴]
    B --> C[기간 선택]
    C --> D{통계 카테고리}
    D -->|전체 통계| E[전체 통계 조회]
    D -->|상담 통계| F[상담 통계 조회]
    D -->|리포트| G[리포트 조회]
    E --> H[통계 데이터 집계]
    F --> H
    G --> H
    H --> I[차트/그래프 생성]
    I --> J[통계 표시]
    J --> K{리포트 다운로드?}
    K -->|예| L[리포트 생성]
    K -->|아니오| M[완료]
    L --> N[리포트 다운로드]
```

## 🔄 주요 프로세스 상세

### 상담 예약 → 상담 진행 → 상담 완료 플로우

```mermaid
flowchart TD
    A[내담자: 상담 예약] --> B[상담사: 예약 확인]
    B --> C[상담 일시 도래]
    C --> D[상담사: 상담 시작]
    D --> E[상담 진행]
    E --> F[상담사: 상담 기록 작성]
    F --> G[상담 완료]
    G --> H[내담자: 상담 내역 확인]
    H --> I[상담사: 상담 기록 공개]
    I --> J[내담자: 상담 기록 확인]
```

### 매핑 관리 프로세스

```mermaid
flowchart TD
    A[관리자: 매핑 관리 메뉴] --> B[상담사-내담자 매핑]
    B --> C[매핑 추가]
    C --> D[상담사 선택]
    D --> E[내담자 선택]
    E --> F[회기 수 설정]
    F --> G[매핑 저장]
    G --> H[매핑 승인]
    H --> I[알림 발송]
```

## ⚠️ 중요 사항

1. **MindGarden은 예시일 뿐**
   - MindGarden의 프로세스는 CONSULTATION 업종의 예시
   - 다른 상담소는 다른 프로세스를 가질 수 있음

2. **동적 프로세스**
   - 프로세스는 역할별로 동적으로 구성
   - 테넌트 관리자가 프로세스 커스터마이징 가능

3. **업종별 차이**
   - 학원: 강좌 신청, 출석 체크, 성적 조회
   - 상담소: 상담 예약, 상담 기록, 결제
   - 기타: 업종별 다른 프로세스
