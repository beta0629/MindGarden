# 급여·세금 자동 계산 — 한 번에 계산·미리보기·확정 UX 제안

**버전**: 1.0.0  
**작성일**: 2025-03-16  
**담당**: core-designer  
**참조**: `docs/project-management/SALARY_TAX_INTEGRATION_SURVEY.md`, `docs/debug/SALARY_TAX_LOGIC_ANALYSIS.md`, `PENCIL_DESIGN_GUIDE.md`, `SALARY_MANAGEMENT_LAYOUT_DESIGN.md`, 어드민 대시보드 샘플 https://mindgarden.dev.core-solution.co.kr/admin-dashboard-sample  
**범위**: 사용성·정보 노출·레이아웃 제안만. 코드·구현 상세 없음.

---

## 1. 목표

- 관리자가 급여를 **한 번에** 자동 계산·미리보기·확정할 수 있는 플로우를 제안한다.
- 기간·등급·회기 반영, **최소 클릭**으로 진입 → 미리보기 → 확정까지 이어지도록 한다.
- 자동 계산 결과(총급여·공제·세금·실지급액), 세금 항목별(원천징수/지방소득세/부가세/4대보험) 표시, **실제 계산 기간**(기산일 기준)을 명확히 노출한다.
- B0KlA·AdminCommonLayout·`unified-design-tokens.css` 기준으로 1페이지(또는 1개 플로우) 내 완결을 제안한다.

---

## 2. 플로우 단계

| 단계 | 설명 | 사용자 동작 | 화면 위치 |
|------|------|-------------|-----------|
| **1. 진입** | 급여 관리 메인에서 “한 번에 계산” 진입 | 메인 상단 또는 필터 영역의 **주요 CTA** 클릭 | ContentHeader 우측 또는 필터 블록 내 1차 버튼 |
| **2. 기간·대상 선택** | 계산할 월(또는 기간)과 상담사(1명 또는 전체) 선택 | 월 선택(드롭다운), 상담사 선택(드롭다운/전체) | 본문 상단 필터 블록 또는 인라인 폼 |
| **3. 자동 계산** | 선택에 따라 미리보기 API 호출(저장 없음) | “계산하기” 버튼 1회 클릭 | 동일 블록 내 버튼 |
| **4. 미리보기** | 총급여·공제·세금·실지급액, 세금 항목별, 실제 계산 기간 표시 | 결과 카드/테이블 확인, 필요 시 기간·대상 변경 후 재계산 | 본문 섹션 블록(카드 + 테이블) |
| **5. 확정** | 계산 결과를 저장하고(선택 시 ERP 연동) 상태를 확정으로 전환 | “확정” 버튼 1회 클릭 | 미리보기 블록 하단 또는 ContentHeader 우측 |

- **1페이지 완결**: 진입 → 선택 → 계산 → 미리보기 → 확정을 **같은 페이지(급여 관리 메인)** 에서 순차적으로 진행. 탭 전환 없이, 필요 시 모달은 “기산일 기준 기간 설명” 등 보조 정보만 사용.
- **대안(모달 플로우)**: “한 번에 계산” 클릭 시 **모달**을 띄워 모달 내에서 2→3→4→5를 진행하고, 확정 후 모달 닫기. 메인에는 “최근 계산 결과” 요약만 남길 수 있음. 제안은 **메인 1페이지 플로우**를 우선하고, 공간이 부족하면 모달 플로우를 권장.

## 3. 화면별 배치

### 3.1 상단 (ContentHeader)

- **좌**: 브레드크럼(ERP > 급여 관리), 제목 “급여 관리”, 부제(선택).
- **우**: [급여 기산일 설정], [기간 선택 드롭다운], **「한 번에 계산」** 버튼(주조색, `var(--mg-color-primary-main)`).
  - “한 번에 계산”이 이 플로우의 **진입점**이 되도록 시인성 확보.

### 3.2 본문 — 필터·제어 영역 (섹션 블록 1)

- **블록 제목**: 세로 악센트(4px, `var(--mg-color-primary-main)`) + “계산 대상 선택”.
- **내용**:  
  - **기간**: 월 단위 선택 1개(예: 2025-03). API/공통코드 기반 동적 목록(최근 12개월 등).  
  - **대상**: 상담사 단일 선택 또는 “전체” 선택.  
  - **실제 계산 기간** 표시(기산일 기준): 예) “2025-02-26 ~ 2025-03-25”(라벨 12px, `var(--mg-color-text-secondary)`). 백엔드 `getCalculationPeriod` 또는 동일 로직 API로 조회해 표시.  
  - **버튼**: “계산하기”(주조 버튼, `mg-v2-button` / `mg-v2-btn--primary`).
- **레이아웃**: 가로 배치(데스크톱), 좁을 때 세로 스택. gap 16px, 패딩 24px.

### 3.3 본문 — 미리보기 영역 (섹션 블록 2)

- **표시 조건**: “계산하기” 실행 후 결과가 있을 때만 표시.
- **블록 제목**: 세로 악센트 + “계산 결과 미리보기”.
- **상단 요약 카드(4열 또는 2×2)**  
  - 총 급여(총급여), 공제 합계, 세금 합계, 실지급액.  
  - 숫자 24px / fontWeight 600, `var(--mg-color-text-main)`. 라벨 12px, `var(--mg-color-text-secondary)`.  
  - 카드 왼쪽 세로 악센트(4px)로 구분(주조/보조/포인트 등).
- **세금 항목별 행**  
  - 원천징수, 지방소득세, 부가세, 4대보험(국민연금·건강·장기요양·고용) 항목별 금액 표시.  
  - 테이블 또는 정의 리스트. 헤더/라벨 12px, 값 14~16px.  
  - *(백엔드 세목별 breakdown 제공 시 표시; 미제공 시 “세금 합계”만 표시하고, 항목별는 “준비 중” 등 처리.)*
- **실제 계산 기간**  
  - 미리보기 블록 상단 또는 요약 카드 바로 아래에 “적용 기간: YYYY-MM-DD ~ YYYY-MM-DD(기산일 기준)” 문구로 노출.
- **확정 버튼**  
  - 블록 하단 우측: “확정”(주조). 클릭 시 저장·상태 변경(및 선택 시 ERP 연동). 취소/다시 계산은 아웃라인 또는 텍스트 버튼.

### 3.4 모달(선택)

- **용도**: “기산일이란?” 등 도움말, 또는 기간/대상 선택을 모달 안에 두는 대안 플로우.
- **스타일**: `UnifiedModal` + B0KlA. 배경 `var(--mg-color-surface-main)`, 테두리 `var(--mg-color-border-main)`, radius 16px. 제목 20px/600.

## 4. 표시할 필드(급여·세금·기간)

| 구분 | 필드 | 비고 |
|------|------|------|
| **요약** | 총 급여(총급여), 공제 합계, 세금 합계, 실지급액 | 미리보기 API(gross/net/tax 등) 반환값 매핑 |
| **세금 항목별** | 원천징수, 지방소득세, 부가세, 국민연금, 건강보험, 장기요양, 고용보험 | 백엔드 breakdown 제공 시 표시(현재 갭 있음) |
| **기간** | 적용(실제 계산) 기간: YYYY-MM-DD ~ YYYY-MM-DD (기산일 기준) | `getCalculationPeriod` 또는 동일 API로 노출 |
| **대상** | 선택한 상담사명 또는 “전체” | 필터 선택 값 표시 |
| **상태** | 미리보기 중 / 확정됨 등 | 확정 후 메인 목록과 일치하도록 표시 |

## 5. 디자인 토큰·B0KlA 참조

- **색상**: `var(--mg-color-background-main)`, `var(--mg-color-primary-main)`, `var(--mg-color-text-main)`, `var(--mg-color-text-secondary)`, `var(--mg-color-surface-main)`, `var(--mg-color-border-main)` (PENCIL_DESIGN_GUIDE §2.1).
- **레이아웃**: 사이드바 260px, 상단 바 56~64px, 본문 패딩 24~32px. 섹션 블록: 배경 surface, 테두리 1px, radius 16px, 좌측 세로 악센트 4px.
- **타이포**: Noto Sans KR. 제목 20~24px/600, 본문 14~16px, 라벨/캡션 12px.
- **버튼**: 주조 `mg-v2-button` / `mg-v2-btn--primary`(height 40px, radius 10px), 아웃라인 `mg-v2-button-outline`.
- **참조**: `mindgarden-design-system.pen`, `pencil-new.pen`, `unified-design-tokens.css`, `AdminCommonLayout`, `docs/design-system/SALARY_MANAGEMENT_LAYOUT_DESIGN.md`.
