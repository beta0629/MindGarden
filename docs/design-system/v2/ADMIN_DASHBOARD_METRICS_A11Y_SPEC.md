# 관리자 대시보드 지표 영역 — 접근성(A11y) 상세 스펙

**버전**: 1.0  
**상태**: 디자인/접근성 스펙 (코드 작성 없음)  
**참조**: `ADMIN_DASHBOARD_METRICS_TEST_PROPOSAL.md` 3장, `ADMIN_DASHBOARD_METRICS_OPTION_A_SPEC.md`, `ADMIN_DASHBOARD_PIPELINE_MARKUP_SPEC.md`, `PENCIL_DESIGN_GUIDE.md`, `unified-design-tokens.css`

---

## 1. 개요

- **대상**: 관리자 대시보드 지표 영역 — KPI 카드, CoreFlowPipeline(5단계), 파이프라인 단계 카드·배지·숫자, 로딩/에러 상태.
- **목적**: 스크린 리더·키보드·포커스·동적 콘텐츠에 대한 **접근성 요구사항을 한 문서로 정리**하여 코더가 구현 시 추측 없이 적용할 수 있도록 한다.
- **코드 작성 없음**: 본 문서는 스펙·문구 예시·토큰명만 포함한다.

---

## 2. 스크롤 영역 안내 (core-flow-pipeline__steps)

### 2.1 역할·시맨틱

- **대상**: 가로 스크롤이 적용된 영역 `core-flow-pipeline__steps`.
- **역할**: `role="region"` 사용. 스크린 리더가 “영역”으로 인식하고, 사용자가 해당 영역으로 이동해 콘텐츠를 탐색할 수 있도록 한다.
- **레이블**: `aria-label`로 스크롤 가능 여부와 목적을 함께 전달.

### 2.2 aria-label 문구 예시

| 언어/상황 | aria-label 값 (예시) |
|-----------|----------------------|
| 한글 (권장) | `"파이프라인 단계 목록, 가로 스크롤 가능"` |
| 대안 (한글) | `"5단계 파이프라인, 좌우로 스크롤하여 단계 확인"` |
| 영문 (필요 시) | `"Pipeline steps list, horizontally scrollable"` |

- **규칙**: 한 가지 문구로 팀 내 통일. 위 표에서 선택한 문구를 마크업 스펙(`ADMIN_DASHBOARD_PIPELINE_MARKUP_SPEC.md`)과 동일하게 적용한다.
- **상위 섹션**: `core-flow-pipeline`에는 `aria-label="5단계 핵심 파이프라인"` 유지. 스크롤 영역(`core-flow-pipeline__steps`)과 상위 섹션이 각각 다른 목적(전체 섹션 vs 스크롤 가능 목록)으로 읽히도록 구분한다.

### 2.3 스크린 리더 안내 요약

- 진입 시: “파이프라인 단계 목록, 가로 스크롤 가능” 등으로 **가로 스크롤 가능**함을 반드시 안내.
- 좁은 뷰포트(태블릿·모바일)에서만 스크롤이 활성화되더라도, 마크업은 **항상 동일**하게 적용(스크린 리더는 뷰포트와 무관하게 동일 DOM을 읽음).

## 3. 키보드·포커스

### 3.1 포커스 순서 (Tab order)

지표 영역 내 포커스 가능 요소가 있을 때, 다음 순서를 권장한다.

| 순서 | 영역 | 설명 |
|------|------|------|
| 1 | 페이지 헤더 | 브레드크럼, 제목, 상단 액션 버튼 |
| 2 | KPI 카드 영역 | `mg-v2-ad-b0kla__kpi-card` 등 — 클릭 가능한 카드가 있으면 카드 단위로 포커스 |
| 3 | 파이프라인 섹션 | `core-flow-pipeline` 내부 — 단계 카드가 포커스 가능한 경우 **단계 1 → … → 단계 5** 순서 |
| 4 | (해당 시) 스크롤 영역 | `core-flow-pipeline__steps`가 포커스 받을 수 있는 구조라면, 카드들 다음에 스크롤 컨테이너 |
| 5 | 차트·토글·상담사 등 | 차트 영역 토글(월간/주간), 상담사 목록, 하단 대기열 등 |

- **원칙**: DOM 순서와 시각적 배치가 일치하도록 마크업 순서를 조정한다. `tabIndex`로 순서를 바꾸는 것은 예외적으로만 사용.

### 3.2 파이프라인 단계 카드 포커스

- **포커스 가능한 경우**: 단계 카드가 링크/버튼으로 동작(클릭 시 해당 업무 화면으로 이동)할 때는 **카드 전체가 하나의 포커스 대상**이 되도록 한다. 각 카드에 `tabindex="0"` 또는 `<a>`, `<button>` 적용.
- **포커스 불가능한 경우**: 정보만 표시하고 상호작용이 없으면 포커스 순서에서 제외 가능. 이때는 섹션·스크롤 영역의 `aria-label`로 전체 맥락만 전달.

### 3.3 포커스 링 스타일 (B0KlA 토큰)

- **적용 대상**: 지표 영역 내 포커스 가능 요소 — KPI 카드, 파이프라인 단계 카드, 버튼, 토글 등.
- **선택자**: `:focus-visible` 사용. 마우스 클릭 시에는 링을 숨기고, 키보드 포커스 시에만 링 표시(기존 프로젝트 패턴과 동일).
- **토큰 (unified-design-tokens.css 기준)**  
  - 링 색: `var(--mg-focus)` 또는 `var(--mg-primary-500)` (B0KlA 주조와 일치).  
  - 링 두께: `var(--focus-ring-width)` (2px).  
  - 오프셋: `var(--focus-ring-offset)` (2px).  
- **구현 예시 (스펙 수준)**:  
  - `outline: var(--focus-ring-width) solid var(--mg-focus);`  
  - `outline-offset: var(--focus-ring-offset);`  
  - 또는 `box-shadow: 0 0 0 var(--focus-ring-width) var(--mg-focus);` (border-radius가 있는 카드에 자연스럽게 적용되도록).
- **대비**: 포커스 링이 배경과 구분되도록, 필요 시 `--mg-focus`가 주조색(`--ad-b0kla-*` 계열)과 동일한 톤임을 확인한다. `unified-design-tokens.css`의 `--mg-focus`는 이미 `--mg-primary-500`으로 정의된 경우가 많음.

### 3.4 Skip link 필요 여부

- **현황**: 지표 영역은 대시보드 본문 상단에 위치하므로, “본문으로 건너뛰기” 같은 **페이지 레벨 skip link**만으로도 첫 포커스가 헤더 → 본문(지표) 순으로 들어갈 수 있음.
- **권장**:  
  - **지표 영역 전용 skip link는 필수 아님.** 페이지에 이미 “메인 콘텐츠로 건너뛰기”가 있고, 그 대상이 지표+KPI+파이프라인을 포함한 본문 블록이면 충분.  
  - **필요 시**: 본문이 매우 길고 “지표 블록”과 “하단 테이블/차트”가 명확히 구분되는 구조라면, “지표 영역으로 건너뛰기” 링크를 추가할 수 있음. 이 경우 `id`로 대상 영역을 지정하고, skip link의 `href="#id"`로 연결.
- **결론**: 기본은 **페이지 단 skip link만 유지**, 지표 전용 skip link는 선택 사항으로 둔다.

## 4. 배지·숫자 (PipelineStepBadge, KPI 숫자)

### 4.1 PipelineStepBadge

- **역할**: 단계별 “숫자 + 상태 라벨”(예: 3건, 매칭됨)을 한 단위로 스크린 리더에 전달.
- **권장**: 배지 루트 요소에 **`aria-label`** 로 “값 + 라벨”을 한 문장으로 제공.  
  - 예: `aria-label="3건 매칭됨"`, `aria-label="2건 대기중"`, `aria-label="0건 의견수렴중"`.
- **형식**: `"{숫자}건 {상태라벨}"` 또는 값이 숫자가 아닌 경우(예: “배치/일지작성”) `"{값} {라벨}"`.
- **내부 텍스트**: `pipeline-step-badge__value`, `pipeline-step-badge__label`은 시각적으로만 사용하고, 스크린 리더가 중복 읽지 않도록 **배지 컨테이너에만** `aria-label`을 두고, 내부 `<span>`에는 `aria-hidden="true"`를 두지 않는다(일부 리더는 aria-label이 있으면 자식 텍스트를 생략함). 팀 정책에 따라 “배지에만 aria-label, 자식은 그대로 두기”로 통일하면 됨.

### 4.2 KPI 카드 숫자 (상단 KPI 타일)

- **구성**: 카드별로 “라벨(총 사용자, 예약된 상담, 완료율 등)” + “숫자(또는 퍼센트)”.
- **권장**:  
  - **방법 1**: 카드 루트에 `role="group"` + `aria-labelledby="id1 id2"`. `id1` = 라벨 요소 id, `id2` = 숫자 요소 id. 그러면 “총 사용자 1,234”처럼 한 그룹으로 읽힌다.  
  - **방법 2**: 카드 루트에 `aria-label="총 사용자 1,234"` 형태로 한 문장 제공. 숫자가 동적으로 바뀌면 라벨을 업데이트해야 함.
- **숫자 포맷**: “1,234”처럼 천 단위 구분이 있으면, 스크린 리더가 “1천2백34” 또는 “1234”로 읽을 수 있음. 필요 시 `aria-label`에서 읽기 형태를 보정(예: “총 사용자 1,234명”)할 수 있다. “완료율 85%”는 “완료율 85퍼센트”로 읽히도록 문구만 통일하면 됨.

### 4.3 요약 표

| 요소 | 권장 속성 | 예시 |
|------|-----------|------|
| PipelineStepBadge | `aria-label` | `"3건 매칭됨"`, `"2건 대기중"` |
| KPI 카드 (그룹) | `role="group"` + `aria-labelledby` 또는 `aria-label` | `"총 사용자 1,234"` |
| 차트 영역 | `role="img"` + `aria-label` | `"최근 6개월 상담 완료 추이"` |

## 5. 에러·로딩 상태 접근성

### 5.1 로딩 중

- **위치**: 지표 영역 전체 또는 KPI·파이프라인 단위 스켈레톤/스피너.
- **권장**:  
  - **라이브 리전**: 로딩 상태만 바뀌고 별도 “알림”이 필요하지 않다면 `aria-live`는 생략 가능.  
  - **스켈레톤**: 장식용 플레이스홀더에는 `aria-hidden="true"`를 두어, 스크린 리더가 “로딩 중” 문구만 읽도록 한다.  
  - **명시적 문구**: “지표를 불러오는 중입니다” 같은 텍스트를 `aria-live="polite"` 영역에 넣으면, 로딩 완료 후 해당 문구가 사라질 때 스크린 리더가 변경을 인지할 수 있음. 과도한 알림을 피하려면 **polite**만 사용.
- **정책**: 테스트 제안서(ADMIN_DASHBOARD_METRICS_TEST_PROPOSAL.md) 3.3절과 같이, “동적 갱신 시 라이브 리전 사용 여부는 정책에 따름, 과도한 알림 없음”을 유지.

### 5.2 에러 메시지

- **위치**: 파이프라인 전체 또는 KPI 영역 **상단** 1줄 에러 문구(옵션 A 스펙과 동일).
- **권장**:  
  - **라이브 리전**: 에러 메시지가 동적으로 삽입되는 경우 `aria-live="assertive"` 또는 `aria-live="polite"`를 사용해, 스크린 리더가 즉시 에러를 읽도록 한다.  
  - **역할**: 에러 컨테이너에 `role="alert"`를 두면 자동으로 assertive와 유사하게 동작하는 리더가 많음.  
  - **문구 예시**: “지표를 불러오지 못했습니다. 잠시 후 다시 시도해 주세요.” (옵션 A 스펙과 동일)
- **포커스**: 에러 발생 시 포커스를 에러 메시지로 옮기는 것은 선택. 오토 포커스는 키보드 사용자에게 혼란을 줄 수 있으므로, **알림만 하고 포커스 이동은 하지 않는 것**을 기본으로 권장.

### 5.3 빈 상태 (데이터 없음)

- **문구**: “데이터가 없습니다”, “기간 내 완료된 상담이 없습니다” 등. 해당 텍스트는 **포함된 요소가 스크린 리더에 의해 읽히도록** 시맨틱 HTML(예: `<p>`)에 넣는다. `aria-label`만 있고 시각적 텍스트가 없으면 일부 사용자에게 불리하므로, **실제 보이는 문구와 동일**하게 노출하는 것을 원칙으로 한다.

### 5.4 요약 표

| 상태 | 권장 접근성 처리 |
|------|------------------|
| 로딩 중 | 스켈레톤 `aria-hidden="true"`. 필요 시 “로딩 중” 문구 + `aria-live="polite"`. |
| 에러 | 에러 메시지 컨테이너에 `role="alert"` 또는 `aria-live="assertive"`. 문구 한글 명시. |
| 빈 데이터 | 보이는 문구를 그대로 시맨틱 요소에 포함해 스크린 리더가 읽도록 함. |
