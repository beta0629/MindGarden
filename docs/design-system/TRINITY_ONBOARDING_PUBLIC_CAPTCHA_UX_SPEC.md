# Trinity 온보딩·공개 발송 — CAPTCHA(Turnstile / hCaptcha) UX 스펙

**역할**: 디자인(core-designer) 산출물 · **코드 비포함**  
**기준 비주얼**: `PENCIL_DESIGN_GUIDE.md`, `MINDGARDEN_DESIGN_SSOT.md`, B0KlA 토큰(`var(--mg-color-*)`)  
**범위**: 온보딩·공개 발송 **Primary CTA 직전**에 두는 사람 확인(봇 방지) UI

---

## 1. 컴포넌트 배치 (위치·모바일)

| 항목 | 스펙 |
|------|------|
| **수직 순서** | 폼 본문(필드) → **CAPTCHA 영역** → (선택) 보조 설명 한 줄 → **Primary CTA**(제출·발송 등) |
| **위치** | Primary 버튼 **바로 위**에 고정. 동일 섹션 블록 안에서 폼 하단·액션 그룹 위에 둔다. |
| **간격** | CAPTCHA 컨테이너와 Primary 버튼 사이 **16px**(`gap` 또는 `margin-top` — 표준 섹션 내부 gap과 동일). CAPTCHA 위쪽과 마지막 입력 필드 사이도 **16~24px**. |
| **폭** | 데스크톱: 본문 컨테이너 폭 내 **왼쪽 정렬**(위젯 기본 최소 폭 유지). 모바일(375 기준): **가로 100%**, 좌우는 본문 패딩(24px 등) 안에서만 사용 — **가로 스크롤 없음**. |
| **높이·터치** | 위젯·재시도 링크·보조 버튼은 **최소 44px** 터치 영역 확보(모바일). |
| **시각** | CAPTCHA를 감싸는 래퍼는 필요 시 `var(--mg-color-surface-main)` 배경 + `var(--mg-color-border-main)` 1px + radius **16px**·패딩 **16px**로 섹션 블록과 톤 맞춤(강조 과하지 않게). |

## 2. 상태별 카피 (한국어)

| 상태 | 카피(권장) |
|------|------------|
| **로딩** | 보안 확인을 불러오는 중입니다. 잠시만 기다려 주세요. |
| **성공** | 확인이 완료되었습니다. 아래 버튼으로 진행할 수 있습니다. |
| **실패** | 사람 확인에 실패했습니다. 아래에서 다시 시도해 주세요. |
| **네트워크 오류** | 네트워크 오류로 확인을 완료할 수 없습니다. 연결을 확인한 뒤 다시 시도해 주세요. |

- 실패·네트워크 시 **재시도**가 가능하면 버튼/링크 라벨: **다시 시도**, 보조: **문제가 계속되면 페이지를 새로고침한 뒤 다시 시도해 주세요.**

## 3. 접근성 (focus, 대체 텍스트)

| 항목 | 스펙 |
|------|------|
| **영역 이름** | 외곽에 `role="region"`(또는 적절한 래퍼) + **접근 가능한 이름**: 예) `aria-label="보안 확인(자동 입력 방지)"`. |
| **상태 알림** | 로딩·성공·실패·오류 문구는 **`aria-live="polite"`** 영역에 반영해 스크린 리더가 순서대로 읽도록 한다. |
| **포커스** | 위젯이 로드되면 키보드 사용자가 **위젯(또는 첫 포커스 가능 요소)** → **Primary CTA** 순으로 이동 가능해야 한다. 위젯 내부는 공급자 iframe이므로 **앱에서 불필요한 포커스 가로채기 금지**. |
| **Primary CTA** | CAPTCHA 미완료 시 버튼은 **비활성**이면 `aria-disabled="true"` 및 `aria-describedby`로 위의 보안 영역 또는 안내 문구와 연결. |
| **대체 텍스트** | 공급자 기본 iframe/이미지에 의존; **커스텀 로고·배너**를 넣을 경우에만 의미 있는 `alt` 제공. 순수 장식은 `alt=""`. |
| **색만으로 상태 표현 금지** | 성공·실패는 **문구**로 항상 병기한다. |

## 4. core-coder 구현 체크리스트 (8개 이내)

1. CAPTCHA를 Primary CTA **직상단**에 두고, 위·아래 간격 **16~24px**·토큰 색·섹션 블록 톤을 `PENCIL_DESIGN_GUIDE`와 맞출 것.  
2. **미검증 시 Primary 비활성** 또는 동일 의미의 차단 + `aria-describedby`로 안내 연결.  
3. 로딩·성공·실패·네트워크 오류에 위 **한국어 카피**와 **`aria-live="polite"`** 적용.  
4. 위젯 로드 실패 시 **다시 시도**·필요 시 **새로고침** 안내 UI 제공.  
5. 모바일에서 **가로 스크롤 없이** 100% 폭 사용, 터치 최소 **44px**.  
6. **서버 검증** 실패 시에도 동일 톤의 한국어 메시지로 폼 상단 또는 CAPTCHA 인근에 표시.  
7. Turnstile/hCaptcha **site key·환경**(dev/prod) 분리 및 **하드코딩 금지** — 설정·ENV로만 주입.  
8. 개인정보·자동 수집 관련 문구는 **중복 없이** 기존 온보딩/발송 화면 정책 문구와 합치거나, 접이식 도움말 한 곳에만 둘 것.

---

**참고**: 위젯 공급자(Turnstile / hCaptcha) 전환 시에도 **배치·간격·카피·a11y 패턴**은 본 문서를 유지하고, iframe API만 교체한다.
