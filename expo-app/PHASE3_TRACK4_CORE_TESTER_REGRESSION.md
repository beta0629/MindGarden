# Expo Phase 3 Track 4 — core-tester 회귀 체크리스트 (웰니스·커뮤니티 API층)

다음은 `expo-app` Phase 3-B/3-C API 연동·Mock 폴백 변경 후 **core-tester**가 수행하면 좋은 최소 회귀 항목입니다.

1. **감정 일기**: 월 이동·달력 이모지·기록 작성·상세·삭제가 동작하고, 백엔드 미구동 시에도 MMKV 로컬로 저장·표시되는지 확인한다 (`/api/v1/mood-journals` 우선).
2. **자가 심리검사**: PHQ-9/GAD-7/PSS 검사·결과·이력이 동작하고, API 실패 시에도 로컬 제출·목록이 유지되는지 확인한다 (`/api/v1/self-assessments`).
3. **명상**: 목록 새로고침·추천 카드·카테고리 필터·플레이어 진입이 되고, `GET /api/v1/meditations` 실패 시에도 데모 목록·CC0 스트림 폴백으로 크래시 없는지 확인한다.
4. **심리 교육**: 힐링 콘텐츠·`GET /api/v1/psycho-education` 각각 실패 시 EmptyState + 재시도가 보이고, 샘플 카드뉴스·상세(서버 id 포함)가 계속 열리는지 확인한다.
5. **커뮤니티(내담자·상담사)**: 데이터 소스 배지·동기화 버튼·당겨서 새로고침으로 `GET /api/v1/community` 재시도가 되고, 실패 시 샘플(MMKV) 피드·상세·좋아요가 깨지지 않는지 확인한다.

부가: 로그인·테넌트 헤더가 있는 상태에서 위 API가 404여도 앱이 중단되지 않아야 한다.
