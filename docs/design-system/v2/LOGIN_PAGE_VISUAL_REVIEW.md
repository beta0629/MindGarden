## 1. 좌측 히어로 영역(`mg-v2-login-hero`) — 이미지 미노출 원인 및 Fallback 권장

### 1.1 이미지가 보이지 않을 수 있는 원인 (스펙 대비 점검)

- **반응형에서 숨김/축소**: 스펙상 모바일(767px 이하)에서 좌측 영역은 "숨김(`display: none`) 또는 높이 200px 상단 배너로 축소"로 정의되어 있다. 현재 구현은 **숨기지 않고** `height: 200px`로만 축소하므로, 데스크톱에서 이미지가 안 보이는 이유는 아니다. 다만 모바일에서는 영역이 작아져 이미지 임팩트가 약해지고, **이미지 로드 실패 시** 같은 200px 영역에 배경이 비어 보일 수 있다.
- **외부 URL 차단**: 배경 이미지가 `https://images.unsplash.com/...` 단일 외부 URL에 의존한다. 회사 방화벽·CORS·CSP·네트워크 오류·Unsplash 일시 장애 시 요청이 차단되거나 실패할 수 있으며, 이때 **대체 배경이 없으면** 좌측 패널이 배경 없이 오버레이(`rgba(44,44,44,0.4)`)와 텍스트만 보이게 되어 "이미지 없이 텍스트만 있다"는 체감으로 이어진다.
- **대체 콘텐츠 없음**: `background-image` 로드 실패 시 사용할 `background-color` 또는 그라데이션 명세가 스펙·구현 모두에 없음. 브라우저 기본 배경 또는 투명에 가까운 상태 위에 다크 오버레이만 적용되면 시각적으로 빈 칸 + 텍스트로 인식된다.

**정리**: 체감상 "이미지 없이 텍스트만 있다"는 인상은, **이미지 미로드 시 비주얼 fallback이 없기 때문**일 가능성이 크다. 반응형 자체보다 **외부 이미지 실패 + fallback 부재**가 주된 원인으로 보는 것이 타당하다.

### 1.2 Fallback 권장 및 시각적 명세

**권장**: 이미지가 로드되지 않을 때를 대비해 **배경 fallback**을 스펙에 명시하고 구현 시 적용하는 것을 권장한다. 좌측 히어로는 "고품질 이미지 + 다크 오버레이"가 기본이되, 이미지 실패 시에도 B0KlA 톤을 유지하려면 **동일 오버레이 아래에 브랜드 그라데이션 또는 단색**을 두는 방식이 적합하다.

**시각적 명세 (B0KlA·unified-design-tokens 기준)**  
- **Fallback 배경 (이미지 미로드 시)**  
  - **옵션 A (권장)**: 선형 그라데이션. `var(--mg-color-primary-main)`(#3D5246) → `var(--mg-color-primary-light)`(#4A6354). 방향: `135deg` 또는 `to bottom right`.  
  - **옵션 B**: 단색. `var(--mg-color-primary-main)`(#3D5246).  
  - **옵션 C**: 메인 배경 톤 유지. `var(--mg-color-background-main)`(#FAF9F7) → `var(--mg-color-surface-main)`(#F5F3EF) 그라데이션 + 기존 다크 오버레이 유지.  
- **오버레이**: 기존과 동일 `rgba(44, 44, 44, 0.4)` (다크 오버레이)를 fallback 위에 그대로 적용하여, 이미지가 있을 때와 없을 때 모두 로고·슬로건 가독성을 유지한다.  
- **구현 시**: `mg-v2-login-hero`에 **이미지 URL보다 먼저 적용되는** `background-color` 또는 `linear-gradient`를 지정하고, `background-image`를 그 위에 두면, 이미지 실패 시 자동으로 fallback이 노출된다. 토큰이 없는 환경을 대비한 fallback 값: `#3D5246`, `#4A6354`, `#FAF9F7`, `#F5F3EF`.

## 2. 로그인 페이지 전체 — "텍스트만 있다" 인상 완화 제안

- **좌측 히어로만으로 충분한지**: 비율(5:5)·대비(다크 오버레이 + 흰 텍스트)·타이포(32px 로고, 18px 슬로건)만으로도 **이미지가 정상 로드될 때**는 한쪽 패널이 비주얼로 채워져 "텍스트만 있다"는 느낌을 줄 수 있다. 다만 **이미지 미로드 시** 현재는 fallback이 없어 좌측이 빈 칸처럼 보이므로, **1절의 fallback 적용**이 선행되면 좌측만으로도 체감이 크게 개선된다.
- **우측 폼 영역 서브 비주얼 추가 권장**: "텍스트만 있다"는 인상을 더 줄이려면, 우측 `mg-v2-login-content`(또는 `mg-v2-login-form-wrapper` 상단/하단)에 **저자극 서브 비주얼**을 두는 것을 권장한다. 예: B0KlA 포인트/보조 색(`var(--mg-color-accent-main)`, `var(--mg-color-secondary-main)`)을 쓴 소형 일러스트·아이콘(잎·식물·마음 관련 실루엣 등)을 폼 위쪽 여백 또는 구분선 근처에 배치하고, opacity 0.15~0.25 수준으로 낮춰 본문·입력 필드 가독성을 해치지 않도록 한다. 필수는 아니며, **좌측 fallback 적용 후에도 비주얼 밸런스가 부족하다고 판단될 때** 도입해도 된다.

**요약**: (1) 좌측 이미지 미노출은 외부 URL 차단·로드 실패 시 **fallback 배경 부재**가 주된 원인일 가능성이 크므로, B0KlA 토큰 기반 그라데이션/단색 fallback 명세를 스펙에 추가하고 구현 시 적용할 것을 권장한다. (2) 이미지가 잘 로드되면 좌측 비율·대비·타이포만으로도 어느 정도 비주얼이 채워지며, fallback 적용 후에는 "텍스트만 있다"는 체감이 줄어든다. (3) 필요 시 우측 폼 영역에 저대비 서브 비주얼(아이콘·일러스트)을 추가해 전체 밸런스를 보강할 수 있다.
