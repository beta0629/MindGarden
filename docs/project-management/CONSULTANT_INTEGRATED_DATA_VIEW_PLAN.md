# 상담사 별 통합데이터 카드 — 뷰 전환·순위·시각화 정책

## 1. 제목·목표

- **제목**: 관리자 대시보드 V2 "상담사 별 통합데이터" 카드 — 순위 표시·다중 뷰·시각화 추가
- **목표**: 동일 데이터(`consultantIntegratedData`)에 대해 **순위를 명확히 노출**하고, **테이블 / 그래프 / 프로그레스 바**를 버튼으로 전환해 관리자가 한 카드에서 다양한 방식으로 확인할 수 있게 한다.

## 2. 범위

| 구분 | 내용 |
|------|------|
| **포함** | `AdminDashboardV2.js` 내 "상담사 별 통합데이터" 카드 영역(`mg-v2-ad-b0kla__card`, `consultantIntegratedData` 사용 구간). 순위 정의·표시, 뷰 전환 state·버튼, 테이블(순위 컬럼)·막대 그래프·프로그레스 바 뷰 |
| **제외** | 다른 대시보드 카드, API·정렬 로직 변경(기존 정렬 유지), 상담사 상세 페이지 |
| **참조** | 기존 정책: `docs/project-management/CONSULTANT_INTEGRATED_DATA_CARD_PLAN.md` (지표·데이터 소스) |

## 3. 순위 정의·노출 방식

- **정렬 기준(변경 없음)**: 기존과 동일 — `completedCount` 내림차순 → 동점 시 `rating` 내림차순, 상위 10명.
- **순위 산출**: 배열 인덱스 기반 **순위 = 인덱스 + 1** (1위~10위). 정렬 순서가 곧 순위.
- **노출 방식**:
  - **테이블 뷰**: 컬럼 추가 — "순위" 컬럼(1, 2, … 10) 또는 "1위", "2위" 형태 라벨. 기존 상담사명 | 평점 | 완료 건수 | 완료율 유지.
  - **그래프·프로그레스 바 뷰**: 각 행/막대에 순위(1위~10위)를 라벨로 표시해 누가 1위인지 즉시 알 수 있게 한다.

## 4. 뷰 종류

| 뷰 | 설명 | 노출 내용 |
|----|------|------------|
| **테이블** | 기존 테이블형 + 순위 컬럼 | 순위 | 상담사명 | 평점 | 완료 건수 | 완료율 |
| **그래프** | 막대 차트 | 상담사별 **완료 건수** 또는 **완료율** 막대 비교(1차: 완료 건수). B0KlA·기존 `Chart`(CHART_TYPES.BAR) 활용 |
| **프로그레스** | 행별 프로그레스 바 | 상담사(순위)·이름 + **완료율** 프로그레스 바(0~100%). 필요 시 완료 건수/전체 건수 텍스트 보조 |

## 5. 버튼·탭 UI (뷰 전환)

- **위치**: 카드 상단, 제목("상담사 별 통합데이터")·부제 아래, 본문(테이블/그래프/프로그레스 영역) **위**에 배치.
- **요소**: "테이블" | "그래프" | "프로그레스" 세 가지 선택지. **버튼 그룹** 또는 **탭** 형태. 한 번에 하나만 선택, 선택 시 해당 뷰만 렌더.
- **기본 선택**: **테이블** (기존과 동일한 진입 경험 유지).
- **접근성**: 선택된 뷰에 대해 `aria-pressed` 또는 `aria-selected`, 역할에 맞는 라벨.

## 6. 사용성·정보 노출·레이아웃 (디자이너 전달용)

- **사용성**: 관리자가 카드에서 "순위를 보고 → 숫자 비교(테이블) / 막대 비교(그래프) / 완료율 비율(프로그레스)" 중 선호하는 방식으로 전환해 한 화면에서 확인.
- **정보 노출**: 동일 데이터(상위 10명, 순위·상담사명·평점·완료 건수·완료율). 뷰에 따라 강조만 다름.
- **레이아웃**: 카드 = [제목·부제] → [뷰 전환 버튼] → [본문: 테이블 | 그래프 | 프로그레스 중 하나].

## 7. 의존성·순서

- **선행**: 기존 `consultantIntegratedData` 및 `loadStats()` 로드 완료(변경 없음).
- **순서**: 기획 정책(본 문서) → **Phase 1 core-designer** (뷰 전환 버튼·그래프·프로그레스 바 레이아웃·비주얼 스펙) → **Phase 2 core-coder** (구현).

## 8. 리스크·제약

- 기존 B0KlA 클래스·`unified-design-tokens.css` 유지. 차트 색상은 B0KlA·Chart 기존 패턴 준수.
- 데이터 없음 시: 기존 empty 문구 "상담사 통합 데이터가 없습니다." 유지, 뷰 전환 버튼은 그대로 두거나 비활성 처리 중 하나로 통일(디자이너 스펙에서 결정).
