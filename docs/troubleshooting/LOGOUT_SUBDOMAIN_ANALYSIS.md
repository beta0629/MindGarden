# 로그아웃 시 서브도메인 처리 전체 분석

## 현재 상황

### 1. 로그아웃 플로우 (sessionManager.js)

```javascript
// 로그아웃 시 리다이렉트 로직
1. 현재 호스트 확인
2. 로컬 환경 체크 (localhost, 127.0.0.1)
3. 서브도메인 추출:
   - 서브도메인이 있으면: 해당 서브도메인/login으로 이동
   - 서브도메인이 없으면: dev.core-solution.co.kr/login으로 이동
```

**문제점:**
- 서브도메인이 없으면 `dev.core-solution.co.kr/login`으로 이동
- 이 경우 테넌트 ID를 찾을 수 없어서 오류 발생 가능

### 2. 로그인 페이지 로드 시 (UnifiedLogin.js)

```javascript
// detectTenantFromSubdomain() 함수
1. URL 파라미터에서 tenantId 확인 (로컬 테스트용)
2. 환경 변수에서 tenantId 확인 (로컬 개발용)
3. 서브도메인 추출:
   - 서브도메인이 있으면: API로 테넌트 조회 → sessionStorage에 저장
   - 서브도메인이 없으면: 아무것도 하지 않음 (로컬 환경만 안내 메시지)
```

**문제점:**
- `dev.core-solution.co.kr`에서 로그인 페이지 로드 시 서브도메인이 없음
- 테넌트 ID를 찾을 수 없음
- 하지만 로컬 환경이 아니므로 안내 메시지도 표시하지 않음

### 3. 로그인 시도 시 (UnifiedLogin.js - handleSubmit)

```javascript
// handleSubmit() 함수
1. 서브도메인 체크
2. 서브도메인이 없고 로컬 환경이 아니면:
   - 에러 메시지 표시
   - 로그인 차단 (return)
```

**현재 구현:**
- 서브도메인이 없으면 로그인 자체를 차단함 ✅
- 하지만 사용자 경험이 좋지 않음

### 4. 로그아웃 후 안내 메시지 (UnifiedLogin.js)

```javascript
// 로그아웃 상태 확인 useEffect
- logout=success 파라미터가 있으면
- 서브도메인이 없으면 안내 메시지 표시 (1초 후)
```

**현재 구현:**
- 로그아웃 후 서브도메인이 없으면 안내 메시지 표시 ✅
- 하지만 페이지 로드 시 즉시 표시되지 않음

## 문제 요약

1. **로그아웃 시 서브도메인이 없으면 `dev.core-solution.co.kr/login`으로 이동**
   - 이는 의도된 동작이지만, 테넌트 ID를 찾을 수 없음

2. **로그인 페이지 로드 시 서브도메인이 없으면 테넌트 ID를 찾지 못함**
   - `detectTenantFromSubdomain()`이 아무것도 하지 않음
   - 로컬 환경이 아니므로 안내 메시지도 없음

3. **로그인 시도는 차단되지만, 사용자에게 명확한 안내가 부족함**
   - 페이지 로드 시 즉시 안내 메시지가 보이지 않음
   - 로그인 시도 후에야 에러 메시지 표시

4. **백엔드에서 테넌트 ID가 없으면 오류 발생 가능**
   - 일부 API 호출 시 테넌트 ID가 필수일 수 있음

## 해결 방안

### 방안 1: 로그인 페이지 로드 시 즉시 안내 메시지 표시 (권장)

**변경 사항:**
1. `detectTenantFromSubdomain()`에서 서브도메인이 없고 로컬 환경이 아니면 즉시 안내 메시지 표시
2. 로그아웃 후 `dev.core-solution.co.kr/login`으로 이동 시 서브도메인 안내 메시지를 더 명확하게 표시

**장점:**
- 사용자가 즉시 문제를 인지할 수 있음
- 로그인 시도 전에 미리 안내 가능

### 방안 2: 로그아웃 시 서브도메인 정보 보존 (현재 부분 구현됨)

**현재 구현:**
- `sessionStorage`에 `subdomain_tenant_id`와 `subdomain` 보존
- 로그아웃 후에도 이 정보 유지

**문제점:**
- 하지만 `dev.core-solution.co.kr/login`으로 이동하면 이 정보가 유지되지 않을 수 있음
- 다른 도메인으로 이동 시 sessionStorage는 유지되지 않음

**개선:**
- 로그아웃 전 서브도메인 정보를 쿠키나 다른 방식으로 보존
- 로그인 페이지에서 이 정보를 확인하여 안내

### 방안 3: 백엔드 API 호출 시 테넌트 ID 없이도 처리 가능하도록

**현재 상황:**
- 백엔드 API는 테넌트 ID가 필수인 경우가 많음
- 로그인 페이지에서 일부 API 호출 시 테넌트 ID가 없으면 오류 발생 가능

**해결:**
- 로그인 페이지에서 서브도메인이 없으면 테넌트 ID를 요구하는 API 호출을 하지 않도록
- 또는 백엔드에서 테넌트 ID가 없어도 처리 가능한 API는 에러 처리 개선

## 권장 구현

1. **로그인 페이지 로드 시 서브도메인 체크 및 즉시 안내**
   - `detectTenantFromSubdomain()` 함수에서 서브도메인이 없고 로컬 환경이 아니면 즉시 안내 메시지 표시
   - 페이지 로드 시 즉시 표시하여 사용자 혼란 방지

2. **로그인 폼 비활성화**
   - 서브도메인이 없으면 로그인 폼 자체를 비활성화
   - 명확한 안내 메시지와 함께 서브도메인으로 이동하는 방법 안내

3. **로그아웃 후 안내 메시지 개선**
   - 현재는 1초 후에 표시되는데, 즉시 표시하도록 변경
   - 더 명확한 안내 메시지 제공
