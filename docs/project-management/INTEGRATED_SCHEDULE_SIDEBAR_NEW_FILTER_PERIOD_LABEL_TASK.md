# 통합 스케줄 사이드바 "신규 매칭" 필터 기간 표기 — 실행 태스크

> **목표**: "신규 매칭" 필터에 기간(7일/1주 등) 표기 추가.  
> **담당 실행**: core-coder (1 Phase).  
> **기획**: core-planner.

---

## 1. 요구사항 요약

- **사용자 요구**: 사이드바 보기 필터 중 "신규 매칭"이 **어떤 기간 기준**(하루/한 주/한 달)인지 화면에 표시.
- **현재**: `NEW_DAYS = 7` 기준으로 필터링되지만 라벨은 "신규 매칭"만 표시됨.
- **완료 기준**: 필터 라디오·aria·빈 목록 문구에 기간이 드러나며, 값은 상수(`NEW_DAYS`) 기반으로 유지.

## 3. 기간 표기 스펙

- **표기 형식**: `신규 매칭 (N일)` 또는 `신규 매칭 (1주)` 등. **한국어만 사용** (현재 해당 화면 다국어 미적용).
- **규칙**:
  - `NEW_DAYS` 값에 따라 문구 결정. 예: `7` → `"7일"` 또는 `"1주"` (프로젝트 표준에 맞게 하나로 통일).
  - **상수화**: 기간 문구(예: `"7일"`, `"1주"`)를 `NEW_DAYS`와 함께 상수로 두고, 라벨은 `신규 매칭 (${NEW_DAYS_LABEL})` 형태로 조합. (`core-solution-frontend` 상수화 규칙 준수)
- **적용 위치**:
  - 사이드바 **보기 필터** 라디오: 표시 텍스트 + `aria-label` (L237–246).
  - **빈 목록 메시지**: "신규 매칭이 없습니다." → "신규 매칭 (7일)이 없습니다." 등 기간 포함 (L308–309).
- **제외**: 헤더의 "신규 매칭" 버튼·"신규 매칭 생성" aria-label은 **기간 미표기** 유지.

## 4. 수정 대상 파일 및 병렬 가능성

- **단일 파일**: `IntegratedMatchingSchedule.js` 만 수정.
- **병렬**: 동일 파일 내에서 (1) 기간 라벨 상수 추가, (2) 필터/빈 목록 문구 치환으로 한 번에 구현 가능. **별도 서브에이전트 병렬 분리 없음.** (다른 화면에 동일 패턴 적용 시에는 그때 별도 태스크로 분리·병렬 가능.)

## 5. core-coder 실행 지시 (호출 시 전달용)

**아래 내용을 core-coder 서브에이전트 호출 시 전달할 태스크 설명으로 사용하세요.**

---

### 5.1 적용 스킬

- `/core-solution-frontend` (상수화, 라벨·매직넘버 상수 정의)
- `docs/project-management/INTEGRATED_SCHEDULE_SIDEBAR_NEW_FILTER_PERIOD_LABEL_TASK.md` (본 문서)

### 5.2 작업 내용

1. **상수 추가** (`IntegratedMatchingSchedule.js` 상단 영역, `NEW_DAYS` 인근)
   - `NEW_DAYS`는 유지.
   - 기간 표기용 라벨 문자열 상수 추가. 예: `NEW_DAYS_LABEL = '7일'` (또는 `'1주'` — 팀 표준에 맞게 하나로 통일).
   - "신규 매칭" 필터용 전체 라벨 상수 추가. 예: `VIEW_FILTER_NEW_LABEL = \`신규 매칭 (${NEW_DAYS_LABEL})\`` 또는 동일 의미의 상수.

2. **사이드바 필터 라디오 (L236–246)**
   - `<span className="integrated-schedule__filter-text">신규 매칭</span>` → 상수 사용해 "신규 매칭 (7일)" 형태로 표시.
   - `aria-label="신규 매칭"` → 동일 상수 사용해 기간 포함.

3. **빈 목록 메시지 (L308–309)**
   - `emptyMessage = '신규 매칭이 없습니다.'` → 기간 포함 문구로 변경 (동일 상수 사용).

4. **변경하지 않을 것**
   - 헤더 버튼 텍스트 "신규 매칭", `aria-label="신규 매칭 생성"` — 그대로 유지.

### 5.3 완료 체크리스트

- [ ] `NEW_DAYS` 기반 기간 라벨 상수 추가됨.
- [ ] 필터 라디오 표시 텍스트·aria-label에 기간 표기 반영됨.
- [ ] 빈 목록 시 "신규 매칭 (N일)이 없습니다." (또는 동일 의미) 문구 적용됨.
- [ ] 헤더 "신규 매칭" 버튼/aria는 미변경.
- [ ] 기존 필터 동작(VIEW_FILTER_NEW, cutoff 로직) 변경 없음.
