# ERP UX 라벨 적용 스펙 — 화면·영역별

**작성일**: 2026-03-14  
**참조**:  
- `docs/project-management/ERP_UX_INVISIBLE_ERP_PLAN.md`  
- `docs/project-management/ERP_UX_PLANNER_TASK_HANDOFF.md` §2  

**목표**: "이게 ERP인 줄 모를 정도로" — 메뉴·화면·메시지에서 ERP·분개·원장·전기 노출을 최소화하고, **화면·영역 단위**로 적용할 라벨을 단일 스펙으로 정리한다.  
**적용 주체**: core-coder (코드 작성). 본 문서는 **마크다운 스펙만** 제공하며, 코드 수정은 하지 않는다.

## 7. 권한 관리 화면

권한 설정/역할별 권한 UI에서 **사용자에게 보이는 노출 라벨만** 변경한다.  
**권한 코드**(`ERP_ACCESS`, `ERP_DASHBOARD_VIEW`, `INTEGRATED_FINANCE_VIEW` 등) 값·API·DB는 **변경하지 않는다**.

| 권한 코드 (유지) | 현재 노출 name | 변경 노출 name | 현재 description | 변경 description |
|------------------|----------------|----------------|-------------------|-------------------|
| ERP_ACCESS | ERP 시스템 | **운영·재무 접근** | ERP 시스템 전체 접근 | **운영·재무 메뉴 전체 접근** |
| ERP_DASHBOARD_VIEW | ERP 대시보드 | **운영 현황** | ERP 대시보드 조회 | **운영 현황(수입·지출·구매) 조회** |
| INTEGRATED_FINANCE_VIEW | 통합 재무 관리 | **수입·지출 관리** | 통합 재무 관리 시스템 접근 | **수입·지출·거래·정산 화면 접근** |

**카테고리명 (권한 그룹 라벨)**  
- "대시보드 및 보고서" 내 ERP 관련 권한 3건: 위 name/description만 변경.  
- 카테고리 "ERP 구매 및 승인": 노출 라벨을 **구매 및 승인**으로 변경 권장 (ERP 미노출).

**적용 파일**: `PermissionManagement.js` (PERMISSION_CATEGORIES의 name, description 및 카테고리명).  
**참고**: DB에서 권한 목록을 불러와 `permissionName`/`permissionDescription`을 표시하는 경우, **백엔드 또는 프론트 매핑**으로 노출 문자열만 위와 같이 매핑할 수 있다. 코드값은 그대로 유지.

## 8. 메시지·모달 문구

기획서 §3(메시지·토스트)에 더해, **모달 제목·버튼·안내 문구** 중 "분개/원장/전기/ERP"가 노출되는 부분과 대체 문구를 정리한다.

| 상황 | 현재 스타일 | 변경 스타일 |
|------|-------------|-------------|
| 입금 확인 후 (자동 반영) | (자동 분개) | **입금이 반영되었습니다** |
| 승인/전기 성공 | 분개가 승인되었습니다 / 분개가 전기되었습니다 | **거래가 반영되었습니다** |

| 데이터 동기화 섹션 (관리자용) | "분개가 없는 건에 대해 분개를 생성·전기합니다" | 선택: **"미반영 수입 거래를 자동으로 반영합니다"** (사용자 친화) 또는 관리자용으로 현행 유지 |

## 9. 표시 제약·권장 사항 (B0KlA·어드민 대시보드 샘플 기준)

- **타이틀·카드 라벨 길이**  
  - "운영 현황", "수입·지출 관리", "수입·지출 한눈에", "거래·손익·정산을 한곳에서" 등은 한글 8~14자 내외.  
  - B0KlA 상단 바·섹션 제목은 **20~24px, fontWeight 600**, 라벨/캡션은 **12px**.  
  - **줄바꿈**: 상단 바 제목은 한 줄 유지 권장(넓은 뷰포트). 모바일(375px)에서는 "수입·지출 관리" 6자로 짧아 줄바꿈 이슈 적음.

- **가독성**  
  - "운영·재무", "수입·지출"처럼 중간점(·) 사용 시 자간이 좁지 않도록 `var(--mg-color-text-main)` 및 Noto Sans KR 유지.  
  - 카드 라벨 "계정별 내역", "자산·부채 현황", "손익 현황"은 5~7자로 카드 썸네일·리스트에서 줄바꿈 없이 표시 가능.

- **일관성**  
  - LNB, 운영 현황, 통합 재무, AdminDashboard 위젯에서 **동일 개념은 동일 라벨** 사용(예: "운영 현황" 한 가지로 통일).

- **접근성**  
  - 제목·카드에 사용하는 텍스트는 기존 시맨틱(h1, h2, role)을 유지한 채 **문구만** 교체.  
  - `aria-label`/`title`에 "ERP 대시보드" 등이 있으면 "운영 현황" 등으로 함께 변경 권장.
