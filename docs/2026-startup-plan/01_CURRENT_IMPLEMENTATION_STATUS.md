# CoreSolution 현재 구현도 (2025-12 기준)

> **작성일**: 2025-12-27  
> **목적**: 26년 예비창업자패키지 창업계획서용 현재 시스템 구현 상태 정리

---

## 1. 시스템 개요

### 1.1 제품 정의
**CoreSolution**은 상담센터/교육기관/1인 사업자 등 "서비스 업종"이 운영에 필요한 핵심 기능을 **하나의 SaaS**로 제공하는 통합 운영 플랫폼입니다.

### 1.2 핵심 구성 요소
1. **CoreSolution (메인 플랫폼)**: 멀티테넌트 SaaS 백엔드 및 관리자 대시보드
2. **온보딩 시스템**: 신규 고객 유입 및 전환 엔진
3. **Trinity 포털**: 회사 홈페이지 및 온보딩 신청 시스템
4. **Ops 포털**: 내부 운영 포털 (승인·관제·보안·배포)

## 2. CoreSolution 메인 플랫폼

### 2.1 백엔드 아키텍처

#### 기술 스택
- **프레임워크**: Spring Boot (Java)
- **데이터베이스**: MySQL (멀티테넌트 구조)
- **인증**: JWT 기반 토큰 인증
- **배포**: GitHub Actions 자동 배포 (CI/CD)

#### 주요 모듈 구조

## 3. 온보딩 시스템

### 3.1 시스템 개요
신규 고객(테넌트)이 플랫폼에 입점하는 과정을 자동화하고 표준화하는 시스템입니다.

### 3.2 구현 현황

#### ✅ 완료된 기능

**1. 온보딩 요청 생성**
- Trinity 포털에서 온보딩 신청
- 기본 정보 입력 (회사명, 브랜드명, 지역, 업종 등)
- 요금제 선택
- 결제 수단 등록 (토스페이먼츠 연동)

**2. 온보딩 데이터 저장 표준**
- 필드 단위 저장 (JSON 단독 저장 금지)
- `brand_name`, `region` 등 별도 필드 저장
- 체크리스트 JSON 보조 데이터 저장

**3. 온보딩 승인 프로세스**
- Ops 포털에서 승인/거부 결정
- 승인 시 자동 테넌트 생성
- 기본 역할 템플릿 적용
- 기본 대시보드 생성
- 관리자 계정 자동 생성

**4. PL/SQL 프로시저 연동**
- `ProcessOnboardingApproval` 프로시저 실행
- 테넌트 생성/활성화
- 카테고리 매핑 설정
- 기본 컴포넌트 활성화
- 기본 요금제 구독 생성
- ERD 자동 생성

#### 🟡 진행 중인 기능

**1. 검증 프로세스 강화**
- 이메일 중복 체크
- 서브도메인 중복 체크
- 사업자 정보 검증

**2. 자동 승인 시스템**
- 조건부 자동 승인 로직
- 리스크 평가 시스템

### 3.3 데이터 흐름

```
1. Trinity 포털에서 온보딩 신청
   ↓
2. OnboardingService.create() 호출
   ↓
3. onboarding_request 테이블 저장
   (brand_name, region 등 별도 필드 저장)
   ↓
4. Ops 포털에서 승인 결정
   ↓
5. OnboardingService.decide() 호출
   ↓
6. ProcessOnboardingApproval PL/SQL 실행
   ↓
7. 테넌트 생성 및 초기 설정
   ↓
8. 관리자 계정 생성 및 알림 발송
```
