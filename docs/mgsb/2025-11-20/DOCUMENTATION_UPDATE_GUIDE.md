**작성일**: 2025-11-20

**상태**: 완료

---

## 📋 개요

CoreSolution 플랫폼의 문서(스토리보드, IA 구조도, 메뉴 구조도)는 시스템 변경사항에 따라 지속적으로 업데이트되어야 합니다. 이 문서는 문서 현행화 프로세스와 체크리스트를 제공합니다.

---

## 🔄 문서 현행화 프로세스

### 변경사항 발생 시 체크리스트

```
1. 변경사항 확인
   ↓
2. 영향받는 문서 식별
   ↓
3. 문서 업데이트
   ↓
4. 검증 및 리뷰
   ↓
5. 문서 버전 업데이트
```

## 🔍 변경사항 감지 방법

### 1. 코드 변경 감지

**체크 포인트:**
- 새로운 Controller 추가/삭제
- 새로운 Service 추가/삭제
- 새로운 Entity 추가/삭제
- 메뉴 관련 CommonCode 변경
- 권한 관련 코드 변경
- 공통 코드 관리 기능 변경
- ERP 기능 추가/삭제

### 2. 메뉴 변경 감지

**체크 포인트:**
- `frontend/src/constants/menu.js` 변경
- CommonCode의 메뉴 그룹 변경
- 역할별 메뉴 권한 변경

### 3. 기능 변경 감지

**체크 포인트:**
- 새로운 API 엔드포인트 추가
- 기존 API 엔드포인트 삭제
- 화면 흐름 변경
- 역할별 기능 변경
- 공통 코드 관리 기능 추가/변경
- ERP 기능 추가/변경

## 📋 문서 업데이트 절차

### Step 1: 변경사항 확인

1. 코드 변경사항 확인
2. 메뉴 변경사항 확인
3. 기능 변경사항 확인

### Step 2: 영향받는 문서 식별

1. 스토리보드 문서 영향도 확인
2. IA 구조도 문서 영향도 확인
3. 메뉴 구조도 문서 영향도 확인

### Step 3: 문서 업데이트

1. 관련 문서 열기
2. 변경사항 반영
3. 예시 업데이트 (MindGarden 등)
4. 버전 정보 업데이트

### Step 4: 검증 및 리뷰

1. 문서 내용 검증
2. 예시 검증
3. 관련 문서 간 일관성 확인

### Step 5: 문서 버전 업데이트

1. 문서 상단 버전 정보 업데이트
2. 마지막 업데이트 날짜 업데이트
3. 변경 이력 추가 (필요시)

## ⚠️ 중요 사항

1. **즉시 업데이트 필요**
   - 새로운 기능 추가 시 즉시 문서 업데이트
   - 기존 기능 삭제 시 즉시 문서 업데이트
   - 메뉴 구조 변경 시 즉시 문서 업데이트

2. **일관성 유지**
   - 모든 관련 문서를 동시에 업데이트
   - 예시 문서도 함께 업데이트
   - 버전 정보 일관성 유지

3. **검증 필수**
   - 문서 업데이트 후 검증 필수
   - 예시 검증 필수
   - 관련 문서 간 일관성 확인 필수
