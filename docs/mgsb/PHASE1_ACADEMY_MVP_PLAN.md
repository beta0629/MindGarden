# MindGarden Phase 1 (Academy) MVP Plan

## 1. 사용자 페르소나 및 핵심 시나리오

### 1.1 페르소나

- **학원 원장 (테넌트 관리자)**
  - 목표: 지점 운영, 수강생 관리, 매출 파악
  - Pain Point: 수강료 정산 지연, 수동으로 처리되는 회원/출결 관리
- **데스크 직원 (지점 스태프)**
  - 목표: 상담 예약, 수강 등록, 결제 처리, 알림 발송
  - Pain Point: 여러 시스템을 오가며 정보를 입력해야 하는 비효율
- **학부모/수강생 (소비자)**
  - 목표: 강좌 탐색, 상담 예약, 수강료 결제, 일정 확인
  - Pain Point: 지점별로 다른 가입/결제 방식, 결제 영수증/내역 확인의 어려움

### 1.2 핵심 시나리오

1. **상담 예약/수강 등록**
   1. 소비자가 포털에서 학원을 탐색 후 상담 예약을 생성
   2. 데스크 직원이 예약을 확인하고 상담을 진행
   3. 상담 결과에 따라 수강 등록 및 결제 진행
2. **수강료 결제 및 정산**
   1. 소비자가 카드/계좌/간편결제로 수강료를 결제
   2. 결제 정보가 ERP와 연동되어 정산 배치 대상에 포함
   3. 정산 배치(PL/SQL)가 매출/수수료/정산금을 계산 -> 관리자 포털 리포트 제공
3. **출결 및 알림**
   1. 데스크 직원이 수업 출결 정보를 기록
   2. 학부모에게 출결/공지 알림 발송 (앱/문자/카카오톡 등)
4. **회원/통계 관리**
   1. 원장이 관리자 포털에서 수강생 정보, 결제 이력, 매출 추이를 확인
   2. 상위 본사(HQ)가 전체 학원들의 KPI를 모니터링

## 2. 기능 요구사항 및 우선순위

### 2.1 P0 (MVP 필수)

- 테넌트 온보딩: 학원 정보, 지점, 직원 계정 등록
- 소비자 포털: 학원 탐색, 상담 예약, 수강 등록/결제
- 결제 연동: 카드/계좌/간편결제, 영수증 발급
- 관리자 포털: 예약/수강 관리, 회원 관리, 결제 내역 확인
- 정산 배치: PL/SQL 기반 매출/수수료 계산, 리포트 다운로드
- 알림 발송: 예약 확정/결제 확인/정산 정보 알림 (이메일/문자)
- SessionManager/보안: 공개 페이지 접근 허용, 인증 흐름 정의

### 2.2 P1 (MVP 이후 빠른 확장)

- 출결 관리 UI 및 학부모 알림 템플릿
- 쿠폰/프로모션 관리(학원별)
- 리뷰 및 만족도 조사 기능
- 분석 대시보드(기초 KPI 시각화)

### 2.3 P2 (중기 확장)

- 모바일 앱 알림 푸시 연동
- 외부 학원 플랫폼/마케팅 채널과의 연동 (API 파트너십)
- BI/심화 분석, 추천 시스템

## 4. QA 및 테스트 전략

- **테스트 범위**
  - 유닛 테스트: 핵심 도메인 서비스, 정산 로직, 알림 전송
  - 통합 테스트: 예약→결제→정산→리포트 흐름, 멀티테넌시 접근 제어
  - E2E 테스트: 소비자 예약/결제 플로우, 관리자 승인/리포트 확인
  - 성능 테스트: 정산 배치(10만 건 기준), API SLA 95% ≤ 500ms
- **QA 계획**
  - QA 환경에서 자동화 테스트(Cypress/Playwright) 실행
  - 시각적 회귀(Percy/Chromatic)로 디자인 시스템 검증
  - 보안 테스트: OWASP Top 10 점검, 세션/쿠키 관리 검토
  - 사용자 검증: 파일럿 학원 1~2곳 대상 UAT(User Acceptance Test)
- **릴리즈 체크리스트**
  - 멀티테넌시 마이그레이션 스크립트 적용 여부
  - 정산 결과/알림 로그 검증, 롤백 플랜 준비
  - 운영 매뉴얼 및 지원 조직 교육 완료
