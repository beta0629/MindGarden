# 홈페이지 리뉴얼 기획서 (HOMEPAGE_REDESIGN_PLAN)

## 1. 제목 및 목표
- **제목**: Core Solution 메인 홈페이지 전면 리뉴얼
- **목표**: 텍스트 위주의 딱딱한 기존 인트로 페이지를 벗어나, 고품질 이미지와 조화롭게 어우러지는 현대적이고 시각적인 랜딩 페이지 구축. 기존 "MindGarden" 브랜딩을 제거하고 "CoreSolution" 정체성("비즈니스의 핵심을 솔루션하다" 등)을 전면에 내세웁니다.

## 2. 범위
- **포함 영역**:
  - `frontend/src/components/homepage/Homepage.js` 및 연관 CSS (`Homepage.css` 등) 전면 교체
  - 랜딩 페이지 내 신규 섹션 구성 (Hero 섹션, 핵심 기능 소개, 서비스 강점, Footer 등)
- **제외 영역**:
  - 로그인, 회원가입 등 홈페이지에서 파생되는 다른 라우트 페이지의 내부 로직 수정

## 3. 핵심 요구사항
1. **시각적 완성도 강화**: 멋진 고품질 이미지 및 사진을 적극 활용하여 시각적으로 트렌디하고 몰입감 있는 랜딩 페이지를 구성합니다.
2. **브랜딩 교체**: 기존 'MindGarden', '마음의 정원' 등의 명칭 및 문구 일체 제거. 'CoreSolution', '비즈니스의 핵심을 솔루션하다' 등의 슬로건을 강조합니다.
3. **디자인 시스템 준수**: 프로젝트의 아토믹 디자인 시스템 및 `unified-design-tokens.css`에 정의된 `var(--mg-*)` 형태의 디자인 토큰을 필수적으로 사용합니다. (하드코딩 절대 금지)
4. **반응형 지원**: 데스크탑, 태블릿, 모바일 기기 등 다양한 해상도에 최적화된 반응형 레이아웃을 제공해야 합니다.

## 4. 의존성 및 순서
1. **(선행)** `docs/design-system/` 내 랜딩 페이지 디자인 스펙 산출 (`core-designer` 활용)
2. **(후행)** 도출된 스펙 및 디자인 토큰을 바탕으로 한 실제 홈페이지 컴포넌트 구현 (`core-coder` 활용)

## 5. 단계별 실행 계획 (Phase)

### Phase 1: 디자인 스펙 작성
- **담당**: `core-designer`
- **목표**: 고품질 이미지를 활용한 랜딩 페이지의 UI/UX, 레이아웃 및 비주얼 상세 스펙 설계
- **산출물**: `docs/design-system/HOMEPAGE_REDESIGN_SPEC.md`
- **전달할 프롬프트 초안**:
  > "기존 텍스트 위주의 홈페이지를 완전히 새롭게 개편하기 위한 화면설계서를 `docs/design-system/HOMEPAGE_REDESIGN_SPEC.md`에 작성해 주세요. Hero 섹션, 기능 소개, 서비스 강점, Footer 등으로 구성되며, 고품질 이미지/사진들이 잘 어우러지는 트렌디한 레이아웃으로 설계해야 합니다. 'MindGarden' 브랜딩은 제거하고 'Core Solution - 비즈니스의 핵심을 솔루션하다'를 강조해 주세요. 화면 여백, 색상, 타이포그래피 등은 프로젝트 내의 `var(--mg-*)` 디자인 토큰 및 아토믹 디자인 규칙을 반드시 반영하여 명세해야 합니다."

### Phase 2: 홈페이지 리뉴얼 코드 구현
- **담당**: `core-coder`
- **목표**: Phase 1의 디자인 스펙을 바탕으로 React 컴포넌트 및 CSS 파일 구현
- **산출물**: 리팩토링된 `frontend/src/components/homepage/Homepage.js` 및 `Homepage.css`
- **전달할 프롬프트 초안**:
  > "`docs/design-system/HOMEPAGE_REDESIGN_SPEC.md`에 명시된 디자인 스펙에 따라 `frontend/src/components/homepage/Homepage.js`와 관련 CSS를 완전히 새롭게 구현해 주세요. 멋진 고품질 이미지를 적극 활용하는 랜딩 페이지 구조여야 하며, 기존 'MindGarden' 문구를 모두 삭제하고 'CoreSolution' 브랜딩을 적용해야 합니다. 스타일링 시 하드코딩을 피하고 반드시 `var(--mg-*)` 디자인 토큰을 사용하세요. 기존 코드에 있던 역할별 대시보드 리다이렉션 등 필수 기능 로직은 손상 없이 이식해야 합니다."

## 6. 리스크 및 제약
- **성능 문제**: 고해상도 이미지 사용에 따른 초기 로딩 속도 저하 가능성 고려 (이미지 최적화 혹은 webp 등 효율적인 포맷 사용 권장)
- **기능 유실**: 기존 라우팅 구조나 다른 페이지로의 접근성(로그인/회원가입, 대시보드 리다이렉트 등)이 깨지지 않도록 연결부를 철저히 유지해야 합니다.

- [ ] 'Core Solution', '비즈니스의 핵심을 솔루션하다' 문구가 적절히 반영되었는가?
- [ ] 텍스트 위주 구성에서 벗어나, 고품질 이미지/사진 중심의 Hero, 기능 소개 섹션이 도입되었는가?
- [ ] 모든 인라인 스타일 및 CSS에 `var(--mg-*)` 디자인 토큰이 올바르게 적용되었는가? (하드코딩 확인)
- [ ] 반응형(데스크탑, 태블릿, 모바일) 화면에서 레이아웃이 깨지지 않는가?
- [ ] 로그인, 회원가입, 대시보드 등 필수 기능 버튼과 네비게이션이 정상 작동하는가?
