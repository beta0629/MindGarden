# 온보딩 & 자동결제 운영 표준 절차 (SOP Draft)

작성일: 2025-11-13  
작성자: 코어(Core) 솔루션 운영 포털 v1 팀

---

## 1. 목적
- 외부 온보딩 신청부터 테넌트 승인, 자동 결제, 정산까지의 운영 표준 절차를 정의한다.
- HQ 운영팀, 고객 성공팀, 회계팀이 일관된 프로세스로 협업할 수 있도록 가이드라인을 제공한다.
- 감사(Audit)·보안·고객 커뮤니케이션 정책을 명확히 하여 리스크를 최소화한다.

---

## 2. 역할 정의

| 역할 | 책임 | 주요 시스템 |
| --- | --- | --- |
| **HQ 운영팀** | 온보딩 심사, 승인/반려 결정, 결제 상태 모니터링 | Ops 포털 `/onboarding`, `/billing` |
| **고객 성공팀** | 고객 커뮤니케이션, 결제 실패 대응, 업셀링 제안 | Ops 포털, Slack/이메일 |
| **회계/재무팀** | 정산 검증, 세금계산서 발행, 수익/환불 보고, 법규 준수 확인 | 회계 시스템, Billing 리포트 |
| **시스템 자동화** | 정기 결제 배치, 알림 발송, 감사 로그 기록 | 배치 서버, Notification 서비스 |

## 3. End-to-End 운영 절차

1. **신규 온보딩 요청 도착**
   - 외부 폼 → `POST /api/v1/onboarding/requests`
   - Ops 포털 `온보딩 대기` 목록에 표시, Slack 알림(운영 채널)
2. **초기 스크리닝 (HQ 운영팀)**
   - 온보딩 체크리스트 검토 (사업자 정보, 업종, 위험도 등)
   - 필수 동의 사항(서비스 이용 약관, 개인정보 수집·이용/처리 위탁, 전자금융거래 약관, 광고성 정보 수신 여부 등) 수집 여부 확인
   - 필수 입력 항목(사업자등록증, 대표자 정보, 정산 계좌 등)이 저장되었는지 확인하고 누락 시 다음 단계 진행 금지
   - 추가 서류 필요 시 고객 성공팀을 통해 자료 요청
3. **결제 수단 확인**
   - 결제·구독 도메인 설계서에 정의된 토큰 상태 확인
   - 결제 토큰 검증 실패 시 고객에게 재등록 안내
4. **승인/반려 결정**
   - Ops 포털에서 `APPROVED / REJECTED / ON_HOLD` 상태 선택
   - 승인 시: 구독 생성 + 첫 결제 실행 → 성공 여부 확인
   - 반려 시: 반려 사유 기록, 고객 통보(자동 이메일 + CS 팀)
5. **테넌트 활성화**
   - 결제 성공 시 `ACTIVE` 상태, 코어(Core) 솔루션 SaaS 접근 권한 부여
   - 커미션 테이블 업데이트, 감사 로그 기록
6. **정기 모니터링**
   - 매일 오전 자동 리포트 (결제 성공/실패 목록)
   - 실패 건은 1/3/7일차 자동 재시도 → 실패 시 `SUSPENDED` 전환
   - 고객 성공팀이 연락하여 결제 수단 갱신/해지 여부 확인
7. **정산/세금계산서**
   - 월말 인보이스/정산 리포트 생성
   - 회계팀이 발행 승인 → PG 정산 데이터와 대조
8. **해지/업그레이드**
   - 고객 요청 접수 → Ops 포털에서 요금제 변경/취소 처리
   - 환불 정책에 따라 부분/전체 환불 진행, 감사 로그 기록

## 4. 커뮤니케이션 템플릿

### 4.1 승인 안내 (자동 이메일 + CS 확인)
- 제목: `[코어(Core) 솔루션] 입점 신청이 승인되었습니다`
- 내용: 서비스 시작 가이드, 로그인 URL, 첫 결제 내역, 고객 성공팀 담당자

### 4.2 반려 안내
- 제목: `[코어(Core) 솔루션] 입점 심사 결과 안내`
- 내용: 반려 사유, 재신청 가능 여부, 문의 채널

### 4.3 결제 실패 알림
- 1차: 자동 이메일 + SMS (선택), 계정 관리자 대상
- 2차: 고객 성공팀 전화/채팅
- 3차 이후: SUSPENDED 안내 및 이용 제한 공지

---

## 5. 감사 로그 & 모니터링
- 모든 상태 변경, 결제 재시도, 환불, 커미션 조정은 `OpsAuditLog`에 기록
- 감사 로그는 테넌트별 필터 가능, 최소 5년 보관
- 운영 리포트:
  - 일일 결제 성공률/실패률
  - 월간 신규 테넌트/해지 요약
  - 커미션 누적액
  - 결제 실패 사유 분석(카드 한도, 인증 실패 등)

---

## 6. 장애 대응 플랜
- **PG 장애**: PG 상태 페이지 확인 → 고객 공지 → 수동 결제 안내 (Ops Slack 채널 템플릿)
- **정산/결제 배치 실패**: 배치 모니터링 경고 → DevOps에게 즉시 승격 → 수동 재실행
- **Webhook 누락**: PG 관리자 콘솔에서 이벤트 재전송 → 시스템 로그 확인
- **중대 보안 사고**: 보안팀/경영진 즉시 보고, 결제 시스템 차단, 고객 공지 준비

---

## 7. KPI / 리포트 항목
- 온보딩 승인율, 평균 승인 소요 시간
- 결제 성공률, 결제 실패 재시도 성공률
- SUSPENDED → ACTIVE 복귀율
- 해지 사유별 비중
- MindGarden 커미션 수익 추이
- 고객 만족도(NPS) / CS 티켓 처리 SLA

## 8. 국내 법규 준수 체크리스트
> 적용 법령: 「전자상거래 등에서의 소비자보호에 관한 법률」, 동 시행령, 「소비자분쟁해결기준」 등

- **정보 제공 의무**
  - 청약 전 고지 사항(요금제·애드온 구성, MindGarden 커미션, 이용 제한, 환불 조건)을 신청 페이지/약관에 명시
  - 전자 금융거래 약관 및 개인정보 처리방침에 PG 위탁 내역, 개인정보 수집·이용 목적과 보관 기간 명시
- **청약철회·환불 처리**
  - 서비스 제공 전: 결제일로부터 7일 이내 청약철회 시 전액 환불
  - 서비스 이용 후: 이용 일수·사용량에 따른 부분 환불 기준을 수립하고 약관에 고지
  - 환불 요청 접수 후 3영업일 이내 처리, 결과(금액·수단·일자)를 OpsAuditLog와 `ops_subscription_payment`에 기록
- **전자영수증·세금계산서**
  - 발급 의무 준수, 발급 내역을 5년간 안전하게 보관
  - B2B 거래 시 세금계산서 자동 발행 및 회계 시스템 연계
- **자료 보존**
  - 거래 기록·정산 자료·고객 상담 이력 최소 5년 보존, 개인정보는 법정 보존 기간 이후 즉시 파기
- **개인정보 처리방침 준수**
  - 개인정보 수집·이용 동의, 제3자 제공·처리 위탁 내용을 개인정보 처리방침에 명확히 반영
  - 정보주체 권리(열람·정정·삭제·처리정지 요청)에 대한 처리 절차와 연락처 고지
  - 개인정보 유출/침해 사고 발생 시 사내 보안 정책 지침에 따른 통지 및 신고 절차 실행
- **동의 관리**
  - 서비스 이용 약관, 개인정보 수집·이용/제3자 제공, 전자금융거래 이용약관, 광고성 정보 수신(선택)에 대해 전자적 방식으로 동의 받기
  - 동의 일시·버전·채널을 DB에 저장하고, 향후 철회 요청 시 이력 조회 가능하도록 관리
  - 필수 서류/정보가 저장되지 않고 동의만 진행한 경우 워크플로우가 다음 단계로 이동하지 않도록 프런트/백엔드에서 검증
  - 동의 철회 처리 시 관련 서비스 제공 범위를 즉시 조정하고, OpsAuditLog에 기록
- **AI 기반 침해 사고 예방**
  - 이상 징후 탐지를 위해 AI/머신러닝 기반 보안 관제(예: 로그인 패턴 분석, 비정상 결제 시도 탐지) 도입 검토
  - AI 모델이 개인정보를 처리하는 경우, 데이터 최소화·암호화·익명화를 적용하고 관련 동의/처리방침에 반영
  - AI 탐지 결과는 SIEM/보안팀으로 전달하여 2차 검증 후 대응하며, 오탐 시 학습 데이터 개선

- **고객 문의·분쟁 대응**
  - 고객센터 연락처와 처리 절차를 명확히 고지, 분쟁 발생 시 한국소비자원 기준에 따른 처리
- **정기 결제 해지**
  - 해지 요청 즉시 후속 청구 중단, 해지 안내(이메일/SMS) 발송 및 상태 `CANCELLED`/`TERMINATED` 반영
- **법령 모니터링**
  - 분기별로 공정거래위원회·관계 부처 공지 확인, SOP/약관 최신화
  - 법령 변경 시 운영팀 교육 및 시스템 점검 계획 수립

## 9. 문서/시스템 연계
- 별첨 1. 온보딩~자동결제 End-to-End 로드맵
- 별첨 2. 결제·구독 도메인 설계서
- 별첨 3. PG 연동 가이드 및 법규 대응 계획

## 10. 향후 개선 과제
1. Ops 포털 내 SOP 체크리스트 자동화 (승인 단계별 체크)
2. 결제 실패 시 챗봇/AI 기반 가이드 제공
3. 회계 시스템과 API 기반 연동 (영수증/세금계산서 자동 발행)
4. 해지/업그레이드 워크플로우 자동화 (Self-service Portal)
5. Zero-touch Onboarding 실현을 위한 자동 승인 룰 엔진 설계
