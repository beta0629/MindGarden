## 1. 스토리라인 (제품이 뭔지 모르는 사람에게)

한 문장:

> Core Solution은 **상담센터의 매칭·일정·회기·결제**를 한곳에서 운영하고, **상담사·내담자 앱**으로 당일 일정과 안내를 이어 주는 상담센터 운영 플랫폼입니다.

권장 전체 런타임 **약 3분 30초~5분** (편집 후). 숏폼은 §5.

### 3.3 클릭 동선 (촬영 시 따라할 순서)

**웹 어드민**

1. 로그인 → **홈** (다음 상담·오늘 스케줄)  
2. 탭 **스케줄**  
3. 탭 **더보기** → **알림**(또는 홈 벨 아이콘) — 목록만 훑고 상세 본문은 깊게 안 들어감

**Expo 내담자**

1. 로그인 → **홈**  
2. 탭 **내 상담**  
3. 탭 **더보기** → **회기 · 결제** (목록/잔여 회기만)

## 4. 컷별 테이블

녹화 초는 **한 컷 순수 화면 체류** 기준(클릭·로딩 포함). 편집 시 앞뒤 0.3~0.5초 트림.

### 4.0 싱크 원칙 (녹화 · 내레이션 · 자막)

1. **한 표에서 맞춘다**: 아래 컷 `#`별 **녹화 초 · 내레이션 · 자막(한국어) · 자막 IN–OUT**을 같은 행 기준으로 촬영·더빙·자막 모두 맞춘다. 컷 상대 `IN–OUT`은 해당 컷 **편집 타임라인 시작점 = 0:00**이다.
2. **전체 타임라인은 §4.2**: 본편 편집·SRT는 §4.2 연속 타임코드를 쓴다. 컷 길이(§5.1 편집 초)를 바꾸면 §4 표·§4.2를 **함께** 갱신한다.
3. **AI B-roll도 타임코드 유지**: 구글 옴니/Veo 등 B-roll·전환을 끼울 때는 화면만 교체하고, **자막·내레이션 IN–OUT(전체 타임코드)은 그대로** 둔다. B-roll이 길면 앞뒤 화면을 트림해 **같은 구간 길이**를 맞춘다.
