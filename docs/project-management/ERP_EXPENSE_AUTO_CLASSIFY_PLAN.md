# 비용만 저장하면 알아서 분류·분개 — "ERP를 몰라도 쓰기"

**작성일**: 2026-02-12  
**목표**: 사용자가 "물건 산 비용"처럼 **비용만 DB에 저장**하면, 시스템이 **자동으로 분류·분개**해서 ERP를 몰라도 쓸 수 있게 한다.

---

## 1. 이미 있는 것 (현재 구조)

| 흐름 | 사용자 행동 | 시스템 동작 |
|------|-------------|-------------|
| **빠른 지출** (QuickExpenseForm) | 카테고리(급여/임대료/관리비/세금/사무용품/마케팅 등) 선택 → 금액 입력 | `FinancialTransaction`(EXPENSE) 저장 → `createJournalEntryFromTransaction()` 호출 → **분개 자동 생성·전기** |
| **거래 등록** (FinancialTransactionForm) | 수입/지출 선택, 금액·설명·일자 입력 | 동일: 저장 시 **자동 분개** |
| **입금 확인** | 매칭 입금 확인 버튼 | INCOME 거래 생성 → 자동 분개 |

→ **"비용을 DB에 저장하면 자동으로 분개된다"** 는 이미 구현되어 있음. (`FinancialTransactionServiceImpl.createTransaction()` 144~148행)

---

## 2. 사용자 관점에서 부족한 점

- **진입점이 "ERP/재무"에 묶여 있음**: 빠른 지출·거래 등록이 "수입·지출 관리" 화면 안에 있어서, "그냥 비용 썼다 기록하고 싶다"가 잘 드러나지 않을 수 있음.
- **분류 = 카테고리 선택**: 지금은 사용자가 **카테고리(급여/사무용품 등)를 직접 고르면** 그대로 저장·분개됨. "물건 샀다"만 적고 **시스템이 알아서 분류**해 주는 단계는 아직 없음(선택 사항).

---

## 3. 제안: "비용만 저장 → 알아서 분류" 확장

### 3.1 단순 비용 등록 강조 (바로 적용 가능)

- **메뉴/대시보드**: "비용 썼어요", "지출 기록"처럼 **ERP/회계 용어 없이** 진입점 노출.
- **동일 백엔드**: 기존 `FinancialTransaction`(EXPENSE) + `createJournalEntryFromTransaction()` 그대로 사용.
- **UI**:  
  - "무엇에 썼는지"(한 줄 메모 또는 카테고리 선택), "얼마"만 입력.  
  - 저장 시 기존 API(quick-expense 또는 finance/transactions) 호출 → DB 저장 → **자동 분개** (현재와 동일).

→ 사용자는 "비용 등록"만 알면 되고, ERP·분개·계정과목은 보이지 않음.

### 3.2 자동 분류(선택 사항)

- **의미**: "물건 산 비용"처럼 **설명만** 저장해 두면, 시스템이 **카테고리(또는 계정)를 추정**해 주는 것.
- **방식 예시**  
  1. **기본값**: 카테고리 미선택 시 "기타 지출" 등 기본 카테고리로 저장 → 기존처럼 EXPENSE 1개 계정으로 분개.  
  2. **키워드 규칙**: 설명에 "물건", "비품", "사무용품" 등이 포함되면 특정 카테고리(또는 공통코드)로 매핑.  
  3. **카테고리 → 계정 매핑**: 공통코드(예: OFFICE_SUPPLIES)와 ERP 계정과목(EXPENSE 또는 세부 계정) 매핑 테이블을 두고, 저장 시 **카테고리만 넣으면 계정은 시스템이 선택**하도록 확장.

→ "디비에 저장만 해 두면 알아서 분류"는 (1)은 지금도 가능, (2)(3)은 기획 확정 후 단계적으로 추가 가능.

---

## 4. 정리

- **질문**: "물건 산 비용을 DB에 저장해 두면 알아서 분류해 주면 ERP를 몰라도 쓰지 않을까?"  
  → **맞음.** 그런 흐름을 쓰려면:
  1. **지금**: "비용 등록"을 ERP 노출 없이 쓰기 쉽게 (빠른 지출/거래 등록 진입점·문구 정리).
  2. **저장**: `FinancialTransaction`(EXPENSE) 저장 → **이미 자동 분개**되므로, "비용만 저장하면 분개까지 된다"는 전제 유지.
  3. **추가(선택)**: "분류" = 카테고리 기본값·키워드 규칙·카테고리→계정 매핑으로, 사용자는 최소한만 입력하고 나머지는 시스템이 채우기.

이 문서는 기획·디자이너·코더와 상의 시 참고용이다. 구현 우선순위는 제품 기획에 따라 정하면 된다.
