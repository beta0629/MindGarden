# 운영반영 폴더 분리·문서 정리 계획서

**문서 유형**: 기획(플래너) 산출 — 실행 위임용  
**작성일**: 2025-03-17  
**목표**: "운영 반영(배포 준비)" 관련 문서를 한곳으로 모으기 위해 전용 폴더를 만들고, 기존 문서를 이전·정리한 뒤 링크를 수정한다. 실제 폴더 생성·이동은 **shell**, 마크다운 링크·문서 수정은 **문서관리 서브에이전트**(generalPurpose + /core-solution-documentation)에게 위임한다. 문서 정리·링크 수정은 코더가 아닌 문서 전담이 수행한다. (참조: `docs/standards/SUBAGENT_USAGE.md` — 문서 작성·정리·체계화, 문서 이동·구조 정리는 문서관리 서브에이전트 전담.)

---

## 1. 요구·배경

- 운영 서버 반영 준비 회의 산출물이 `docs/planning/`, `docs/project-management/`, `docs/debug/`에 흩어져 있어 한곳에서 찾기 어렵다.
- 회의 결론·테스터/디버거/셸 의견서·표준 참조를 한 진입점에서 볼 수 있도록 정리하고, 배포 실행 시 참조하는 표준·가이드는 기존 위치를 유지하며 링크로 연결한다.
