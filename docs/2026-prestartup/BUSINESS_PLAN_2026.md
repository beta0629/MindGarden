# 2026 예비창업자 사업계획서 (CoreSolution)

> **작성 기준**: 2025-12 현재 구현을 바탕으로 작성(현행 시스템 기술 근거는 `docs/2026-prestartup/` 및 `docs/standards/` 참조)

## I. 일반 현황

### 1) 창업(아이템) 개요

**CoreSolution**은 상담센터/교육기관/1인 서비스업 등 “서비스 업종”의 운영에 필요한 핵심 기능을 **하나의 SaaS**로 제공하는 통합 운영 플랫폼입니다.

핵심은 다음 두 축입니다.

1. **멀티테넌트 운영 플랫폼**: 테넌트(기관/지점) 단위 데이터 격리 + 본사/관리자 통제 + 운영 표준화
2. **AI 확장 모듈(심리검사 리포트 AI)**: TCI/MMPI 우선 — 스캔 PDF를 업로드하면 **표준화 지표/검증/근거 기반 리포트**로 변환하여 상담 품질과 운영 효율을 향상

### 2) 팀 구성(초안)

- 대표(기술): 플랫폼 아키텍처, 멀티테넌트/권한/보안 표준, 백엔드/프론트엔드 핵심 구현
- 디자인/기획: 운영자 UX, 표준 컴포넌트/디자인 토큰 기반 UI 고도화
- (계획) 추가 개발/보안: 운영 안정화, 보안 점검, AI 추출 파이프라인 고도화

## II. 문제 인식(Problem)

### 1) 시장의 Pain Point

상담센터/교육기관/1인 서비스업은 공통적으로 다음 문제를 겪습니다.

- **파편화된 운영**: 예약/상담기록/결제/정산/고객관리 도구가 분리되어 수기 업무가 누적
- **정산/운영 리스크**: 환불/수수료/부분결제 등 예외 케이스가 많아 재무/세무 리스크 증가
- **확장성 부재**: 다지점/프랜차이즈화 시 본사 통제(권한/정책/지표)가 어려움
- **상담 도메인 특화 리스크(추가)**: 심리검사 결과를 사람이 수기로 정리하면 시간·품질 편차가 크고, 표준화된 기록/근거가 부족해 케이스 관리가 어려움

## III. 실현 가능성(Solution)

### 1) 해결책 개요

CoreSolution은 “운영 통합 + 표준화 + AI 보조”로 문제를 해결합니다.

- 운영 통합: 예약/상담/결제/정산/고객관리/대시보드
- 표준화: 공통코드/동적 조회, 필드 기반 저장, 표준 응답/에러 처리, 배포/보안 표준
- AI 확장(심리검사 리포트): 문서 업로드→암호화 저장→추출/검증→리포트 생성→지표 저장/통계

### 2) 핵심 기능(현재 구현 + 고도화 계획)

#### (A) 멀티테넌트 운영 플랫폼
- 테넌트 단위 데이터 격리(tenant_id)
- 역할/권한 기반 접근 제어(동적 권한 확장 가능)
- 운영 대시보드(지표 조회/관리 화면)

#### (B) 심리검사 리포트 AI (TCI/MMPI 우선)
- 관리자 화면에서 PDF 드래그&드롭 업로드
- 원본 PDF **암호화 저장(AES-GCM)**
- 추출/검증 레코드 생성(누락/범위 이상 등)
- 리포트 생성(규칙 기반 초안 + LLM 보강)
- **오판 방어**: 구조 검증, 근거(scaleCode) 매칭, 금지문구 탐지, 실패 시 폴백 + needsReview

> 기술 근거 문서:  
> - 시스템 개요: `docs/2026-prestartup/SYSTEM_OVERVIEW.md`  
> - 아키텍처: `docs/2026-prestartup/TECH_ARCHITECTURE.md`  
> - 보안/AI 안전: `docs/2026-prestartup/SECURITY_PRIVACY.md`

### 3) 표준화 기반 구현 체계(근거)

본 제품은 “표준화 문서”를 통해 설계/운영 원칙을 문서화하고, 구현을 표준에 맞춰 진행합니다.

- 보안/인증: `docs/standards/SECURITY_AUTHENTICATION_STANDARD.md`
- DB 스키마: `docs/standards/DATABASE_SCHEMA_STANDARD.md`
- API 설계/에러: `docs/standards/API_DESIGN_STANDARD.md`, `docs/standards/API_ERROR_HANDLING_STANDARD.md`
- 환경변수/키관리: `docs/standards/ENVIRONMENT_VARIABLE_STANDARD.md`
- 파일/암호화: `docs/standards/FILE_STORAGE_STANDARD.md`, `docs/standards/ENCRYPTION_STANDARD.md`
- 배포: `docs/standards/DEPLOYMENT_STANDARD.md`

## IV. 성장 전략(Scale-up)

### 1) 시장 진입 전략

- 1차(2026): 상담센터 중심 실증(심리검사 리포트 AI 포함) + 운영 플랫폼 안정화
- 2차(2027): 학원/교육으로 확장(다지점/정산/권한)
- 3차(2028): 1인 서비스업/IT센터 등으로 템플릿 확장

### 2) 경쟁 우위(차별성)

- **테넌트 격리 + 본사 통제 + 표준화**를 기본값으로 제공
- 심리검사 AI를 “단순 요약”이 아니라 **근거/검증/오판 방어/암호화 저장**까지 포함한 운영 모듈로 제공

### 3) 로드맵(요약)

상세 로드맵: `docs/2026-prestartup/PRODUCT_ROADMAP.md`

## V. 수익 모델(Revenue Model) — 초안

- **SaaS 구독(MRR)**: Basic/Pro/Enterprise(테넌트 규모/기능/지점 수 기준)
- **사용량 기반 과금**: 알림/문서 처리/AI 리포트 생성 등(정책 확정 필요)
- **커스터마이징/SI**: 초기 레퍼런스 확보 및 현금흐름 보강

## VI. 실행 계획(2026) — 초안

### 1) 제품/기술
- 운영 플랫폼 안정화(권한/대시보드/표준화)
- 심리검사 AI 고도화: 템플릿 감지 + OCR/추출 정확도 향상 + 검증 규칙 강화
- 품질 지표 수집(검증 실패율/사람 검수율/수정율) 기반 개선 루프

### 2) 시장/고객
- PoC 고객 확보 및 실증(업종별 KPI 정의)
- 유료 전환 기준 확립(사용량/기능/규모 기반 요금제)

## VII. 리스크 및 대응

### 1) 보안/개인정보 리스크
- 대응: 암호화 저장(AES-GCM), 환경변수 키관리, 최소권한, 감사/로그 표준 준수

### 2) AI 오판/환각 리스크
- 대응: 구조 검증 + 근거 매칭 + 금지문구 + 폴백 + needsReview(사람 검수)

### 3) 양식 다양성(병원/센터별 PDF)
- 대응: 템플릿 감지/필드 추출 체계화, 검증 엔진 강화, 단계적 커버리지 확대
