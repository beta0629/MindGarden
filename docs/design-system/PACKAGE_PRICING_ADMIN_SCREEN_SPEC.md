**대상**: 관리자 전용 패키지 가격·회기 수 CRUD 화면

**비주얼 기준**: [어드민 대시보드 샘플](https://mindgarden.dev.core-solution.co.kr/admin-dashboard-sample)

## 1. 화면 개요

| 항목 | 내용 |
|------|------|
| **화면명** | 패키지 요금 관리 (또는 요금·가격 관리) |

## 2. 레이아웃 구조

전체는 **기존 어드민 레이아웃**을 따릅니다.

- **좌측**: 고정 사이드바(약 260px, 다크 `#2C2C2C`), 로고·메뉴(활성: 주조색 `#3D5246`)
- **메인**: 상단 바 + 본문. 본 문서는 **메인 영역 내부**만 상세 정의.

### 2.1 메인 영역 구성 (위→아래)

1. **상단 바(페이지 헤더)**  
   - 배경: `#FAF9F7` (또는 `var(--mg-color-bg-page)` 등 토큰으로 통일 시 동일 톤)  
   - 하단 구분선: 1px `#D4CFC8`  
   - 좌측: (선택) 브레드크럼 + **페이지 제목**  
   - 우측: **주요 액션 버튼** 1개

2. **본문**  
   - 패딩: 24px ~ 32px  
   - 단일 **섹션 블록** 1개 안에 **패키지 목록 테이블**만 배치 (이 페이지는 목록이 핵심이므로 섹션 1개로 충분)

3. **모달**  
   - 목록 위 오버레이로 표시. **UnifiedModal**만 사용.

## 3. 블록·컴포넌트 구성

### 3.1 상단 바 (페이지 헤더)

| 요소 | 스펙 |
|------|------|
| **페이지 제목** | 문구: "패키지 요금 관리" 또는 "요금·가격 관리" |
| **부제목(선택)** | 예: "상담 패키지별 가격과 회기 수를 관리합니다" — 캡션 스타일 |
| **제목 타이포** | 20px ~ 24px, fontWeight 600, 색상 `#2C2C2C` (본문 텍스트). Noto Sans KR. |
| **부제목 타이포** | 12px, `#5C6B61` (보조 텍스트) |
| **액션 버튼** | 라벨: "새 패키지 추가". 주조 버튼(배경 `#3D5246`, 텍스트 `#FAF9F7`). height 40px, padding 10px 20px, border-radius 10px. 우측 정렬. |

**클래스·토큰 참고**  
- 기존 어드민 상단 바와 동일한 래퍼 사용 (예: `mg-v2-ad-b0kla__page-header` 유사 패턴).  
- 버튼: 주조 스타일 — `mg-v2-ad-b0kla` 영역의 primary button 클래스 또는 `var(--mg-*)` 토큰.

---

### 3.2 본문 — 섹션 블록 + 목록 테이블

**섹션 블록 1개**로 본문 전체를 감쌈.

| 속성 | 값 |
|------|-----|
| 배경 | `#F5F3EF` 또는 `#FAF9F7` |
| 테두리 | 1px solid `#D4CFC8` |
| border-radius | 16px |
| 패딩 | 24px |
| 내부 gap | 16px (섹션 제목과 테이블 사이) |

**섹션 제목**  
- 왼쪽 **세로 악센트 바**: 폭 4px, 색상 `#3D5246`, border-radius 2px.  
- 제목 텍스트: 12px, fontWeight 600(굵게), `#2C2C2C`.  
- 예: "패키지 목록" (필수 아님, 목록만 있어도 됨. 사용 시 위 스펙 적용)

**클래스 참고**  
- 블록: `mg-v2-ad-b0kla__card` (기존 AdminDashboardV2, MappingEditModal의 카드와 동일).  
- 섹션 제목: `mg-v2-ad-b0kla__section-title`.  
- 좌측 악센트는 B0KlA 카드/섹션 타이틀 스타일에 이미 포함되어 있으면 그대로 사용.

**목록 테이블**

- 테이블 형태 권장. 컬럼 순서(좌→우):  
  **패키지 코드(또는 라벨)** | **패키지명** | **회기 수** | **가격(원)** | **비고** | **사용 여부** | **작업**
- **작업** 컬럼: 행별 "수정" 버튼 + "비활성화"(또는 "삭제") 버튼 또는 드롭다운 메뉴.
- 테이블 스타일: 어드민 대시보드 샘플·공통코드 관리 테이블과 동일한 톤(헤더 배경, 행 구분선, 패딩).
- **행 클릭**: 행 전체 클릭 시 수정 모달 열기(선택 구현). 수정 버튼만 있어도 됨.
- 금액 컬럼: 우측 정렬, 원 단위 포맷(예: 1,000,000원). 숫자 강조: 14~16px, fontWeight 600, `#2C2C2C`.
- 사용 여부: "사용"/"미사용" 또는 "활성"/"비활성" 텍스트 또는 뱃지. 뱃지 시 기존 `mg-v2-badge` 계열 사용.

**반응형**  
- 목록이 넘치면 가로 스크롤 허용. 또는 좁은 해상도에서 카드형 전환 가능(선택).

### 3.3 등록/수정 모달 (UnifiedModal)

**필수 사항**  
- **UnifiedModal**만 사용. `className="mg-v2-ad-b0kla"` 전달.  
- 커스텀 오버레이/모달 래퍼 사용 금지.

**모달 구조**  
- **헤더**: `mg-modal__header` — 제목(등록 시 "새 패키지 추가", 수정 시 "패키지 수정").  
- **본문**: `mg-modal__body` — 폼 영역.  
- **푸터**: `mg-modal__actions` — 저장(주조 버튼), 취소(아웃라인 또는 보조 버튼).

**모달 크기**  
- `size="medium"` 또는 콘텐츠에 맞춘 `size="auto"`. 폼 필드가 6개 내외이므로 medium 권장.

**폼 필드 (본문)**

| 순서 | 필드명 | 라벨(예) | 타입 | 비고 |
|------|--------|----------|------|------|
| 1 | 패키지 코드 | 패키지 코드 | 텍스트(코드값) | 신규: 입력. 수정: 읽기 전용 가능. |
| 2 | 패키지명 | 패키지명(한글) | 텍스트 | |
| 3 | 회기 수 | 회기 수 | 숫자 입력 | 필수. 0 초과 등 유효성. |
| 4 | 가격 | 가격(원) | 숫자 입력 | 필수. 원 단위. |
| 5 | 비고 | 비고 | 텍스트(한 줄 또는 여러 줄) | 선택. |
| 6 | 사용 여부 | 사용 여부 | 토글 또는 셀렉트(사용/미사용) | 기본값 사용. |

- 라벨 타이포: 12px, `#5C6B61`.  
- 입력 필드: 기존 FormInput/아토믹 디자인 입력 컴포넌트, 테두리 `#D4CFC8`, focus 시 주조색.  
- 폼 레이아웃: 세로 스택, 필드 간 간격 16px.  
- 저장/취소: 우측 정렬. 저장 = 주조(#3D5246), 취소 = 아웃라인(#D4CFC8).

**모달 내부 섹션(선택)**  
- 폼이 길면 "기본 정보"처럼 묶을 수 있음. 이때 모달 내부에도 `mg-v2-ad-b0kla__card` + `mg-v2-ad-b0kla__section-title` 조합 사용 가능.

## 5. 아토믹 디자인 매핑(참고)

| 계층 | 이 화면에서 사용할 요소 |
|------|-------------------------|
| **Atoms** | Button(주조/아웃라인), Input, Label, Badge(사용 여부), Icon |
| **Molecules** | FormField(Label+Input), 테이블 행(셀 묶음) |
| **Organisms** | 페이지 헤더(제목+버튼), 섹션 블록(제목+테이블), 패키지 목록 테이블, 등록/수정 모달 폼 |
| **Templates** | 어드민 메인 템플릿(사이드바 + 상단 바 + 본문) |
| **Pages** | 패키지 요금 관리 페이지(목록 + 모달) |

## 6. 사용성·제약 반영

- **자주 쓰는 동작**: 목록에서 수정(행 클릭 또는 "수정" 버튼 → 모달). 따라서 "수정" 버튼은 행마다 명확히 두기.
- **새 패키지 추가**: 상단 "새 패키지 추가" 버튼 1개.
- **비활성화/삭제**: 행별 버튼 또는 드롭다운. 기존 매핑에서 사용 중인 패키지는 경고 후 처리(정책은 기획서 §2.1).
- **관리자·테넌트**: 메뉴/라우트/API는 관리자만 접근, 테넌트별 데이터만 노출(구현은 Phase 2).
